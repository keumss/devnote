## functools는 언제 쓰는 도구인가요?

`functools`는 함수를 인자로 받거나 새 함수를 만드는 Python 3.12 표준 라이브러리입니다. 같은 함수를 조금 다른 설정으로 여러 번 호출하거나, 함수 앞뒤에 공통 처리를 붙이거나, 계산 결과를 다시 쓰고 싶을 때 사용합니다.

이 섹션에서는 주문 금액을 처리하는 작은 함수를 공통 예제로 사용합니다. 먼저 함수도 변수에 담고 호출할 수 있는 값이라는 점을 확인합니다.

```python
def apply_discount(amount: int, rate: float) -> int:
    return round(amount * (1 - rate))

calculator = apply_discount
print(calculator(12000, 0.1))
```

```text
10800
```

함수를 값처럼 전달할 수 있어야 `partial()`, `wraps()`, 캐시 데코레이터 같은 도구를 조합할 수 있습니다.

## partial로 자주 쓰는 인자를 고정하기

같은 할인율을 반복해서 전달할 때 `partial()`로 일부 인자를 미리 채운 호출 가능 객체를 만듭니다. 이벤트 핸들러나 `map()`에 넘길 함수의 입력을 단순하게 만들 때도 알맞습니다.

```python
from functools import partial

def apply_discount(amount: int, rate: float) -> int:
    return round(amount * (1 - rate))

member_price = partial(apply_discount, rate=0.1)

print(member_price(12000))
print(member_price(25000))
```

```text
10800
22500
```

`partial()`에 준 위치 인자는 새 호출의 앞에 붙고, 키워드 인자는 새 호출에서 같은 이름으로 주면 덮어씁니다. 함수의 동작을 숨기기보다, 반복되는 고정 설정을 이름으로 드러낼 때 사용합니다.

## partial에 위치 인자를 고정할 때 순서를 확인하기

위치 인자는 왼쪽부터 채워집니다. 그래서 원래 함수의 인자 순서를 먼저 보고 `partial()`에 넘깁니다.

```python
from functools import partial

def format_price(currency: str, amount: int) -> str:
    return f'{currency} {amount:,}'

won = partial(format_price, 'KRW')

print(won(12000))
```

```text
KRW 12,000
```

`partial(format_price, 12000)`은 통화 자리에 숫자를 넣으므로 원하는 함수가 아닙니다. 어떤 인자를 고정하는지 분명하지 않다면 앞 예제처럼 `rate=0.1`처럼 키워드 인자로 고정하는 편이 읽기 쉽습니다.

## reduce로 여러 값을 하나의 결과로 누적하기

왼쪽부터 두 값을 합쳐 하나의 결과로 줄여야 할 때 `reduce()`를 사용합니다. 합계처럼 내장 함수로 더 분명하게 쓸 수 있는 경우는 `sum()`을 우선하고, 누적 규칙 자체가 핵심일 때 선택합니다.

```python
from functools import reduce

amounts = [12000, 8000, 15000]
total = reduce(lambda subtotal, amount: subtotal + amount, amounts, 0)

print(total)
```

```text
35000
```

세 번째 인자 `0`은 초기값입니다. 빈 목록에서도 `0`을 반환하게 하므로, 입력이 비었을 수 있는 합계에는 초기값을 명시합니다.

## reduce로 변경 이력을 한 번에 적용하기

누적값과 다음 입력을 합치는 규칙이 복잡할 때는 이름 있는 함수를 만들면 `reduce()`의 역할이 보입니다. 아래에서는 장바구니 금액에 증감 값을 순서대로 적용합니다.

```python
from functools import reduce

def apply_change(balance: int, change: int) -> int:
    return balance + change

changes = [12000, 8000, -3000]
print(reduce(apply_change, changes, 0))
```

```text
17000
```

중간 잔액도 모두 필요하다면 최종 하나만 주는 `reduce()` 대신 `itertools.accumulate()`를 사용합니다. 누적 과정에서 오류를 처리하거나 조건에 따라 멈춰야 하면 일반 `for` 문이 더 읽기 쉽습니다.

## 데코레이터는 함수를 감싸서 새 함수로 바꿉니다

여러 함수에 같은 기록이나 검증을 추가할 때 데코레이터를 만듭니다. 데코레이터는 원본 함수를 받아 `wrapper`라는 새 함수를 반환합니다.

```python
def announce(func):
    def wrapper(*args, **kwargs):
        print(f'run: {func.__name__}')
        return func(*args, **kwargs)

    return wrapper

@announce
def create_order(order_id: str) -> str:
    return f'created {order_id}'

print(create_order('A-1'))
```

```text
run: create_order
created A-1
```

`*args`, `**kwargs`를 받으면 여러 시그니처의 함수를 감쌀 수 있습니다. 하지만 이 상태의 `create_order.__name__`은 원본 이름이 아니라 `wrapper`가 되어, 로그·문서화·테스트 도구에서 원본 정보를 잃습니다.

## wraps로 원본 함수의 정보를 보존하기

직접 만든 데코레이터의 `wrapper`에는 항상 `@wraps(func)`를 붙입니다. 함수 이름, 설명, 애너테이션을 복사하고 원본으로 갈 수 있는 `__wrapped__`도 설정합니다.

```python
from functools import wraps

def announce(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        print(f'run: {func.__name__}')
        return func(*args, **kwargs)

    return wrapper

@announce
def create_order(order_id: str) -> str:
    """주문을 만든다."""
    return f'created {order_id}'

print(create_order.__name__)
print(create_order.__doc__)
```

```text
create_order
주문을 만든다.
```

`wraps()`는 동작을 바꾸는 도구가 아니라 감싼 함수가 원본처럼 보이게 하는 도구입니다. 캐시, 권한 검사, 로깅처럼 함수를 감싸는 모든 자체 데코레이터에 기본으로 적용합니다.
