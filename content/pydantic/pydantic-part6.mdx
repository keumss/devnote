이번에는 실행 환경과 `.env`에서 애플리케이션 설정을 읽고, 타입이 잘못됐거나 필수 비밀값이 없으면 시작 전에 실패하는 로더를 만듭니다. 예제의 `.env`는 학습용 형식이며 실제 자격 증명 파일은 버전 관리에 추가하지 않습니다.

## pydantic-settings 설치하고 AppSettings 선언하기

환경 변수와 설정 파일을 Pydantic 모델로 읽을 때 별도 `pydantic-settings` 패키지의 `BaseSettings`를 사용합니다. 데이터베이스 설정은 중첩 모델로 나누고 API 키는 출력에서 값을 가리는 `SecretStr`로 선언합니다.

```bash
python3.12 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install pydantic-settings
```

`settings.py`에 다음 모델을 준비합니다.

```python
import os
from pathlib import Path
from typing import Literal
from unittest.mock import patch

from pydantic import BaseModel, Field, SecretStr, ValidationError
from pydantic_settings import BaseSettings, SettingsConfigDict


class DatabaseSettings(BaseModel):
    host: str = 'localhost'
    port: int = Field(default=5432, ge=1, le=65_535)
    name: str = 'app'


class AppSettings(BaseSettings):
    model_config = SettingsConfigDict(
        env_prefix='APP_',
        env_nested_delimiter='__',
        env_file='.env',
        env_file_encoding='utf-8',
        extra='forbid',
    )

    environment: Literal['local', 'staging', 'production'] = 'local'
    debug: bool = False
    api_key: SecretStr
    database: DatabaseSettings = Field(default_factory=DatabaseSettings)
```

`BaseSettings`는 생성자에 없는 필드를 설정 소스에서 찾아 검증합니다. `api_key`에는 기본값을 두지 않아 자격 증명이 없는 실행 환경이 조용히 시작되지 않게 합니다.

## env_prefix로 애플리케이션 환경 변수 읽기

다른 프로그램의 환경 변수와 이름이 충돌하지 않게 하려면 `env_prefix='APP_'`를 사용합니다. 필드 이름 앞에 접두사를 붙인 값이 모델 입력이 됩니다.

```bash
export APP_ENVIRONMENT=staging
export APP_DEBUG=true
export APP_API_KEY=test-secret
```

앞 모델 아래에서 설정을 생성해 확인합니다.

```python
settings = AppSettings()
print(settings.environment, settings.debug)
print(settings.api_key)
```

```text
staging True
**********
```

문자열 `'true'`는 bool로 변환되고 허용하지 않은 `environment` 값은 검증 오류가 됩니다. `SecretStr`의 일반 출력은 실제 값을 가리지만, 비밀값 자체의 안전한 저장을 대신하지는 않습니다.

## env_nested_delimiter로 중첩 설정 덮어쓰기

데이터베이스의 일부 값만 실행 환경에서 바꿀 때 이중 밑줄로 중첩 경로를 표현합니다. 나머지 필드는 모델 기본값을 유지합니다.

```bash
export APP_API_KEY=test-secret
export APP_DATABASE__HOST=db.internal
export APP_DATABASE__PORT=6543
```

```python
settings = AppSettings()
print(settings.database.model_dump())
```

```text
{'host': 'db.internal', 'port': 6543, 'name': 'app'}
```

`APP_DATABASE__PORT`는 `database.port`로 해석되어 정수 범위 검증을 받습니다. 중첩 구조가 커져도 환경 변수 이름과 모델 경로가 직접 대응합니다.

## env_file로 로컬 기본 설정 불러오기

로컬 개발에서 반복하는 값을 파일로 준비할 때 `SettingsConfigDict(env_file='.env')`가 dotenv 파일을 읽습니다. 다음 내용은 형식 예시이며 저장소에는 비밀값이 없는 `.env.example`만 둘 수 있습니다.

```dotenv
APP_ENVIRONMENT=local
APP_DEBUG=false
APP_API_KEY=local-placeholder
APP_DATABASE__PORT=5544
```

같은 필드를 실제 환경 변수로 지정하면 환경 변수가 `.env`보다 우선합니다.

```bash
export APP_DEBUG=true
python settings.py
```

```python
settings = AppSettings()
print(settings.debug, settings.database.port)
```

```text
True 5544
```

기본 우선순위는 생성자 인수, 환경 변수, `.env`, secrets 디렉터리, 필드 기본값 순입니다. `.env` 경로는 현재 작업 디렉터리를 기준으로 찾으므로 실행 위치도 함께 정합니다.

## ValidationError로 시작 전에 잘못된 설정 막기

필수 키 누락이나 잘못된 포트를 애플리케이션 실행 중간이 아니라 시작 단계에서 발견하려면 설정 생성을 로더 함수에 모읍니다. 외부 오류에는 실패한 경로만 남기고 비밀 입력값은 포함하지 않습니다.

```python
def load_settings(*, env_file: str | Path | None = '.env') -> AppSettings:
    try:
        return AppSettings(_env_file=env_file)
    except ValidationError as exc:
        paths = [
            '.'.join(str(part) for part in error['loc'])
            for error in exc.errors(include_url=False, include_input=False)
        ]
        raise RuntimeError(f'invalid settings: {", ".join(paths)}') from exc
```

애플리케이션 진입점에서 `load_settings()`를 가장 먼저 호출하면 설정이 유효한 상태에서만 서버나 작업자를 시작합니다. 내부 원인 예외는 연결하되 사용자 로그에 원본 환경 변수 값을 출력하지 않습니다.

## patch.dict로 설정 소스 격리해 테스트하기

개발자 컴퓨터의 `.env`나 기존 환경 변수가 테스트에 영향을 주지 않게 하려면 환경을 임시 딕셔너리로 교체하고 `env_file=None`으로 dotenv 로딩을 끕니다. 테스트가 끝나면 원래 환경은 자동 복원됩니다.

```python
with patch.dict(os.environ, {
    'APP_API_KEY': 'test-secret',
    'APP_DATABASE__PORT': '6543',
}, clear=True):
    settings = load_settings(env_file=None)
    assert settings.database.port == 6543
    assert settings.api_key.get_secret_value() == 'test-secret'
    print(settings.database.port, settings.api_key)

with patch.dict(os.environ, {}, clear=True):
    try:
        load_settings(env_file=None)
    except RuntimeError as exc:
        print(exc)
```

```text
6543 **********
invalid settings: api_key
```

`get_secret_value()`는 실제 외부 클라이언트를 만들 때처럼 값이 꼭 필요한 좁은 위치에서만 호출합니다. 테스트는 성공 변환과 필수 설정 누락을 함께 확인해 실행 환경에 따른 우연한 성공을 막습니다.
