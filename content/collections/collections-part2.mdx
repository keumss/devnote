## deque로 FIFO 큐와 LIFO 스택 구현하기

`deque`는 양끝에서 추가하고 제거하는 연산이 대략 O(1)인 양방향 큐입니다. FIFO 큐는 오른쪽에 `append()`하고 왼쪽에서 `popleft()`하며, LIFO 스택은 같은 쪽의 `append()`와 `pop()`을 사용합니다. 빈 `deque`에서 `pop()`이나 `popleft()`를 호출하면 `IndexError`가 발생합니다.

```python
from collections import deque

jobs = deque(['compile', 'test'])
jobs.append('deploy')

while jobs:
    print(f'실행: {jobs.popleft()}')
```

```text
실행: compile
실행: test
실행: deploy
```

`list.pop(0)`은 나머지 원소를 이동하므로 O(n)이지만 `deque.popleft()`는 큐의 앞쪽 제거에 적합합니다. 반대로 중간 인덱스 접근은 `deque`에서 O(n)까지 느려질 수 있으므로 무작위 접근이 핵심이면 `list`를 사용합니다.

## maxlen과 rotate로 최근 기록과 순환 처리 만들기

`maxlen`이 있는 `deque`가 가득 찬 뒤 한쪽에 원소를 추가하면 반대쪽의 오래된 원소가 자동으로 버려집니다. 최근 N개 기록, 슬라이딩 윈도, 실행 이력처럼 보관량에 상한이 분명할 때 유용합니다. `rotate(n)`은 양수면 오른쪽, 음수면 왼쪽으로 원소를 회전합니다.

```python
from collections import deque

recent_latencies = deque(maxlen=3)
for latency in [120, 80, 150, 90]:
    recent_latencies.append(latency)

print(list(recent_latencies))

workers = deque(['api-1', 'api-2', 'api-3'])
for _ in range(4):
    worker = workers[0]
    print(worker)
    workers.rotate(-1)
```

```text
[80, 150, 90]
api-1
api-2
api-3
api-1
```

`extendleft(iterable)`은 각 원소를 차례로 왼쪽에 넣기 때문에 입력 순서가 뒤집힙니다. 또한 꽉 찬 제한 큐에서 `insert()`로 길이를 늘리려 하면 자동 폐기 대신 `IndexError`가 발생하므로 추가 방식의 차이를 구분합니다.

## ChainMap으로 설정 우선순위 표현하기

`ChainMap`은 여러 매핑을 복사해 합치지 않고 하나의 조회 뷰로 연결합니다. 조회는 앞쪽 매핑부터 키를 찾고, 원본 매핑이 바뀌면 결과에도 즉시 반영됩니다. 명령줄 옵션, 환경 설정, 기본값처럼 우선순위가 있는 값에 적합합니다.

```python
from collections import ChainMap

defaults = {'host': 'localhost', 'port': 8000, 'debug': False}
environment = {'port': 9000}
command_line = {'debug': True}

config = ChainMap(command_line, environment, defaults)

print(config['host'])
print(config['port'])
print(config['debug'])
```

```text
localhost
9000
True
```

`dict(defaults, **environment, **command_line)` 같은 병합은 그 시점의 복사본을 만들지만 `ChainMap`은 연결된 원본을 참조합니다. 최종 스냅샷이나 직렬화 가능한 평평한 매핑이 필요할 때는 `dict(config)`로 명시적으로 변환합니다.

## ChainMap의 쓰기 범위와 중첩 컨텍스트 이해하기

`ChainMap`의 조회는 모든 매핑을 탐색하지만 대입·갱신·삭제는 첫 번째 매핑에만 적용됩니다. 부모 설정을 실수로 바꾸지 않고 임시 재정의를 만들려면 `new_child()`를 사용하고, 현재 계층을 제외한 조회에는 `parents`를 사용합니다.

```python
from collections import ChainMap

base = {'theme': 'light', 'language': 'ko'}
session = ChainMap({}, base)
request = session.new_child({'theme': 'dark'})

request['language'] = 'en'

print(request['theme'], request['language'])
print(request.maps[0])
print(base)
print(request.parents['theme'])
```

```text
dark en
{'theme': 'dark', 'language': 'en'}
{'theme': 'light', 'language': 'ko'}
light
```

부모에만 있는 키를 `del request[key]`로 지우려 하면 첫 매핑에 그 키가 없으므로 `KeyError`가 발생합니다. 깊은 매핑까지 수정해야 한다면 직접 대상 매핑을 선택하거나 쓰기 정책이 명확한 별도 클래스를 설계해야 합니다.
