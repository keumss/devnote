## new_child로 더 구체적인 설정 레이어 만들기

기존 설정을 바꾸지 않고 요청 하나에만 더 높은 우선순위의 값을 추가할 때 `new_child()`를 사용합니다. 새 `ChainMap`의 맨 앞에 새 매핑이 생기고, 기존 매핑들은 그 뒤에 그대로 연결됩니다.

```python
from collections import ChainMap

base = ChainMap({'format': 'json'}, {'page_size': 20, 'locale': 'ko'})
request = base.new_child({'page_size': 50})

print(request['page_size'])
print(request['format'])
print(base['page_size'])
```

```text
50
json
20
```

`request`는 `page_size`만 새 값으로 가리고 나머지는 `base`에서 찾습니다. 요청 처리가 끝난 뒤 `request`를 버리면 되므로, 앞 레이어를 직접 수정했다가 되돌릴 필요가 없습니다.

## new_child의 키워드 인자로 임시 값 추가하기

미리 딕셔너리를 만들 필요 없이 몇 개의 임시 옵션만 추가할 때는 `new_child()`에 키워드 인자를 전달합니다. Python 3.12에서는 키워드 인자가 새 첫 번째 매핑에 들어갑니다.

```python
from collections import ChainMap

settings = ChainMap({'region': 'ap-northeast-2'})
preview = settings.new_child(dry_run=True, retries=0)

print(preview['dry_run'])
print(preview['region'])
print('dry_run' in settings)
```

```text
True
ap-northeast-2
False
```

원본 `settings`에는 `dry_run`이 추가되지 않습니다. 실제 요청에만 필요한 플래그와 미리 보기용 옵션은 이처럼 자식 레이어에 넣어 범위를 분명히 합니다.

## parents로 한 단계 위 설정 보기

현재 레이어의 오버라이드를 무시하고 부모 설정을 확인해야 할 때 `parents`를 사용합니다. 이 프로퍼티는 첫 번째 매핑을 제외한 나머지 매핑으로 만든 새 `ChainMap`을 반환합니다.

```python
from collections import ChainMap

root = ChainMap({'mode': 'release'})
request = root.new_child({'mode': 'debug', 'trace': True})

print(request['mode'])
print(request.parents['mode'])
print(dict(request.parents))
```

```text
debug
release
{'mode': 'release'}
```

`parents`는 현재 요청의 값을 빼고 상위 컨텍스트를 읽는 용도입니다. 특정 키를 강제로 기본값으로 되돌리려면 그 키가 든 첫 매핑을 수정하는 편이 더 직접적입니다.

## maps[0]으로 현재 레이어만 저장하기

현재 실행에서 추가한 값만 별도로 기록해야 할 때 `maps[0]`을 읽습니다. `dict(settings)`는 모든 레이어를 합친 결과이고, `maps[0]`은 직접 쓴 오버라이드만 담습니다.

```python
from collections import ChainMap

runtime = {}
defaults = {'retries': 2, 'timeout': 30}
settings = ChainMap(runtime, defaults)

settings['retries'] = 4
settings['request_id'] = 'req-7'

print(settings.maps[0])
print(dict(settings))
```

```text
{'retries': 4, 'request_id': 'req-7'}
{'retries': 4, 'timeout': 30, 'request_id': 'req-7'}
```

감사 로그처럼 사용자가 지정한 항목만 남길 때는 첫 레이어를 사용합니다. 외부 API에 최종 설정을 보낼 때는 다음 토픽처럼 일반 딕셔너리 스냅샷을 만듭니다.

## ChainMap 뷰와 합친 딕셔너리 구분하기

나중에 원본 변경도 반영해야 하는지, 그 시점의 값만 보관해야 하는지에 따라 `ChainMap`과 딕셔너리 병합을 선택합니다. `ChainMap`은 뷰이고, `{**defaults, **overrides}`는 생성 시점의 복사본입니다.

```python
from collections import ChainMap

defaults = {'retries': 2, 'timeout': 30}
overrides = {'timeout': 10}

view = ChainMap(overrides, defaults)
snapshot = {**defaults, **overrides}

defaults['retries'] = 4

print(view['retries'])
print(snapshot['retries'])
```

```text
4
2
```

한 번 정한 배포 요청처럼 값이 바뀌면 안 되는 결과에는 스냅샷이 알맞습니다. 현재 환경 설정을 계속 읽어야 하는 처리에는 복사 비용 없이 연결되는 `ChainMap`이 더 자연스럽습니다.

## dict로 외부 경계에 스냅샷 전달하기

JSON 직렬화·템플릿 엔진·서드 파티 함수처럼 일반 딕셔너리가 필요한 경계에서는 `dict()`로 펼칩니다. 만든 딕셔너리는 이후 `ChainMap`이나 원본 매핑이 바뀌어도 함께 바뀌지 않습니다.

```python
from collections import ChainMap

defaults = {'region': 'ap-northeast-2', 'retries': 2}
runtime = {'retries': 3}
settings = ChainMap(runtime, defaults)

payload = dict(settings)
payload['region'] = 'us-west-2'

print(settings['region'])
print(payload)
```

```text
ap-northeast-2
{'region': 'us-west-2', 'retries': 3}
```

`payload`를 수정해도 `settings`의 원본 레이어는 바뀌지 않습니다. 반대로 `ChainMap` 자체를 외부 함수에 넘기면 그 함수의 쓰기 동작이 첫 매핑에 영향을 줄 수 있으므로, 소유권 경계에서는 복사본을 전달합니다.

Python 3.12의 `new_child()`, `parents`, `maps`와 참조 기반 동작은 [collections 공식 문서](https://docs.python.org/3.12/library/collections.html#chainmap-objects)를 기준으로 확인합니다.
