## MappingProxyType으로 기본 레이어 보호하기

여러 요청이 함께 쓰는 기본 설정을 실수로 수정하지 못하게 할 때 `MappingProxyType`으로 읽기 전용 레이어를 만듭니다. `ChainMap`의 첫 매핑에는 쓰기 가능한 실행별 딕셔너리를 두므로, 오버라이드는 그대로 지원할 수 있습니다.

```python
from collections import ChainMap
from types import MappingProxyType

default_source = {'retries': 2, 'timeout': 30}
defaults = MappingProxyType(default_source)
runtime = {}
settings = ChainMap(runtime, defaults)

settings['retries'] = 5
default_source['timeout'] = 10

print(settings['retries'])
print(settings['timeout'])
print(default_source)
```

```text
5
10
{'retries': 2, 'timeout': 10}
```

프록시를 통해 `defaults['timeout'] = 10`처럼 쓰면 `TypeError`가 나지만, 원본 `default_source`를 소유한 초기화 코드는 변경할 수 있고 그 결과는 뷰에 반영됩니다. 전역 기본값의 소유자를 한 곳으로 제한할 때 유용한 패턴입니다.

## DeepChainMap으로 값이 있던 레이어 갱신하기

키가 처음 발견된 원본 레이어를 직접 수정해야 할 때는 `ChainMap`의 기본 쓰기 규칙 대신 전용 서브클래스를 사용합니다. 아래 `DeepChainMap`은 기존 키를 찾으면 그 매핑에서 갱신하고, 새 키만 첫 매핑에 추가합니다.

```python
from collections import ChainMap

class DeepChainMap(ChainMap):
    def __setitem__(self, key: str, value: object) -> None:
        for mapping in self.maps:
            if key in mapping:
                mapping[key] = value
                return
        self.maps[0][key] = value

    def __delitem__(self, key: str) -> None:
        for mapping in self.maps:
            if key in mapping:
                del mapping[key]
                return
        raise KeyError(key)

runtime = {}
project = {'region': 'ap-northeast-2'}
defaults = {'retries': 2}
settings = DeepChainMap(runtime, project, defaults)

settings['region'] = 'us-west-2'
settings['retries'] = 4
settings['dry_run'] = True

print(settings.maps)
```

```text
[{'dry_run': True}, {'region': 'us-west-2'}, {'retries': 4}]
```

이 방식은 설정 편집기처럼 각 레이어의 소유자가 분명할 때만 사용합니다. 요청별 값이 아래의 공용 기본값을 바꾸면 안 되는 일반적인 경우에는 기본 `ChainMap`의 첫 레이어 쓰기가 더 안전합니다.

## |와 |=로 첫 번째 레이어 조합하기

기존 설정은 보존한 채 파생 설정을 만들 때 `|`를, 현재 실행 레이어를 한 번에 갱신할 때 `|=`를 사용합니다. `|`의 결과는 첫 번째 매핑을 복사해 갱신한 새 `ChainMap`이고, `|=`는 현재 객체의 첫 매핑을 갱신합니다.

```python
from collections import ChainMap

defaults = {'retries': 2}
runtime = {'retries': 1}
settings = ChainMap(runtime, defaults)

preview = settings | {'dry_run': True}
settings |= {'timeout': 10}

print(settings.maps[0])
print(preview.maps[0])
```

```text
{'retries': 1, 'timeout': 10}
{'retries': 1, 'dry_run': True}
```

`preview`의 첫 레이어는 `runtime`의 사본이므로 `dry_run`은 원래 `settings`에 나타나지 않습니다. 다만 두 체인은 낮은 우선순위 매핑을 공유하므로, 완전히 독립적인 결과가 필요하면 `dict(settings)`로 평탄화한 뒤 새 딕셔너리를 사용합니다.

Python 3.12의 `ChainMap` 사용자 정의 갱신 조리법과 `|`·`|=` 지원은 [collections 공식 문서](https://docs.python.org/3.12/library/collections.html#chainmap-objects)를 기준으로 확인합니다.
