## 입력 출처와 우선순위 계약 정하기

이번 예제는 프로젝트 설정과 명령행 옵션 위에 기본값을 적용하는 `deploy_options.py`를 만듭니다. 호출자가 넘긴 두 매핑은 읽기 전용 입력으로 취급하고, 맨 앞의 빈 딕셔너리만 실행 중에 바뀌는 값의 자리로 둡니다.

```python
# deploy_options.py
from collections import ChainMap
from collections.abc import Mapping

DEFAULTS = {
    'region': 'ap-northeast-2',
    'retries': 2,
    'dry_run': False,
}

def make_options(
    project_options: Mapping[str, object],
    command_line_options: Mapping[str, object],
) -> ChainMap[str, object]:
    runtime_options: dict[str, object] = {}
    return ChainMap(
        runtime_options,
        command_line_options,
        project_options,
        DEFAULTS,
    )
```

우선순위는 `runtime_options` → 명령행 → 프로젝트 → 기본값입니다. 빈 첫 레이어를 둔 덕분에 이후 `options['retries'] = 5` 같은 대입도 입력 딕셔너리를 바꾸지 않습니다.

## make_options로 최종 값 조회하기

여러 출처의 값이 실제로 어떤 우선순위로 선택되는지 확인할 때는 하나의 `ChainMap`에서 필요한 키를 읽습니다. 같은 키를 모두 합치는 별도 조건문이 필요하지 않습니다.

```python
project_options = {'region': 'us-west-2', 'retries': 3}
command_line_options = {'dry_run': True}

options = make_options(project_options, command_line_options)

print(options['region'])
print(options['retries'])
print(options['dry_run'])
```

```text
us-west-2
3
True
```

명령행에 `region`을 넣으면 프로젝트 설정보다 먼저 선택되고, 어느 쪽에도 없는 키만 `DEFAULTS`에서 가져옵니다. 입력 값의 유효성 검사는 `make_options()`를 부르기 전에 각 입력 경계에서 끝내는 편이 단순합니다.

## 최종 옵션으로 배포 명령 만들기

선택된 설정을 실제 작업에 전달할 때는 필요한 키를 읽어 명령을 만듭니다. 이 함수는 어느 레이어에서 값이 왔는지 알 필요 없이, 최종 값이라는 하나의 인터페이스만 사용합니다.

```python
def build_deploy_command(options: Mapping[str, object]) -> list[str]:
    command = [
        'deploy',
        '--region', str(options['region']),
        '--retries', str(options['retries']),
    ]

    if options['dry_run']:
        command.append('--dry-run')

    return command

print(' '.join(build_deploy_command(options)))
```

```text
deploy --region us-west-2 --retries 3 --dry-run
```

`Mapping` 타입으로 받으면 이 함수가 `ChainMap`뿐 아니라 일반 딕셔너리도 받을 수 있습니다. 반대로 이 단계에서 `options`에 새 값을 대입하면 첫 레이어를 바꾸므로, 읽기 전용 함수에서는 조회만 합니다.

## new_child로 미리 보기 옵션 격리하기

실제 배포 설정을 바꾸지 않고 강제로 미리 보기 명령을 만들 때 `new_child()`로 한 단계 더 구체적인 레이어를 추가합니다. 미리 보기 객체를 버리면 임시 `dry_run` 값도 함께 사라집니다.

```python
def make_preview(options: ChainMap[str, object]) -> ChainMap[str, object]:
    return options.new_child(dry_run=True)

release_options = make_options(project_options, {})
preview = make_preview(release_options)

print(' '.join(build_deploy_command(preview)))
print(' '.join(build_deploy_command(release_options)))
```

```text
deploy --region us-west-2 --retries 3 --dry-run
deploy --region us-west-2 --retries 3
```

`release_options`에는 `dry_run`을 지정한 명령행 값이 없으므로, `preview`에만 `--dry-run`이 붙습니다. 원래 객체는 기본값 `False`를 계속 사용합니다.

## dict로 전송용 옵션 스냅샷 만들기

배포 API에 보낼 본문처럼 실행 순간의 값이 고정돼야 할 때는 `dict()`로 평탄화합니다. `ChainMap`의 레이어 구조를 외부에 노출하지 않고, 전송 뒤의 설정 변경과 분리할 수 있습니다.

```python
import json

def make_payload(options: Mapping[str, object]) -> dict[str, object]:
    return dict(options)

payload = make_payload(options)
print(json.dumps(payload, sort_keys=True))
```

```text
{"dry_run": true, "region": "us-west-2", "retries": 3}
```

실제 API가 허용하는 필드가 더 적다면 `dict(options)` 전체를 보내지 말고 필요한 키만 새 딕셔너리에 담습니다. 특히 인증 토큰처럼 같은 설정에 섞인 민감한 값은 전송 본문과 로그에서 명시적으로 제외합니다.

## 우선순위와 입력 불변성을 테스트하기

배포 옵션의 핵심 계약은 높은 우선순위가 이기고 호출자 입력을 바꾸지 않는 것입니다. 작은 테스트에서 두 조건을 함께 확인하면, 새 레이어를 추가할 때도 동작을 안전하게 바꿀 수 있습니다.

```python
# test_deploy_options.py
from deploy_options import make_options

def test_command_line_options_override_project_without_mutation() -> None:
    project_options = {'region': 'ap-northeast-2', 'retries': 3}
    command_line_options = {'region': 'us-west-2'}

    options = make_options(project_options, command_line_options)
    options['retries'] = 5

    assert options['region'] == 'us-west-2'
    assert options['retries'] == 5
    assert project_options == {'region': 'ap-northeast-2', 'retries': 3}
    assert command_line_options == {'region': 'us-west-2'}
```

이 테스트의 `options['retries'] = 5`는 `runtime_options`에만 기록됩니다. 프로젝트 설정 자체를 편집하는 기능을 추가한다면, 기본 `ChainMap` 대입이 아니라 명시적인 저장 함수와 별도 테스트를 만듭니다.

Python 3.12의 `ChainMap` 우선순위, 첫 레이어 쓰기, `new_child()` 규칙은 [collections 공식 문서](https://docs.python.org/3.12/library/collections.html#chainmap-objects)를 기준으로 확인합니다.
