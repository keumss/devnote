## 이벤트 모델과 집계 요구사항 정의하기

실전 예제로 서비스 이벤트를 받아 종류별 개수를 세고, 사용자별 이벤트를 모으며, 최근 이벤트만 보관하는 집계기를 만듭니다. 입력 레코드는 필드가 고정된 `namedtuple`, 빈도는 `Counter`, 그룹은 `defaultdict`, 제한 이력은 `deque`가 맡습니다. 각 타입의 책임을 나누면 중첩된 일반 딕셔너리보다 의도가 분명해집니다.

```python
from collections import namedtuple

Event = namedtuple('Event', ['user_id', 'kind', 'duration_ms'], defaults=[0])

event = Event('u-1', 'login')
print(event)
print(event._asdict())
```

```text
Event(user_id='u-1', kind='login', duration_ms=0)
{'user_id': 'u-1', 'kind': 'login', 'duration_ms': 0}
```

여기서는 학습을 위해 `namedtuple`을 사용합니다. 실제 도메인에서 값 범위 검증이나 타입 검사, 스키마 진화가 중요하다면 `dataclass`나 별도 검증 모델이 더 적합할 수 있습니다.

## 여러 collections 타입으로 집계기 구현하기

집계기는 이벤트가 들어올 때 한 번의 `add()` 호출로 모든 상태를 갱신합니다. `deque(maxlen=recent_limit)`은 오래된 이벤트를 자동 제거하므로 최근 이력의 메모리 사용량이 입력 건수와 함께 무한히 늘지 않습니다.

```python
from collections import Counter, defaultdict, deque, namedtuple

Event = namedtuple('Event', ['user_id', 'kind', 'duration_ms'], defaults=[0])

class EventAggregator:
    def __init__(self, recent_limit: int = 100) -> None:
        if recent_limit <= 0:
            raise ValueError('recent_limit은 양수여야 합니다.')
        self.counts = Counter()
        self.by_user = defaultdict(list)
        self.recent = deque(maxlen=recent_limit)

    def add(self, event: Event) -> None:
        if not event.user_id or not event.kind:
            raise ValueError('user_id와 kind는 비어 있을 수 없습니다.')
        if event.duration_ms < 0:
            raise ValueError('duration_ms는 음수일 수 없습니다.')

        self.counts[event.kind] += 1
        self.by_user[event.user_id].append(event)
        self.recent.append(event)

    def summary(self) -> dict[str, object]:
        return {
            'total': self.counts.total(),
            'by_kind': dict(self.counts),
            'active_users': len(self.by_user),
            'recent': [event._asdict() for event in self.recent],
        }
```

검증을 상태 변경보다 먼저 수행해야 실패한 이벤트가 일부 컨테이너에만 남지 않습니다. 실제 서비스에서 여러 스레드나 비동기 작업이 동시에 같은 집계기를 수정한다면 잠금, 메시지 큐, 단일 소비자 같은 별도 동시성 설계가 필요합니다.

## ChainMap으로 실행 환경별 설정 적용하기

최근 이력 크기처럼 실행 환경에 따라 달라지는 값은 명령줄, 환경 변수, 기본값의 우선순위를 `ChainMap`으로 표현할 수 있습니다. 외부 입력은 모두 문자열일 수 있으므로 우선순위 조회와 타입 변환·검증을 분리합니다.

```python
from collections import ChainMap

def load_recent_limit(
    cli: dict[str, str],
    env: dict[str, str],
) -> int:
    config = ChainMap(cli, env, {'RECENT_LIMIT': '100'})
    raw_limit = config['RECENT_LIMIT']

    try:
        limit = int(raw_limit)
    except ValueError as error:
        raise ValueError('RECENT_LIMIT은 정수여야 합니다.') from error

    if limit <= 0:
        raise ValueError('RECENT_LIMIT은 양수여야 합니다.')
    return limit

print(load_recent_limit({}, {'RECENT_LIMIT': '50'}))
print(load_recent_limit({'RECENT_LIMIT': '10'}, {'RECENT_LIMIT': '50'}))
```

```text
50
10
```

명령줄에서 옵션이 지정되지 않았다는 뜻의 `None`은 앞쪽 매핑에서 제거한 뒤 연결해야 합니다. 그렇지 않으면 `None`이 환경 변수와 기본값을 가려 버립니다. 비밀 값은 요약이나 오류 메시지에 포함하지 않습니다.

## 정상 흐름과 경계 조건을 자동 테스트하기

테스트는 최빈값 하나만 확인하지 말고 자료구조별 계약을 함께 고정합니다. 최근 이력의 자동 제거, 사용자 그룹, 총합, 잘못된 입력이 상태를 변경하지 않는지 확인하면 구현을 바꾸더라도 외부 동작을 유지할 수 있습니다.

```python
import pytest

from aggregator import Event, EventAggregator

def test_aggregates_events_and_limits_recent_history() -> None:
    aggregator = EventAggregator(recent_limit=2)
    aggregator.add(Event('u-1', 'login', 20))
    aggregator.add(Event('u-2', 'login', 30))
    aggregator.add(Event('u-1', 'purchase', 50))

    assert aggregator.counts == {'login': 2, 'purchase': 1}
    assert [event.kind for event in aggregator.recent] == ['login', 'purchase']
    assert len(aggregator.by_user['u-1']) == 2
    assert aggregator.summary()['total'] == 3

def test_rejects_invalid_event_without_changing_state() -> None:
    aggregator = EventAggregator()

    with pytest.raises(ValueError, match='음수'):
        aggregator.add(Event('u-1', 'login', -1))

    assert aggregator.counts.total() == 0
    assert not aggregator.recent
```

`defaultdict` 테스트에서 존재하지 않는 키를 `aggregator.by_user['missing']`으로 조회하면 테스트 자체가 상태를 바꿉니다. 부재를 검사할 때는 `'missing' not in aggregator.by_user`를 사용합니다. 운영에서는 처리량, 최근 이력 폐기량, 잘못된 이벤트 수를 관찰하고, 사용자별 전체 이력이 계속 증가한다면 보관 기한이나 외부 저장소를 설계합니다.

## 직렬화 경계와 운영 문제를 점검하기

`Counter`와 `defaultdict`는 `dict()`로, `deque`는 `list()`로, `namedtuple` 인스턴스는 `_asdict()`로 바꾸면 JSON 같은 외부 형식에 전달하기 쉽습니다. 변환은 컨테이너의 특수 동작을 제거한 스냅샷이므로, 다시 읽을 때 필요한 타입과 기본 팩터리는 명시적으로 복원합니다.

```python
import json

aggregator = EventAggregator(recent_limit=2)
aggregator.add(Event('u-1', 'login', 25))

payload = json.dumps(
    aggregator.summary(),
    ensure_ascii=False,
    sort_keys=True,
)
print(payload)
```

```text
{"active_users": 1, "by_kind": {"login": 1}, "recent": [{"duration_ms": 25, "kind": "login", "user_id": "u-1"}], "total": 1}
```

프로세스 재시작 후에도 집계가 유지되어야 한다면 메모리 컨테이너만으로는 충분하지 않습니다. 저장 시점과 중복 처리 정책을 정하고 데이터베이스나 로그에 원본 이벤트를 남깁니다. `collections`는 프로세스 내부의 표현과 연산을 개선하지만 내구성, 분산 동기화, 접근 제어를 대신하지 않습니다.
