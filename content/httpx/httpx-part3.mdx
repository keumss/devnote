프로덕션 환경의 관측 가능성(Observability), 고급 네트워크 모킹(Mocking) 및 FastAPI 인메모리 연동 테스트 기법을 다룹니다.

## event_hooks로 요청·응답 로깅 및 메트릭 수집하기

모든 HTTP 요청 및 응답 시점에 자동으로 실행할 콜백 함수를 등록하여 공통 로깅이나 실행 시간 측정 메트릭을 수집합니다.

```python
import httpx


def log_request(request: httpx.Request):
    print(f'[REQ] {request.method} {request.url}')


def log_response(response: httpx.Response):
    print(f'[RES] {response.status_code} {response.url}')


client = httpx.Client(
    event_hooks={
        'request': [log_request],
        'response': [log_response],
    })

res = client.get('https://httpbin.org/get')
```

```text
[REQ] GET https://httpbin.org/get
[RES] 200 https://httpbin.org/get
```

`event_hooks`에 콜백 함수 목록을 지정하면 애플리케이션 전체 HTTP 통신을 중앙에서 일관되게 감시할 수 있습니다.

## httpx.AsyncHTTPTransport로 재시도 및 커스텀 트랜스포트 구성하기

일시적인 네트워크 지연이나 5xx 상태 코드 발생 시 HTTPX 트랜스포트 수준에서 자동 재시도(Retry) 개수를 설정합니다.

```python
import asyncio
import httpx

# retries 옵션 지정 (기본값 0)
transport = httpx.AsyncHTTPTransport(retries=3)

async def main():
    async with httpx.AsyncClient(transport=transport) as client:
        res = await client.get('https://httpbin.org/get')
        print(res.status_code)

asyncio.run(main())
```

```text
200
```

`AsyncHTTPTransport(retries=N)`을 전달하면 네트워크 연결 실패 시 지정한 횟수만큼 자동 재시도를 수행합니다.

## httpx.ASGITransport로 FastAPI 앱 직접 인메모리 테스트하기

실제 HTTP 서버(Uvicorn 등)를 띄우지 않고, FastAPI ASGI 애플리케이션 객체를 메모리 내에서 직접 HTTPX로 호출하여 빠르게 통합 테스트를 수행합니다.

```python
import asyncio
from fastapi import FastAPI
import httpx

app = FastAPI()


@app.get('/api/health')
def health_check():
    return {'status': 'ok'}


async def test_app():
    transport = httpx.ASGITransport(app=app)
    async with httpx.AsyncClient(transport=transport,
                                 base_url='http://test') as client:
        response = await client.get('/api/health')
        print(response.status_code, response.json())


asyncio.run(test_app())
```

```text
200 {'status': 'ok'}
```

`ASGITransport`는 네트워크 소켓 통신 과정 없이 Python 인메모리 인터페이스로 요청을 전달하므로 테스트 속도가 매우 빠릅니다.

## respx 라이브러리로 비동기 HTTPX 요청 Mocking 테스트하기

외부 API 서버(예: 결제 게이트웨이, 소셜 로그인)를 실제로 호출하지 않고, 테스트 코드에서 외부 응답을 의사(Mock)로 정의하여 격리 테스트를 수행합니다.

```bash
python -m pip install respx
```

```python
import asyncio
import httpx
import respx


@respx.mock
async def test_external_api():
    # 외부 URL 요청 가로채기 정의
    respx.get('https://api.external.com/user').mock(
        return_value=httpx.Response(200, json={'id': 42, 'name': 'Alice'}))

    async with httpx.AsyncClient() as client:
        res = await client.get('https://api.external.com/user')
        print(res.status_code, res.json())


asyncio.run(test_external_api())
```

```text
200 {'id': 42, 'name': 'Alice'}
```

`respx` 데코레이터나 컨텍스트 매니저를 통해 네트워크 요청을 차단하고 격리된 환경에서 단위 테스트와 통합 테스트를 실행할 수 있습니다.

## httpx.HTTP2 옵션으로 HTTP/2 프로토콜 활성화하기

단일 TCP 커넥션에서 다중 프레임 송수신(Multiplexing)을 지원하는 HTTP/2 프로토콜로 요청을 전송합니다.

```bash
python -m pip install 'httpx[http2]'
```

```python
import httpx

client = httpx.Client(http2=True)
response = client.get('https://httpbin.org/get')

print(response.http_version)
```

```text
HTTP/2
```

`http2=True` 옵션을 활성화하려면 `h2` 라이브러리가 설치되어 있어야 하며, `response.http_version` 속성으로 프로토콜 버전을 확인합니다.

## httpx.Proxy로 HTTP/HTTPS 프록시 연동하기

사내 프록시 서버나 외부 게이트웨이를 거쳐서 HTTP 요청을 보내야 할 때 `httpx.Proxy` 객체를 지정합니다.

```python
import httpx

proxy = httpx.Proxy(
    url='http://proxy.internal.com:8080',
    auth=('proxy_user', 'proxy_pass')
)

client = httpx.Client(proxy=proxy)
print(client.proxy.url)
```

```text
http://proxy.internal.com:8080
```

`proxy` 인수를 전달하면 모든 나가는 통신이 프록시 서버를 경유하며, URL별 다른 프록시를 적용하려면 딕셔너리 형태로 지정할 수도 있습니다.

## httpx.Cookies 및 follow_redirects로 세션 리다이렉트 추적하기

301, 302 리다이렉트를 자동으로 추적하고 요청 간 쿠키(Cookie) 상태를 유지하도록 세션을 구성합니다.

```python
import httpx

# follow_redirects=True로 자동 30x 이동 설정
with httpx.Client(follow_redirects=True) as client:
    response = client.get('https://httpbin.org/redirect/2')
    print(response.status_code, response.url.path)
    print(len(response.history))  # 지나온 리다이렉트 응답 목록
```

```text
200 /get
2
```

`follow_redirects=True`를 설정하면 30x 상태 응답을 만났을 때 최종 200 OK 경로까지 자동으로 추적하며, `response.history`로 이력을 조회할 수 있습니다.
