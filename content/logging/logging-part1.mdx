## logging으로 이벤트를 관찰하는 이유

`logging`은 프로그램이 실행되는 동안 일어난 이벤트를 중요도와 함께 남기는 Python 표준 라이브러리입니다. `print()`는 사용자가 읽을 일반 출력에 적합하지만, 장애 조사·상태 관찰·파일 보관처럼 실행 환경에 따라 출력 위치와 형식을 바꿔야 할 때는 로그가 적합합니다. 이 섹션의 모든 예제는 Python 3.12 표준 라이브러리를 기준으로 하며, 명령줄 프로그램부터 서버와 라이브러리까지 일관되게 로그를 설계하는 방법을 다룹니다.

## 로그 레벨을 사건의 심각도에 맞추기

레벨은 `DEBUG`, `INFO`, `WARNING`, `ERROR`, `CRITICAL` 순으로 심각도가 높아집니다. 개발 중 세부 진단에는 `DEBUG`, 정상 흐름의 확인에는 `INFO`, 계속 실행할 수 있는 예상 밖의 상황에는 `WARNING`을 사용합니다. 어떤 작업을 수행하지 못했지만 예외를 잡아 프로그램이 계속 실행한다면 `ERROR`를, 프로그램 자체가 계속 실행하기 어려우면 `CRITICAL`을 선택합니다.

```python
import logging

logging.basicConfig(
    level=logging.INFO,
    format='%(levelname)s %(name)s: %(message)s',
)

logging.debug('캐시 키를 계산했습니다.')  # INFO보다 낮아 출력되지 않습니다.
logging.info('서버를 시작했습니다.')
logging.warning('남은 디스크 공간이 10%입니다.')
logging.error('주문 저장에 실패했습니다.')
```

```text
INFO root: 서버를 시작했습니다.
WARNING root: 남은 디스크 공간이 10%입니다.
ERROR root: 주문 저장에 실패했습니다.
```

`DEBUG` 줄이 출력에 없는 것이 핵심입니다. 설정 레벨 `INFO`보다 낮은 이벤트는 핸들러까지 전달되지 않습니다. 로그는 기본적으로 표준 오류로 나가므로, 일반 출력과 분리해 수집할 수 있습니다.

기본 루트 로거의 레벨은 `WARNING`이므로, `INFO`와 `DEBUG`를 보고 싶다면 애플리케이션 시작 지점에서 레벨을 명시해야 합니다.

## 모듈별 로거와 계층 구조 만들기

애플리케이션 코드에서는 `logging.getLogger(__name__)`으로 모듈마다 로거를 하나 만듭니다. `__name__`은 패키지 이름과 같은 점(`.`) 구분 계층을 이루므로, `shop.api`의 이벤트는 상위 `shop`과 루트 로거의 핸들러로 전파될 수 있습니다. 같은 이름으로 다시 호출해도 같은 로거 객체를 얻습니다.

```python
# shop/api.py
import logging

logger = logging.getLogger(__name__)

def find_product(product_id: int) -> dict[str, int]:
    logger.info('상품을 조회합니다. product_id=%s', product_id)
    return {'id': product_id}
```

실행 진입점에서 `shop` 로거를 `INFO`로 설정하고 `find_product(42)`를 호출하면 로거 이름이 모듈 경로로 남습니다.

```text
INFO shop.api: 상품을 조회합니다. product_id=42
```

`logger.info('...%s', product_id)`처럼 값은 인자로 전달합니다. 로그가 비활성화된 경우 불필요한 문자열 조합을 피할 수 있고, 포맷과 값이 분리되어 로그 수집 도구에서도 다루기 쉽습니다.

모듈 내부에서 루트 로거에 직접 설정을 추가하지 마세요. 설정은 실행 파일의 시작 지점에 모으고, 각 모듈은 자기 이름의 로거로 이벤트만 기록해야 테스트와 재사용이 쉬워집니다.

## 가장 작은 실행 프로그램을 설정하기

`basicConfig()`는 작은 프로그램에서 루트 로거의 출력 대상, 레벨, 형식을 빠르게 설정합니다. 로그 호출보다 먼저 한 번만 실행하고, 파일에는 인코딩을 지정해 한글 등의 텍스트가 환경에 따라 깨지지 않게 합니다.

```python
# app.py
import logging

logging.basicConfig(
    filename='app.log',
    encoding='utf-8',
    level=logging.INFO,
    format='%(asctime)s %(levelname)s %(name)s: %(message)s',
)

logger = logging.getLogger(__name__)
logger.info('애플리케이션을 시작했습니다.')
```

`app.log`에는 다음과 같은 한 줄이 추가됩니다. 시각은 실행한 순간에 따라 달라집니다.

```text
2026-07-13 09:30:15,204 INFO __main__: 애플리케이션을 시작했습니다.
```

기본 파일 모드는 이어쓰기(`'a'`)입니다. 매 실행마다 파일을 새로 시작해야 하는 진단용 작업에만 `filemode='w'`를 사용합니다. 운영 로그를 무심코 덮어쓰지 않도록 목적을 분명히 합니다.
