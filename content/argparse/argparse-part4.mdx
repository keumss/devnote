이번에는 `convert.py` 하나로 CSV 파일을 JSON으로 변환하는 CLI 도구를 완성합니다. 목표는 입력 파일, 구분자, 최대 행 수, 출력 경로, 상세 출력 옵션을 `argparse`로 받아, 파일이 없거나 형식이 잘못된 경우에는 유용한 에러를 출력하고 정상 케이스에서는 검증 가능한 JSON 파일을 만드는 것입니다. 아래 코드 블록을 순서대로 같은 파일에 추가합니다.

## 임포트와 인수 파서 준비하기

`argparse`로 파서를 정의하고, 입력 파일 존재 여부를 `type=` 단계에서 검증하는 함수를 만듭니다. 이렇게 하면 변환 로직에 도달하기 전에 에러가 출력됩니다.

```python
# convert.py
from __future__ import annotations

import argparse
import csv
import json
import sys
from pathlib import Path


def existing_file(value: str) -> Path:
    path = Path(value)
    if not path.exists():
        raise argparse.ArgumentTypeError(f'파일을 찾을 수 없습니다: {value}')
    if not path.is_file():
        raise argparse.ArgumentTypeError(f'파일이 아닙니다: {value}')
    return path


def build_parser() -> argparse.ArgumentParser:
    parser = argparse.ArgumentParser(
        prog='convert',
        description='CSV 파일을 JSON으로 변환합니다.',
        formatter_class=argparse.ArgumentDefaultsHelpFormatter,
    )
    parser.add_argument('input', type=existing_file, help='변환할 CSV 파일')
    parser.add_argument('--output', '-o', default='-',
                        help='저장할 JSON 파일 경로. "-"이면 stdout')
    parser.add_argument('--delimiter', '-d', default=',',
                        help='CSV 구분자')
    parser.add_argument('--limit', type=int, default=0,
                        help='최대 행 수. 0이면 전체')
    parser.add_argument('--verbose', '-v', action='store_true',
                        help='처리 결과를 stderr에 출력')
    return parser
```

`existing_file`은 `argparse.ArgumentTypeError`를 발생시켜 argparse가 사용자 친화적인 에러 메시지를 출력하게 합니다. `build_parser()`를 분리하면 테스트에서 `sys.argv` 없이 `parser.parse_args([...])`를 호출할 수 있습니다.

## CSV 읽기와 행 수 제한 적용하기

파일을 열어 헤더를 포함한 딕셔너리 목록을 만들고, `--limit`이 0이 아니면 지정한 수만큼만 가져옵니다.

```python
def read_csv(path: Path, delimiter: str, limit: int) -> list[dict[str, str]]:
    with path.open(encoding='utf-8', newline='') as f:
        reader = csv.DictReader(f, delimiter=delimiter)
        if reader.fieldnames is None:
            raise ValueError('CSV 헤더가 없습니다.')
        rows = []
        for i, row in enumerate(reader):
            if limit and i >= limit:
                break
            rows.append(dict(row))
    return rows
```

`csv.DictReader`는 첫 행을 헤더로 읽어 각 행을 `{컬럼: 값}` 딕셔너리로 반환합니다. `limit=0`이면 조건이 거짓이 되어 전체 행을 읽습니다.

## JSON 직렬화와 출력 경로 제어하기

변환 결과를 파일 또는 stdout으로 씁니다. `--output -`이면 표준 출력으로, 파일 경로면 해당 경로에 저장합니다.

```python
def write_json(rows: list[dict[str, str]], output: str) -> None:
    content = json.dumps(rows, ensure_ascii=False, indent=2)
    if output == '-':
        print(content)
    else:
        Path(output).write_text(content, encoding='utf-8')
```

`ensure_ascii=False`로 한글·특수문자를 그대로 직렬화합니다. `indent=2`는 사람이 읽기 편한 들여쓰기를 적용합니다. stdout으로 출력할 때는 `print()`가 줄바꿈을 자동으로 추가합니다.

## `--verbose`로 처리 결과 보고하기

`--verbose`가 켜져 있을 때 읽은 행 수와 저장 위치를 stderr에 출력합니다. 정상 결과 stdout과 분리하면 파이프 연결이 쉬워집니다.

```python
def main() -> int:
    parser = build_parser()
    args = parser.parse_args()

    try:
        rows = read_csv(args.input, args.delimiter, args.limit)
    except (ValueError, UnicodeDecodeError) as e:
        print(f'convert: {e}', file=sys.stderr)
        return 1

    write_json(rows, args.output)

    if args.verbose:
        destination = args.output if args.output != '-' else 'stdout'
        print(f'변환 완료: {len(rows)}행 → {destination}', file=sys.stderr)

    return 0


if __name__ == '__main__':
    raise SystemExit(main())
```

`raise SystemExit(main())`는 반환 코드를 프로세스 종료 코드로 전달합니다. 변환 성공은 0, 실패는 1로 쉘 스크립트에서 확인할 수 있습니다.

## `parse_args([])`로 단위 테스트 작성하기

`parse_args()`에 리스트를 전달하면 `sys.argv` 없이 파싱합니다. 실제 파일 경로를 `tmp_path` fixture로 만들어 전체 변환 흐름을 테스트합니다.

```python
# test_convert.py
import json
from pathlib import Path

import pytest

from convert import build_parser, main, read_csv, write_json


def test_parser_defaults() -> None:
    parser = build_parser()
    # existing_file이 실제 파일을 요구하므로 파서 기본값만 확인합니다.
    args = parser.parse_args.__func__  # noqa: just confirm defaults exist
    assert parser.get_default('delimiter') == ','
    assert parser.get_default('limit') == 0
    assert parser.get_default('output') == '-'


def test_read_csv_with_limit(tmp_path: Path) -> None:
    csv_file = tmp_path / 'sample.csv'
    csv_file.write_text('name,score\nalice,90\nbob,80\ncarol,70\n', encoding='utf-8')

    rows = read_csv(csv_file, delimiter=',', limit=2)

    assert len(rows) == 2
    assert rows[0] == {'name': 'alice', 'score': '90'}


def test_write_json_to_file(tmp_path: Path) -> None:
    output = tmp_path / 'result.json'
    rows = [{'name': 'alice', 'score': '90'}]

    write_json(rows, str(output))

    data = json.loads(output.read_text(encoding='utf-8'))
    assert data == rows


def test_main_end_to_end(tmp_path: Path, monkeypatch: pytest.MonkeyPatch) -> None:
    csv_file = tmp_path / 'data.csv'
    csv_file.write_text('id,value\n1,사과\n2,바나나\n', encoding='utf-8')
    output_file = tmp_path / 'out.json'

    monkeypatch.setattr(
        'sys.argv',
        ['convert', str(csv_file), '--output', str(output_file), '--limit', '1'],
    )

    code = main()

    assert code == 0
    data = json.loads(output_file.read_text(encoding='utf-8'))
    assert data == [{'id': '1', 'value': '사과'}]
```

```bash
# 실행 예시 — 입력 파일 준비
echo "name,score\nalice,90\nbob,80" > students.csv
python convert.py students.csv --output students.json --verbose
```

```text
# stderr
변환 완료: 2행 → students.json
```

```text
# students.json
[
  {
    "name": "alice",
    "score": "90"
  },
  {
    "name": "bob",
    "score": "80"
  }
]
```

`monkeypatch.setattr('sys.argv', ...)`로 `main()`의 전체 실행 흐름을 테스트합니다. 실제 파일 생성·읽기·쓰기가 모두 포함되므로 단위 테스트와 통합 테스트의 중간 수준에서 변환 계약을 검증합니다.

공식 Python [argparse](https://docs.python.org/3.12/library/argparse.html), [csv.DictReader](https://docs.python.org/3.12/library/csv.html#csv.DictReader), pytest [tmp_path](https://docs.pytest.org/en/stable/how-to/tmp_path.html)를 기준으로 구성했습니다.
