## type으로 문자열을 의미 있는 값으로 바꾸기

명령줄에서 받은 값은 처음에는 문자열입니다. `type=int`처럼 변환 함수를 지정하면 파싱 단계에서 값이 바뀌고, 변환할 수 없는 입력은 일관된 오류로 처리됩니다. 포트, 재시도 횟수, 비율처럼 프로그램이 숫자로 계산할 값에는 반드시 타입을 지정합니다.

```python
import argparse

parser = argparse.ArgumentParser()
parser.add_argument('--port', type=int, default=8000, help='1~65535 범위의 포트')
args = parser.parse_args()

print(args.port + 1)
```

`type`에는 문자열 하나를 받아 값을 반환하는 호출 가능 객체도 넣을 수 있습니다. 그 함수에서 `argparse.ArgumentTypeError`, `TypeError`, 또는 `ValueError`를 발생시키면 사용자는 파서 형식의 오류 메시지를 받습니다.

## 사용자 정의 변환 함수로 범위를 검사하기

`type=int`은 정수 여부까지만 검사합니다. 범위나 도메인 규칙도 함께 확인해야 한다면 작은 변환 함수를 만듭니다. 오류 문구는 사용자가 고칠 행동을 알 수 있도록 값과 허용 조건을 설명합니다.

```python
import argparse

def port_number(value: str) -> int:
    port = int(value)
    if not 1 <= port <= 65535:
        raise argparse.ArgumentTypeError('포트는 1부터 65535 사이여야 합니다.')
    return port

parser = argparse.ArgumentParser()
parser.add_argument('--port', type=port_number, default=8000)
args = parser.parse_args()
```

파일 경로를 `type=pathlib.Path`로 바꾸는 것도 편리하지만, 파일 존재 여부나 쓰기 권한은 파싱 직후가 아니라 실제 사용할 때 확인하는 편이 경쟁 상태를 줄이고 오류 처리 위치를 분명하게 합니다.

## choices와 metavar로 유효한 입력을 드러내기

정해진 모드나 형식에는 직접 `if`문을 쓰기보다 `choices`를 사용합니다. 도움말과 오류 메시지에 허용값이 자동으로 나타나며, 잘못된 값이 프로그램의 나중 단계까지 흘러가지 않습니다. `metavar`는 도움말에 표시할 값 이름만 바꿉니다.

```python
import argparse

parser = argparse.ArgumentParser()
parser.add_argument(
    '--format',
    choices=('text', 'json'),
    default='text',
    metavar='FORMAT',
    help='출력 형식 (기본값: text)',
)
args = parser.parse_args()
```

선택지가 적고 안정적일 때 `choices`가 특히 적합합니다. 외부 시스템에서 계속 바뀌는 이름 목록을 시작 시점에 검증해야 한다면, 목록을 가져오는 비용과 네트워크 오류를 고려해 별도의 검증 단계로 분리합니다.

## nargs와 append로 여러 값을 표현하기

`nargs`는 한 인자가 소비할 값의 개수를 정합니다. `nargs='+'`는 하나 이상, `nargs='*'`는 0개 이상을 목록으로 받습니다. 같은 옵션을 여러 번 허용할 때는 `action='append'`를 사용하면 입력 순서대로 목록에 쌓입니다.

```python
import argparse

parser = argparse.ArgumentParser()
parser.add_argument('files', nargs='+', help='처리할 파일을 하나 이상 지정')
parser.add_argument('--exclude', action='append', default=[], metavar='PATTERN')
args = parser.parse_args()

print(args.files)
print(args.exclude)
```

```bash
python scan.py a.py b.py --exclude '*.test.py' --exclude 'build/*'
```

가변 길이 위치 인자는 뒤에 다른 위치 인자를 두면 해석이 어려워질 수 있습니다. 목록 입력은 마지막 위치 인자로 배치하고, 빈 목록을 허용해야 하는지와 옵션을 반복할지 한 번에 여러 값을 받을지를 도움말에 분명히 적습니다.
