## type과 choices로 변환과 허용값 검사하기

명령줄 토큰은 처음에는 문자열입니다. `type`은 문자열 하나를 필요한 타입으로 변환하고, `choices`는 변환된 값이 허용 집합에 속하는지 검사합니다. 변환이 먼저 일어나므로 `type=int`와 함께 쓸 `choices`도 정수여야 합니다.

```python
import argparse

parser = argparse.ArgumentParser(prog='serve')
parser.add_argument('--port', type=int, choices=range(8000, 8010), default=8000)
parser.add_argument('--mode', choices=('development', 'production'), default='development')

args = parser.parse_args(['--port', '8003', '--mode', 'production'])
print(args.port, type(args.port).__name__, args.mode)
```

```text
8003 int production
```

기본값과 `type`의 관계에는 주의해야 합니다. 문자열 기본값은 명령줄 값처럼 `type` 변환을 거치지만, 문자열이 아닌 기본값은 그대로 사용됩니다. `type=int, default=10.5`는 `10`이 아니라 `10.5`를 만듭니다. 타입이 일관되도록 변환이 끝난 값인 `default=8000`처럼 지정하는 편이 명확합니다.

`type=bool`은 피합니다. `bool('false')`도 비어 있지 않은 문자열이므로 `True`입니다. 불리언은 `store_true`, `store_false`, `BooleanOptionalAction` 같은 액션으로 표현합니다.

## 사용자 정의 type은 작고 순수하게 유지하기

내장 타입만으로 표현할 수 없는 범위나 형식을 검사하려면 문자열 하나를 받아 값을 반환하는 함수를 만듭니다. 사용자가 고칠 수 있는 오류에는 `ArgumentTypeError`를 발생시킵니다.

```python
import argparse

def positive_int(value: str) -> int:
    try:
        number = int(value)
    except ValueError as error:
        raise argparse.ArgumentTypeError('정수를 입력해야 합니다.') from error

    if number < 1:
        raise argparse.ArgumentTypeError('1 이상의 정수를 입력해야 합니다.')
    return number

parser = argparse.ArgumentParser(prog='retry')
parser.add_argument('--attempts', type=positive_int, default=3)
args = parser.parse_args(['--attempts', '5'])
print(args.attempts)
```

```text
5
```

`type` 함수에서는 문자열 변환과 값 자체의 제약만 검사합니다. 파일 존재 여부, 접근 권한, 네트워크 연결처럼 외부 상태에 의존하거나 자원 정리가 필요한 작업은 파싱 뒤에 수행합니다. 파싱 중 파일을 열면 뒤쪽 인자가 실패했을 때 이미 연 파일이나 생성한 파일을 정리하기 어려워집니다.

경로를 `type=Path`로 바꾸는 것은 외부 작업이 없는 단순 변환이라 적합합니다. 반면 `type=lambda value: Path(value).read_text()`처럼 읽기까지 수행하면 도움말과 테스트가 실행 로직에 결합됩니다.

## nargs로 소비할 토큰 수와 결과 모양 정하기

`nargs`는 한 인자가 몇 개의 토큰을 소비할지 정합니다. 결과의 타입과 빈 입력 허용 여부도 함께 달라집니다.

- 생략: 정확히 하나를 소비해 단일 값으로 저장합니다.
- `?`: 0개 또는 1개를 소비해 단일 값으로 저장합니다.
- `*`: 0개 이상을 소비해 목록으로 저장합니다.
- `+`: 1개 이상을 소비해 목록으로 저장합니다.
- 정수 `N`: 정확히 N개를 소비해 목록으로 저장합니다.

```python
import argparse

parser = argparse.ArgumentParser(prog='crop')
parser.add_argument('images', nargs='+')
parser.add_argument('--size', nargs=2, type=int, metavar=('WIDTH', 'HEIGHT'))

args = parser.parse_args(['a.png', 'b.png', '--size', '640', '480'])
print(args.images)
print(args.size)
```

```text
['a.png', 'b.png']
[640, 480]
```

가변 길이 위치 인자는 가능한 한 마지막에 둡니다. 여러 가변 길이 위치 인자가 이어지면 어느 토큰을 어느 인자가 가져갈지 사용자가 예측하기 어렵습니다. `nargs='?'`를 옵션에 쓰면 옵션 자체가 없는 경우의 `default`와 옵션은 있지만 값이 없는 경우의 `const`까지 생기므로, 세 상태가 실제로 필요할 때만 사용합니다.

## append와 extend로 반복 옵션 수집하기

같은 옵션을 여러 번 받을 때는 `action='append'`가 호출 한 번의 값을 목록 항목 하나로 추가합니다. 한 호출에서 여러 값을 받고 모든 호출의 값을 평평한 목록으로 합치려면 `action='extend'`와 `nargs`를 함께 사용합니다.

```python
import argparse

parser = argparse.ArgumentParser(prog='search')
parser.add_argument('--tag', action='append', default=[])
parser.add_argument('--include', action='extend', nargs='+', default=[])

args = parser.parse_args([
    '--tag', 'python',
    '--tag', 'cli',
    '--include', 'src', 'tests',
    '--include', 'scripts',
])
print(args.tag)
print(args.include)
```

```text
['python', 'cli']
['src', 'tests', 'scripts']
```

`default=[]`은 파서가 값을 추가할 목록의 초기값을 분명히 합니다. 문자열 하나를 받는 옵션에 무심코 `extend`를 쓰면 문자열의 각 문자가 추가될 수 있으므로, `extend`에는 보통 `nargs='+'`나 `nargs='*'`를 함께 둡니다.

쉼표로 연결한 하나의 문자열보다 옵션을 반복하는 형식이 셸 인용, 값 검증, 도움말에서 더 명확한 경우가 많습니다. 한 값 자체에 쉼표가 들어갈 수 있는지도 계약을 정할 때 고려합니다.

## 불리언과 횟수를 내장 action으로 표현하기

`action`은 옵션을 만났을 때 값을 저장하는 방식을 결정합니다. 기능을 켜는 플래그에는 `store_true`, 명시적인 켜기와 끄기를 모두 제공할 때는 `BooleanOptionalAction`, 반복 횟수에 의미가 있을 때는 `count`가 적합합니다.

```python
import argparse

parser = argparse.ArgumentParser(prog='build')
parser.add_argument(
    '--color',
    action=argparse.BooleanOptionalAction,
    default=True,
    help='색상 출력을 사용하거나 사용하지 않음',
)
parser.add_argument('-v', '--verbose', action='count', default=0)

args = parser.parse_args(['--no-color', '-vv'])
print(args.color, args.verbose)
```

```text
False 2
```

`BooleanOptionalAction`은 `--color`와 `--no-color`를 함께 만듭니다. Python 3.12에서는 이 액션에 `type`, `choices`, `metavar`를 전달하는 방식이 폐기 예정이므로 불리언 액션 자체에만 사용합니다. `count`는 기본값을 생략하면 옵션이 없을 때 `None`이므로 산술이나 레벨 선택에 쓸 때는 `default=0`을 명시합니다.

사용자가 값을 직접 선택해야 한다면 `--level debug`처럼 `choices`를 쓰고, 반복할수록 강도가 커지는 관례가 자연스러울 때만 `-vv`를 사용합니다.

## 도움말에 기본값, 단위, 예제를 드러내기

도움말도 CLI의 공개 인터페이스입니다. `description`에는 명령의 목적, 각 `help`에는 동작과 단위, `epilog`에는 짧은 예제를 둡니다. `ArgumentDefaultsHelpFormatter`는 기본값이 있는 인자의 설명에 기본값을 자동으로 덧붙입니다.

```python
import argparse

parser = argparse.ArgumentParser(
    prog='serve',
    description='정적 파일을 HTTP로 제공합니다.',
    epilog='예: serve public --port 8080',
    formatter_class=argparse.ArgumentDefaultsHelpFormatter,
)
parser.add_argument('directory', metavar='DIR', help='제공할 디렉터리')
network = parser.add_argument_group('네트워크')
network.add_argument('--host', default='127.0.0.1', help='바인딩할 주소')
network.add_argument('--port', type=int, default=8000, help='수신할 TCP 포트')
```

`metavar`는 도움말의 자리표시자만 바꾸며 `args.directory`라는 속성 이름은 유지합니다. 인자 그룹도 도움말의 시각적 분류만 바꾸고 파싱 규칙에는 영향을 주지 않습니다. `help` 문자열에는 `%(default)s`, `%(prog)s` 같은 서식을 쓸 수 있고, 리터럴 `%`는 `%%`로 적어야 합니다.

자동 생성되는 `usage`는 실제 인자 규칙과 함께 바뀌므로 특별한 이유가 없다면 직접 작성하지 않습니다. 도움말에서만 감춘 옵션도 계속 사용할 수 있으므로, 제거된 기능처럼 오해하게 만드는 `help=argparse.SUPPRESS`는 호환성 전환 기간에 제한적으로 사용합니다.
