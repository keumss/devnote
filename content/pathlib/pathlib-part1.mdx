## pathlib가 문자열 경로보다 나은 이유

`pathlib`는 파일 시스템 경로를 객체로 표현하는 Python 3.12 표준 라이브러리입니다. 별도 설치 없이 `Path`를 가져와 사용합니다. 경로 조합, 이름 분석, 파일 읽기 같은 연관 동작을 한 객체에서 이어 쓸 수 있어 문자열 연결보다 의도가 분명하고 운영체제별 구분자도 직접 다룰 필요가 줄어듭니다.

```python
from pathlib import Path

report = Path('reports') / '2026' / 'summary.txt'
print(report)
print(report.parent)
print(report.name)
```

POSIX 환경의 예상 결과는 다음과 같습니다. Windows에서는 출력 구분자가 `\\`이지만 같은 코드가 올바른 경로를 만듭니다.

```text
reports/2026/summary.txt
reports/2026
summary.txt
```

`Path` 객체는 경로를 표현할 뿐 생성만으로 파일이나 디렉터리를 만들지 않습니다. 실제 파일 시스템에 접근하는 `exists()`, `read_text()`, `mkdir()` 같은 메서드를 호출할 때 입출력이 일어납니다. 이 구분을 알면 경로 계산과 파일 작업을 분리해 테스트하기 쉽습니다.

문자열을 `f'{directory}/{name}'`처럼 합치면 Windows 구분자, 중복 구분자, 절대 경로 입력의 의미를 직접 책임져야 합니다. 경로 조합에는 `/` 연산자나 `joinpath()`를 사용하고, 사용자에게 보여 주거나 문자열만 받는 오래된 API를 호출할 때만 `str(path)`로 변환합니다.

## Path와 슬래시 연산자로 경로 조합하기

`Path()`는 현재 운영체제의 경로 규칙을 따르는 구체 경로 객체를 만듭니다. 여러 조각을 생성자에 전달하거나 `/` 연산자로 자식 경로를 추가할 수 있습니다. `Path.cwd()`는 현재 작업 디렉터리, `Path.home()`은 현재 사용자의 홈 디렉터리를 반환합니다.

```python
from pathlib import Path

workspace = Path('workspace')
source = workspace / 'src' / 'main.py'
same_source = workspace.joinpath('src', 'main.py')

print(source == same_source)
print(Path.cwd().is_absolute())
print(Path.home().is_absolute())
```

```text
True
True
True
```

경로 객체는 불변입니다. `/`나 `with_name()`을 호출하면 원본을 바꾸지 않고 새 객체를 반환하므로 설정값이나 딕셔너리 키로 안전하게 공유할 수 있습니다.

조합할 오른쪽 값이 절대 경로라면 앞부분은 버려집니다. 따라서 외부 입력을 기준 디렉터리 아래에 붙일 때는 절대 경로인지 검사하지 않고 단순히 `/`로 결합해서는 안 됩니다.

```python
from pathlib import PurePosixPath

base = PurePosixPath('/srv/app')
print(base / 'data/users.json')
print(base / '/etc/passwd')
```

```text
/srv/app/data/users.json
/etc/passwd
```

두 번째 결과처럼 기준 경로가 사라지는 동작은 정상적인 경로 의미입니다. 신뢰할 수 없는 입력을 안전한 루트 아래로 제한하는 방법은 Part 3에서 다룹니다.

## name, stem, suffix로 파일 이름 분석하기

경로의 마지막 조각은 `name`, 마지막 확장자를 제외한 이름은 `stem`, 마지막 확장자는 `suffix`로 읽습니다. 여러 확장자를 모두 보고 싶으면 `suffixes`를 사용합니다. `parent`와 `parents`는 논리적인 상위 경로를 제공합니다.

```python
from pathlib import Path

archive = Path('backups/notes.tar.gz')

print(archive.name)
print(archive.stem)
print(archive.suffix)
print(archive.suffixes)
print(archive.parent)
print(list(archive.parents))
```

```text
notes.tar.gz
notes.tar
.gz
['.tar', '.gz']
backups
[PosixPath('backups'), PosixPath('.')]
```

마지막 줄의 구체 클래스 이름과 구분자는 운영체제에 따라 달라집니다. 파일 형식을 확인할 때 `.tar.gz` 전체가 필요하다면 `''.join(path.suffixes)`를 사용하고, 대소문자를 구분하지 않는 정책이라면 `path.suffix.lower()`처럼 정책을 코드에 명시합니다. 실제 파일 시스템의 대소문자 규칙은 운영체제와 볼륨 설정에 따라 다를 수 있습니다.

이름을 바꾼 새 경로는 `with_name()`, `with_stem()`, `with_suffix()`로 계산합니다. 이 메서드들도 파일 이름을 실제로 바꾸지는 않습니다.

```python
from pathlib import Path

draft = Path('notes/draft.md')
print(draft.with_stem('published'))
print(draft.with_suffix('.txt'))
```

```text
notes/published.md
notes/draft.txt
```

실제 이동이나 이름 변경에는 Part 2의 `rename()` 또는 `replace()`가 필요합니다.

## PurePath로 운영체제별 경로를 입출력 없이 다루기

대부분의 애플리케이션은 실행 중인 운영체제에서 입출력할 수 있는 `Path`를 사용합니다. 반면 `PurePath` 계열은 파일 시스템에 접근하는 메서드가 없고 문자열 수준의 경로 계산만 수행합니다. Unix에서 Windows 경로를 분석하거나 그 반대 상황을 테스트할 때 `PureWindowsPath`, `PurePosixPath`가 유용합니다.

```python
from pathlib import PurePosixPath, PureWindowsPath

windows = PureWindowsPath(r'C:\Users\mina\notes.txt')
posix = PurePosixPath('/home/mina/notes.txt')

print(windows.drive, windows.root, windows.name)
print(windows.as_posix())
print(posix.anchor, posix.parts)
```

```text
C: \ notes.txt
C:/Users/mina/notes.txt
/ ('/', 'home', 'mina', 'notes.txt')
```

순수 경로는 해당 파일이 존재하는지 확인하지 않습니다. `PureWindowsPath('NUL').is_reserved()`처럼 경로 문법을 검사하거나 다른 플랫폼용 설정을 생성할 수는 있지만 `read_text()`나 `exists()`는 제공하지 않습니다.

`Path('a/../b').parent` 같은 연산도 파일 시스템을 보지 않는 어휘적 계산입니다. `..` 앞의 `a`가 심볼릭 링크라면 단순 조각 계산과 실제 도착 위치가 다를 수 있습니다. 실제 위치가 중요할 때는 `resolve()`를 사용합니다.

## PathLike 인터페이스로 다른 API와 연결하기

`Path`와 순수 경로는 `os.PathLike` 인터페이스를 구현합니다. 표준 라이브러리와 많은 서드파티 API는 문자열뿐 아니라 경로 객체를 직접 받으므로, 필요 이상으로 `str()`로 바꾸지 않아도 됩니다.

```python
import json
import os
from pathlib import Path

config_path = Path('config.json')
raw_path = os.fspath(config_path)

print(raw_path)
print(type(raw_path).__name__)
print(json.dumps({'path': config_path.as_posix()}))
```

```text
config.json
str
{"path": "config.json"}
```

`open(config_path)`처럼 `PathLike`를 받는 함수에는 객체를 그대로 전달합니다. JSON 응답이나 환경 변수처럼 문자열만 저장하는 경계에서는 `str(path)` 또는 항상 `/`를 써야 하는 문서·URL용 표현이라면 `as_posix()`를 선택합니다. 파일 URL이 필요하면 절대 경로에서만 동작하는 `as_uri()`를 사용합니다.

경로 객체를 애플리케이션 내부의 공통 타입으로 유지하고 외부 경계에서만 변환하면 경로 조합과 이름 분석을 계속 `pathlib`로 처리할 수 있습니다.
