## pathlib가 문자열 경로보다 나은 이유

`pathlib`는 파일 시스템 경로를 객체로 표현하는 Python 3.12 표준 라이브러리입니다. 별도 설치 없이 대부분의 작업에 `Path`를 사용합니다. 경로 조합, 이름 분석, 파일 읽기 같은 동작을 한 객체에서 이어 쓸 수 있어 문자열 연결보다 의도가 분명하고 운영체제별 구분자도 직접 다룰 필요가 줄어듭니다.

```python
from pathlib import Path

report = Path('reports') / '2026' / 'summary.txt'
print(report)
print(report.parent)
print(report.name)
```

POSIX 환경의 예상 결과는 다음과 같습니다. Windows에서는 `\\`가 표시되지만 같은 코드가 올바른 경로를 만듭니다.

```text
reports/2026/summary.txt
reports/2026
summary.txt
```

`Path`를 만드는 것만으로 파일이나 디렉터리가 생기지는 않습니다. `/`, `parent`, `with_suffix()` 같은 연산은 경로만 계산하고, `read_text()`, `write_text()`, `mkdir()` 같은 메서드를 호출할 때 실제 입출력이 일어납니다. 이 구분을 알면 경로 계산과 실패할 수 있는 파일 작업을 분리하기 쉽습니다.

다른 운영체제의 경로 문법만 계산하거나 입출력을 금지하고 싶을 때는 `PurePosixPath`, `PureWindowsPath` 같은 순수 경로가 필요하지만, 일반 애플리케이션은 실행 환경에 맞는 `Path`에서 시작하면 됩니다.

## Path와 슬래시 연산자로 경로 조합하기

여러 경로 조각은 생성자나 `/` 연산자로 조합합니다. `Path.cwd()`는 현재 작업 디렉터리, `Path.home()`은 현재 사용자의 홈 디렉터리를 반환합니다.

```python
from pathlib import Path

workspace = Path('workspace')
source = workspace / 'src' / 'main.py'
same_source = workspace.joinpath('src', 'main.py')

print(source == same_source)
print(Path.cwd().is_absolute())
print(Path.home().is_absolute())
```

```text
True
True
True
```

경로 객체는 불변입니다. `/`나 `with_name()`은 원본을 바꾸지 않고 새 객체를 반환합니다. 문자열을 `f'{directory}/{name}'`처럼 합치면 Windows 구분자와 절대 경로의 의미를 직접 책임져야 하므로 경로 조합에는 `Path` 연산을 사용합니다.

오른쪽 값이 절대 경로라면 앞부분은 버려집니다.

```python
from pathlib import PurePosixPath

base = PurePosixPath('/srv/app')
print(base / 'data/users.json')
print(base / '/etc/passwd')
```

```text
/srv/app/data/users.json
/etc/passwd
```

두 번째 결과는 정상적인 경로 규칙입니다. 신뢰할 수 없는 입력을 기준 디렉터리 아래에 붙일 때는 단순 조합만으로 안전하다고 판단하지 않습니다. 실제 경로를 검증하는 방법은 Part 3에서 다룹니다.

## name, stem, suffix로 파일 이름 분석하기

마지막 경로 조각은 `name`, 마지막 확장자를 제외한 이름은 `stem`, 마지막 확장자는 `suffix`로 읽습니다. 여러 확장자는 `suffixes`, 논리적인 상위 경로는 `parent`와 `parents`로 확인합니다.

```python
from pathlib import Path

archive = Path('backups/notes.tar.gz')

print(archive.name)
print(archive.stem)
print(archive.suffix)
print(archive.suffixes)
print(archive.parent)
```

```text
notes.tar.gz
notes.tar
.gz
['.tar', '.gz']
backups
```

이름을 바꾼 새 경로는 `with_name()`, `with_stem()`, `with_suffix()`로 계산합니다. 이 메서드들은 실제 파일 이름을 바꾸지 않습니다.

```python
draft = Path('notes/draft.md')
print(draft.with_stem('published'))
print(draft.with_suffix('.txt'))
```

```text
notes/published.md
notes/draft.txt
```

파일 형식을 대소문자와 무관하게 비교하는 정책이라면 `path.suffix.lower()`처럼 정책을 코드에 명시합니다. 실제 파일 시스템의 대소문자 규칙은 운영체제와 볼륨 설정에 따라 다를 수 있습니다.

## read_text와 write_text로 작은 파일 처리하기

파일 전체를 한 번에 다룰 때는 `read_text()`와 `write_text()`가 간결합니다. 텍스트 해석이 실행 환경에 따라 달라지지 않도록 `encoding='utf-8'`을 명시합니다.

```python
from pathlib import Path
from tempfile import TemporaryDirectory

with TemporaryDirectory() as directory:
    note = Path(directory) / 'note.txt'
    written = note.write_text('첫 줄\n둘째 줄\n', encoding='utf-8')

    print(written)
    print(note.read_text(encoding='utf-8'), end='')
```

```text
9
첫 줄
둘째 줄
```

`write_text()`의 반환값은 쓴 문자 수이며 UTF-8 바이트 수가 아닙니다. 기본 동작은 기존 파일을 덮어씁니다. 이미지나 압축 데이터에는 `read_bytes()`와 `write_bytes()`를 사용하고, 큰 파일이나 이어쓰기는 Part 2의 `open()`으로 처리합니다.

`Path`는 `os.PathLike`를 구현하므로 `open(path)`처럼 경로 객체를 받는 API에는 그대로 전달합니다. JSON, 환경 변수처럼 문자열만 받는 경계에서만 `str(path)`로 변환합니다.

## mkdir와 상태 확인으로 기본 작업 흐름 완성하기

`mkdir()`는 기본적으로 부모가 없거나 대상이 이미 있으면 예외를 발생시킵니다. 여러 단계의 부모를 만들려면 `parents=True`, 이미 있는 디렉터리를 허용하려면 `exist_ok=True`를 지정합니다.

```python
from pathlib import Path
from tempfile import TemporaryDirectory

with TemporaryDirectory() as directory:
    root = Path(directory)
    notes = root / 'workspace' / 'notes'
    notes.mkdir(parents=True, exist_ok=True)

    guide = notes / 'guide.md'
    guide.write_text('# Path\n', encoding='utf-8')

    print(notes.is_dir())
    print(guide.is_file())
    print(guide.exists())
    print(guide.read_text(encoding='utf-8'), end='')
```

```text
True
True
True
# Path
```

`exist_ok=True`는 같은 경로에 디렉터리가 있을 때만 기존 상태를 허용합니다. 파일이 그 자리를 차지하면 여전히 `FileExistsError`가 발생합니다. `touch()`는 빈 파일을 만들거나 수정 시각을 갱신할 때 사용하지만, 내용을 쓸 목적이라면 `write_text()`나 `open()`이 더 분명합니다.

`exists()`는 화면에 상태를 표시하거나 선택 대상을 거르는 데 유용하지만 다음 작업의 성공을 보장하지 않습니다. 확인과 사용 사이에 파일이 사라지거나 권한이 바뀔 수 있으므로 실제 작업의 예외를 처리하는 방법은 Part 2에서 이어서 다룹니다.
