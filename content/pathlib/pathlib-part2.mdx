## open으로 큰 파일과 이어쓰기를 처리하기

`read_text()`는 파일 전체를 메모리에 올립니다. 큰 파일을 한 줄씩 처리하거나 이어서 쓸 때는 `Path.open()`과 컨텍스트 관리자를 사용합니다. 블록을 벗어나면 파일이 자동으로 닫힙니다.

```python
from pathlib import Path
from tempfile import TemporaryDirectory

with TemporaryDirectory() as directory:
    log_path = Path(directory) / 'app.log'

    with log_path.open('a', encoding='utf-8') as log_file:
        log_file.write('시작\n')
        log_file.write('완료\n')

    with log_path.open(encoding='utf-8') as log_file:
        messages = [line.rstrip() for line in log_file]

    print(messages)
```

```text
['시작', '완료']
```

텍스트 모드에는 인코딩을 명시하고, 이진 형식은 `open('rb')`나 `open('wb')`로 디코딩 없이 처리합니다. 파일을 열어 반환하는 함수보다는 필요한 데이터까지 읽고 닫거나, 호출자가 `with`로 수명을 관리할 수 있는 명확한 인터페이스를 선택합니다.

## iterdir와 glob으로 필요한 항목 찾기

직계 자식은 `iterdir()`, 패턴과 일치하는 항목은 `glob()`으로 찾습니다. 반환 순서는 파일 시스템에 따라 달라질 수 있으므로 출력이나 테스트처럼 순서가 계약이면 `sorted()`를 적용합니다.

```python
from pathlib import Path
from tempfile import TemporaryDirectory

with TemporaryDirectory() as directory:
    root = Path(directory)
    (root / 'guide.md').write_text('guide', encoding='utf-8')
    (root / 'README.md').write_text('readme', encoding='utf-8')
    (root / 'draft.txt').write_text('draft', encoding='utf-8')

    children = sorted(path.name for path in root.iterdir())
    markdown = sorted(path.name for path in root.glob('*.md'))
    print(children)
    print(markdown)
```

```text
['README.md', 'draft.txt', 'guide.md']
['README.md', 'guide.md']
```

`*`는 한 디렉터리 단계, `**`는 하위 트리까지 일치합니다. `rglob('*.py')`는 `glob('**/*.py')`와 같은 재귀 탐색 의도를 짧게 표현합니다. Python 3.12에서는 `glob()`과 `rglob()`의 `case_sensitive=True` 또는 `False`로 플랫폼 기본 대소문자 규칙을 덮어쓸 수 있습니다.

```python
root = Path('src')
for path in sorted(root.rglob('*.py', case_sensitive=False)):
    print(path.relative_to(root).as_posix())
```

출력은 실제 트리에 따라 달라집니다. 보고서나 테스트에서 `as_posix()`를 사용하면 운영체제와 관계없이 `/` 구분자로 표시할 수 있습니다.

## stat과 예외 처리로 파일 상태 다루기

`exists()`, `is_file()`, `is_dir()`, `is_symlink()`는 대상의 상태를 질문합니다. 크기와 수정 시각 같은 메타데이터는 `stat()` 결과에서 읽습니다.

```python
from pathlib import Path
from tempfile import TemporaryDirectory

with TemporaryDirectory() as directory:
    data = Path(directory) / 'data.bin'
    data.write_bytes(b'abcde')

    info = data.stat()
    print(data.is_file())
    print(info.st_size)
```

```text
True
5
```

`exists()`로 미리 확인했다고 다음 작업이 반드시 성공하는 것은 아닙니다. 실제 목표가 읽기라면 읽기를 바로 시도하고 처리 가능한 실패를 작업 경계에서 다룹니다.

```python
from pathlib import Path

def load_note(path: Path) -> str | None:
    try:
        return path.read_text(encoding='utf-8')
    except FileNotFoundError:
        return None
```

권한 오류까지 `None`으로 바꾸면 파일이 없는 경우와 접근할 수 없는 경우를 구분할 수 없습니다. 복구 정책이 같은 예외만 함께 처리하고, `OSError` 전체를 잡을 때는 어떤 실패까지 성공으로 인정하는지 먼저 정합니다.

## rename, replace, unlink로 항목 변경하기

`rename(target)`은 파일이나 디렉터리를 이동하고 새 대상의 `Path`를 반환합니다. 기존 대상이 있을 때의 동작은 Unix와 Windows가 다르므로, 기존 파일을 덮어쓸 의도라면 `replace(target)`를 사용합니다.

```python
from pathlib import Path
from tempfile import TemporaryDirectory

with TemporaryDirectory() as directory:
    root = Path(directory)
    draft = root / 'draft.txt'
    final = root / 'final.txt'

    draft.write_text('완료', encoding='utf-8')
    moved = draft.rename(final)

    print(moved.name)
    print(draft.exists())
    print(final.read_text(encoding='utf-8'))
```

```text
final.txt
False
완료
```

상대 `target`은 원본의 부모가 아니라 현재 작업 디렉터리를 기준으로 해석됩니다. 같은 디렉터리에서 이름만 바꾸려면 `path.with_name('new-name.txt')`로 대상을 계산합니다.

파일과 심볼릭 링크는 `unlink()`, 빈 디렉터리는 `rmdir()`로 삭제합니다. `unlink(missing_ok=True)`는 대상이 없을 때만 오류를 무시합니다. 내용이 있는 디렉터리 트리는 `shutil.rmtree()`의 역할이며, 삭제 전 허용된 루트와 대상을 검증해야 합니다.

## PurePath와 PathLike로 입출력 경계 정하기

대부분은 현재 운영체제에서 입출력할 수 있는 `Path`를 사용합니다. `PureWindowsPath`와 `PurePosixPath`는 파일 시스템을 읽지 않고 다른 운영체제의 경로 문법을 계산해야 할 때 유용합니다.

```python
import os
from pathlib import Path, PureWindowsPath

windows = PureWindowsPath(r'C:\Users\mina\notes.txt')
config = Path('config.json')

print(windows.drive, windows.name)
print(windows.as_posix())
print(os.fspath(config))
```

```text
C: notes.txt
C:/Users/mina/notes.txt
config.json
```

순수 경로에는 `read_text()`나 `exists()`가 없습니다. 또한 `Path`와 순수 경로는 `os.PathLike`를 구현하므로 표준 라이브러리와 많은 서드파티 API에 그대로 전달할 수 있습니다. `str(path)`는 환경 변수, JSON 같은 문자열 경계에서만 사용하고, URL이나 문서용 `/` 표현에는 `as_posix()`를 선택합니다.

경로 객체를 애플리케이션 내부 공통 타입으로 유지하면 조합과 이름 분석을 계속 `pathlib`로 처리할 수 있고, 문자열을 다시 경로로 해석하는 중복도 줄어듭니다.
