## read_text와 write_text로 작은 텍스트 파일 처리하기

파일 전체를 한 번에 다룰 때는 `read_text()`와 `write_text()`가 간결합니다. 두 메서드는 파일을 열고 작업한 뒤 자동으로 닫습니다. 텍스트의 해석이 실행 환경에 따라 달라지지 않도록 `encoding='utf-8'`을 명시합니다.

```python
from pathlib import Path
from tempfile import TemporaryDirectory

with TemporaryDirectory() as directory:
    note = Path(directory) / 'note.txt'
    written = note.write_text('첫 줄\n둘째 줄\n', encoding='utf-8')

    print(written)
    print(note.read_text(encoding='utf-8'), end='')
```

```text
9
첫 줄
둘째 줄
```

`write_text()`의 반환값은 쓴 문자 수이며 UTF-8 바이트 수가 아닙니다. 기본 모드는 기존 파일 내용을 덮어씁니다. 파일이 클거나 한 줄씩 처리해야 한다면 전체를 메모리에 올리는 `read_text()` 대신 `open()`과 컨텍스트 관리자를 사용합니다.

```python
from pathlib import Path

log_path = Path('app.log')

with log_path.open('a', encoding='utf-8') as log_file:
    log_file.write('작업 완료\n')
```

`read_bytes()`와 `write_bytes()`는 이미지, 압축 파일처럼 디코딩하지 않을 이진 데이터에 사용합니다. 텍스트를 바이트로 임의 변환하기보다 데이터 형식의 경계에서 인코딩을 명시합니다.

## mkdir와 touch로 디렉터리와 파일 만들기

`mkdir()`는 기본적으로 부모가 없거나 대상이 이미 있으면 예외를 발생시킵니다. 여러 단계의 부모를 함께 만들려면 `parents=True`, 디렉터리가 이미 있어도 성공하게 하려면 `exist_ok=True`를 지정합니다.

```python
from pathlib import Path
from tempfile import TemporaryDirectory

with TemporaryDirectory() as directory:
    output = Path(directory) / 'build' / 'reports'
    output.mkdir(parents=True, exist_ok=True)

    marker = output / '.ready'
    marker.touch(exist_ok=False)

    print(output.is_dir())
    print(marker.is_file())
```

```text
True
True
```

`exist_ok=True`는 같은 경로에 디렉터리가 있을 때만 기존 상태를 허용합니다. 파일이 그 자리를 차지하고 있다면 여전히 `FileExistsError`가 발생하므로 잘못된 구조를 조용히 넘기지 않습니다.

`touch()`는 빈 파일을 만들거나 기존 파일의 수정 시각을 갱신합니다. 내용을 써야 한다면 `write_text()`나 `open()`이 의도를 더 잘 드러냅니다. 동시 실행 중 “없을 때만 생성”해야 한다면 `touch(exist_ok=False)` 또는 `open('x')`가 `exists()`로 미리 확인한 뒤 생성하는 것보다 경쟁 조건을 줄입니다.

## iterdir와 glob으로 필요한 항목 찾기

한 디렉터리의 직계 자식을 순회할 때는 `iterdir()`, 패턴에 맞는 항목을 찾을 때는 `glob()`을 사용합니다. 반환 순서는 파일 시스템에 따라 달라질 수 있으므로 사용자 출력이나 테스트처럼 순서가 계약이면 `sorted()`를 적용합니다.

```python
from pathlib import Path
from tempfile import TemporaryDirectory

with TemporaryDirectory() as directory:
    root = Path(directory)
    (root / 'guide.md').write_text('guide', encoding='utf-8')
    (root / 'README.md').write_text('readme', encoding='utf-8')
    (root / 'draft.txt').write_text('draft', encoding='utf-8')

    names = sorted(path.name for path in root.glob('*.md'))
    print(names)
```

```text
['README.md', 'guide.md']
```

`glob()`의 패턴은 기준 `Path`에 상대적입니다. `*`는 한 디렉터리 단계에서 일치하고, `**`는 하위 트리를 재귀적으로 탐색합니다. `rglob('*.py')`는 `glob('**/*.py')`와 같은 의도를 더 짧게 표현합니다.

```python
from pathlib import Path

root = Path('src')
python_files = sorted(root.rglob('*.py'))

for path in python_files:
    print(path.relative_to(root).as_posix())
```

출력 항목은 실제 `src` 트리에 따라 달라지며, 각 줄은 `src`를 제외한 상대 경로입니다. `as_posix()`를 사용해 보고서나 테스트에서는 운영체제와 관계없이 `/` 구분자로 표시합니다.

Python 3.12의 `glob()`과 `rglob()`은 `case_sensitive=True` 또는 `False`로 플랫폼 기본 대소문자 규칙을 덮어쓸 수 있습니다. 이 옵션을 생략하면 일반적으로 POSIX에서는 대소문자를 구분하고 Windows에서는 구분하지 않습니다. 여러 플랫폼에서 같은 결과가 필요하다면 정책을 명시합니다.

## exists와 stat으로 종류와 메타데이터 확인하기

`exists()`, `is_file()`, `is_dir()`, `is_symlink()`는 경로가 가리키는 대상의 상태를 질문합니다. 크기, 수정 시각, 권한 비트 같은 자세한 정보는 `stat()` 결과에서 읽습니다.

```python
from pathlib import Path
from tempfile import TemporaryDirectory

with TemporaryDirectory() as directory:
    data = Path(directory) / 'data.bin'
    data.write_bytes(b'abcde')

    info = data.stat()
    print(data.exists())
    print(data.is_file())
    print(info.st_size)
```

```text
True
True
5
```

`exists()`로 확인했다고 다음 작업이 반드시 성공하는 것은 아닙니다. 확인과 사용 사이에 다른 프로세스가 파일을 삭제하거나 권한을 바꿀 수 있기 때문입니다. 실제 목표가 읽기라면 `read_text()`를 바로 시도하고 `FileNotFoundError`, `PermissionError`, `UnicodeError`처럼 처리 가능한 실패를 작업 경계에서 다룹니다.

```python
from pathlib import Path

def load_note(path: Path) -> str | None:
    try:
        return path.read_text(encoding='utf-8')
    except FileNotFoundError:
        return None
```

`exists()`는 UI에 상태를 표시하거나 선택적으로 처리할 대상을 거르는 데는 유용하지만, 권한이나 성공 가능성을 보증하는 검사로 사용하지 않습니다. 반복 탐색에서 같은 파일에 `is_file()`과 `stat()`을 차례로 호출하면 시스템 호출도 반복될 수 있으므로 필요한 정보만 요청합니다.

## rename, replace, unlink로 항목 변경하기

`rename(target)`은 파일이나 디렉터리를 이동하고 새 대상의 `Path`를 반환합니다. 기존 대상이 있을 때의 동작은 Unix와 Windows가 다릅니다. 기존 파일을 덮어쓰는 의도를 운영체제 간에 분명히 하려면 `replace(target)`를 사용합니다.

```python
from pathlib import Path
from tempfile import TemporaryDirectory

with TemporaryDirectory() as directory:
    root = Path(directory)
    draft = root / 'draft.txt'
    final = root / 'final.txt'

    draft.write_text('완료', encoding='utf-8')
    moved = draft.rename(final)

    print(moved.name)
    print(draft.exists())
    print(final.read_text(encoding='utf-8'))
```

```text
final.txt
False
완료
```

상대적인 `target`은 원본 파일의 부모가 아니라 현재 작업 디렉터리를 기준으로 해석됩니다. 같은 디렉터리 안에서 이름만 바꾸려면 `path.with_name('new-name.txt')`로 대상을 계산해 전달하면 실수를 줄일 수 있습니다.

파일이나 심볼릭 링크는 `unlink()`, 빈 디렉터리는 `rmdir()`로 삭제합니다. `unlink(missing_ok=True)`는 대상이 없을 때만 오류를 무시하며 권한 오류나 디렉터리를 파일처럼 지우려는 오류는 그대로 알립니다. 내용이 있는 디렉터리 트리 삭제는 `pathlib`의 단일 메서드가 아니라 `shutil.rmtree()`의 역할입니다.

삭제와 덮어쓰기는 복구하기 어려우므로 대상 경로를 로그로 남기고, 허용된 루트 안인지 검증하고, 테스트에서는 `TemporaryDirectory`나 pytest의 `tmp_path`로 범위를 격리합니다.
