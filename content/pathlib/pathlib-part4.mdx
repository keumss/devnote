이번 실전 예제에서는 `backup_tool.py` 파일 하나를 점진적으로 채워 나가면서 디렉토리 내의 특정 확장자 파일들을 지정 디렉토리로 안전하게 복사하는 CLI 기능을 구현합니다. 백업 과정에서 타겟 경로에 동일 이름의 파일이 존재할 경우 버전을 나타내는 접미사(`_v1`, `_v2` 등)를 덧붙여 파일이 덮어씌워지는 사고를 방지합니다. 아래 코드 블록들을 순서대로 하나의 스크립트에 덧붙여 실행합니다.

## Path 객체로 원본 및 대상 폴더 매개변수 준비하기

가장 먼저 `pathlib.Path` 클래스를 임포트하고, 사용자로부터 원본 디렉토리와 백업 대상 폴더의 이름을 전달받아 경로 검증을 수행하는 기초 설정을 정의합니다.

```python
# backup_tool.py
from pathlib import Path


def prepare_paths(src_dir_str: str, backup_dir_str: str) -> tuple[Path, Path]:
    src_path = Path(src_dir_str)
    backup_path = Path(backup_dir_str)
    
    # 원본 디렉토리가 부재하면 처리 불가로 에러 처리
    if not src_path.exists():
        raise ValueError(f"원본 디렉토리가 존재하지 않습니다: {src_path}")
        
    return src_path, backup_path


# 테스트용 가상 디렉토리 준비
src = Path('source_logs')
src.mkdir(exist_ok=True)
(src / 'app.log').write_text('Log sample 1', encoding='utf-8')
(src / 'db.log').write_text('Log sample 2', encoding='utf-8')

s_path, b_path = prepare_paths('source_logs', 'backup_logs')
print(f"Source: {s_path.name}, Backup: {b_path.name}")
```

```text
Source: source_logs, Backup: backup_logs
```

입력 문자열들을 즉시 `Path` 인스턴스로 감싸고 `exists()` 검사를 수행하여 프로그램 오작동 가능성을 일차적으로 통제합니다.

## rglob로 백업 대상 로그 파일 수집하기

원본 디렉토리의 모든 깊이(하위 폴더 포함)를 뒤져 백업할 특정 확장자(`.log`)를 가진 모든 파일을 한 번에 스캔하여 목록으로 추출하는 함수를 추가합니다.

```python
# backup_tool.py
# [기존 prepare_paths 함수 및 테스트 디렉토리 생성 코드 아래에 추가]


def find_target_files(src_path: Path) -> list[Path]:
    # 모든 하위 경로를 돌며 *.log 매칭 파일을 리스트로 묶어 리턴
    return list(src_path.rglob('*.log'))


targets = find_target_files(s_path)
for t in sorted(targets):
    print(t.name)
```

```text
app.log
db.log
```

`rglob()` 제너레이터를 리스트로 확장하여 향후 수행할 백업 작업 리스트를 일목요연하게 유지 관리합니다.

## mkdir로 백업 폴더와 동일한 하위 디렉토리 구조 생성하기

원본 파일이 깊은 디렉토리 계층 안에 들어가 있었을 경우, 백업 대상 디렉토리 안에서도 동일한 깊이에 위치하도록 타겟 하위 폴더들을 먼저 안전하게 생성합니다.

```python
# backup_tool.py
# [기존 코드 아래에 추가]


def get_backup_dest(file_path: Path, src_path: Path, backup_path: Path) -> Path:
    # 원본 디렉토리 기준의 상대 경로를 계산
    relative_path = file_path.relative_to(src_path)
    # 백업 폴더 하위에 동일 상대 경로 설정
    dest_path = backup_path / relative_path
    
    # 대상 파일이 쓰여질 상위 디렉토리 생성 (이미 있어도 통과)
    dest_path.parent.mkdir(parents=True, exist_ok=True)
    return dest_path


dest_app = get_backup_dest(targets[0], s_path, b_path)
print(dest_app)
```

```text
backup_logs/app.log
```

`relative_to` 메서드는 공통 최상위 경로를 추출하여 타겟의 고유 분기 경로만 남겨주므로, 백업 시 원본 디렉토리 구조를 고스란히 복제하는 데 유용합니다.

## write_bytes와 이름 충돌 방지 로직으로 안전하게 파일 복사하기

백업 복사 시 만에 하나 파일 이름이 동일한 경우 기존 백업본을 덮어쓰지 않고 파일명에 `_v1`, `_v2` 등의 카운터를 붙여 저장 파일의 고유성을 확보한 뒤 바이너리 형태로 기록합니다.

```python
# backup_tool.py
# [기존 코드 아래에 추가]


def backup_file_safely(src_file: Path, dest_file: Path) -> Path:
    counter = 1
    base_dest = dest_file
    
    # 동일 경로명이 존재하는 동안 파일 이름에 접미사 추가
    while dest_file.exists():
        new_name = f"{base_dest.stem}_v{counter}{base_dest.suffix}"
        dest_file = base_dest.with_name(new_name)
        counter += 1
        
    # 데이터 유실 없는 바이트 복사
    data = src_file.read_bytes()
    dest_file.write_bytes(data)
    return dest_file


copied = backup_file_safely(targets[0], dest_app)
print(f"Copied to: {copied}")
```

```text
Copied to: backup_logs/app.log
```

`stem`(확장자를 뺀 이름)과 `suffix`(확장자) 속성을 이용해 접미사를 안전하게 끼워 넣을 수 있어 복잡한 문자열 파싱 없이 처리합니다.

## 전체 복사 실행 및 정리 작업하기

작성한 서브 함수들을 조립하여 특정 디렉토리 백업 파이프라인을 최종 완성하고, 테스트에 활용했던 가상 디렉토리 및 파일 자원을 원상태로 깔끔하게 청소합니다.

```python
# backup_tool.py
# [기존 코드 아래에 추가]


def run_backup(src_dir: str, backup_dir: str) -> None:
    src, backup = prepare_paths(src_dir, backup_dir)
    targets = find_target_files(src)
    
    print(f"총 {len(targets)}개 파일 백업을 시작합니다.")
    for file_path in targets:
        dest = get_backup_dest(file_path, src, backup)
        final_dest = backup_file_safely(file_path, dest)
        print(f"- 백업 완료: {file_path.name} -> {final_dest.name}")


run_backup('source_logs', 'backup_logs')

# -----------------
# 테스트 리소스 정리
# -----------------
for f in Path('source_logs').iterdir():
    f.unlink()
Path('source_logs').rmdir()

for f in Path('backup_logs').iterdir():
    f.unlink()
Path('backup_logs').rmdir()
```

```text
총 2개 파일 백업을 시작합니다.
- 백업 완료: app.log -> app.log
- 백업 완료: db.log -> db.log
```

`unlink`와 `rmdir`을 순차적으로 호출하여 생성되었던 자원들을 원래 상태로 말끔하게 회수하여 디스크 오염을 방지합니다.

공식 Python [pathlib 문서](https://docs.python.org/3.12/library/pathlib.html) 스펙을 바탕으로 작성되었습니다.
