## 매출 CSV와 화면 결과의 계약 정하기

이번에는 지역과 기간으로 매출 CSV를 필터링하고, 합계·주문 수·일별 추세를 보여 준 뒤 결과 CSV를 내려받는 대시보드를 만듭니다. 입력 파일은 `ordered_at`, `region`, `amount` 열을 반드시 가지며, 금액은 0 이상인 숫자여야 합니다.

먼저 빈 폴더에 아래 내용을 `sales.csv`로 저장하고 `app.py`를 만듭니다. `sales.py`는 다음 토픽에서 추가합니다.

```csv

2026-07-01,서울,120000
2026-07-01,부산,80000
2026-07-02,서울,95000
2026-07-03,제주,60000
2026-07-03,부산,110000
```

## @st.cache_data와 검증으로 CSV 경계 만들기

파일을 읽는 코드는 화면 코드와 분리하고, 필수 열과 값 규칙을 읽는 즉시 확인합니다. 아래 내용을 `sales.py`에 저장합니다. `@st.cache_data`는 파일 경로가 같은 동안 이미 읽은 CSV를 재사용합니다.

```python
# sales.py
from __future__ import annotations

import pandas as pd
import streamlit as st

REQUIRED_COLUMNS = {'ordered_at', 'region', 'amount'}


@st.cache_data
def load_sales(path: str) -> pd.DataFrame:
    sales = pd.read_csv(path)
    missing = REQUIRED_COLUMNS - set(sales.columns)
    if missing:
        columns = ', '.join(sorted(missing))
        raise ValueError(f'필수 열이 없습니다: {columns}')

    sales = sales.loc[:, ['ordered_at', 'region', 'amount']].copy()
    sales['ordered_at'] = pd.to_datetime(sales['ordered_at'], errors='coerce')
    sales['region'] = sales['region'].astype('string').str.strip()
    sales['amount'] = pd.to_numeric(sales['amount'], errors='coerce')
    if (
        sales.isna().any().any()
        or (sales['region'] == '').any()
        or (sales['amount'] < 0).any()
    ):
        raise ValueError('날짜·지역·금액은 비어 있지 않고 금액은 0 이상이어야 합니다.')
    return sales
```

열 이름이 `sales_amount`처럼 다르거나 날짜가 손상되면 대시보드가 잘못된 숫자를 보여 주는 대신 `ValueError`로 중단됩니다. 파일 내용이 갱신되는 환경에서는 수정 시각을 `load_sales()` 인수에 추가하거나 `load_sales.clear()`를 관리 작업에서 호출해 캐시를 무효화합니다.

## 순수 함수으로 지역·기간 필터와 빈 결과 처리하기

이제 `sales.py` 맨 아래에 필터 함수를 추가하고, `app.py`의 제목 아래에 사이드바와 오류 처리를 추가합니다. 화면 외부의 필터 함수를 순수하게 유지하면 같은 규칙을 테스트에서 바로 확인할 수 있습니다.

```python
# sales.py에 추가
from datetime import date


def filter_sales(
    sales: pd.DataFrame,
    region: str,
    start_date: date,
    end_date: date,
) -> pd.DataFrame:
    start = pd.Timestamp(start_date)
    end = pd.Timestamp(end_date)
    result = sales.loc[sales['ordered_at'].between(start, end)].copy()
    if region != '전체':
        result = result.loc[result['region'] == region].copy()
    return result
```

```python
# app.py의 st.caption(...) 아래에 추가
try:
    sales = load_sales('sales.csv')
except (OSError, ValueError) as error:
    st.error(f'매출 파일을 읽을 수 없습니다: {error}')
    st.stop()

regions = ['전체', *sorted(sales['region'].unique())]
min_date = sales['ordered_at'].min().date()
max_date = sales['ordered_at'].max().date()
region = st.sidebar.selectbox('지역', regions)
start_date, end_date = st.sidebar.date_input(
    '주문 기간', value=(min_date, max_date), min_value=min_date, max_value=max_date,
)
filtered = filter_sales(sales, region, start_date, end_date)

if filtered.empty:
    st.info('선택한 조건에 맞는 주문이 없습니다.')
    st.stop()
```

기본 조건에서는 다섯 주문이 남고, 없는 기간을 고르면 안내 문구만 보입니다. `st.stop()`은 그 아래의 합계·차트 코드가 빈 DataFrame을 처리하려다 더 복잡한 오류를 내지 않도록 현재 실행을 끝냅니다.

## metric·line_chart·dataframe으로 분석 결과 완성하기

선택한 주문이 있을 때는 핵심 지표를 먼저 보여 주고, 일별 추세와 원본 행을 이어서 표시합니다. 아래 코드를 `app.py`의 빈 결과 처리 블록 뒤에 추가합니다.

```python
# app.py에 추가
total_sales = filtered['amount'].sum()
order_count = len(filtered)
average_order = total_sales / order_count

first, second, third = st.columns(3)
first.metric('총매출', f'{total_sales:,.0f}원')
second.metric('주문 수', f'{order_count:,}건')
third.metric('평균 주문 금액', f'{average_order:,.0f}원')

daily_sales = (
    filtered.groupby('ordered_at', as_index=False)['amount']
    .sum()
    .rename(columns={'amount': '매출'})
)
st.subheader('일별 매출')
st.line_chart(daily_sales, x='ordered_at', y='매출')

st.subheader('주문 내역')
st.dataframe(
    filtered.sort_values('ordered_at', ascending=False),
    hide_index=True,
    use_container_width=True,
)
```

기본 데이터의 총매출은 `465,000원`, 주문 수는 `5건`, 평균 주문 금액은 `93,000원`입니다. 집계는 필터 이후 DataFrame에서 하므로 지역이나 기간을 바꾸면 세 지표·차트·표가 같은 조건으로 함께 바뀝니다.

## st.download_button으로 필터 결과 내보내기

화면에서 확인한 주문 행을 다른 사람에게 보내거나 후속 작업에 쓰려면, 현재 필터 결과만 CSV로 내려받게 합니다. 아래 코드를 표 코드 다음에 추가하고 앱을 실행합니다.

```python
# app.py에 추가
download_data = filtered.to_csv(index=False).encode('utf-8-sig')
st.download_button(
    '현재 결과 CSV 내려받기',
    data=download_data,
    file_name='filtered-sales.csv',
    mime='text/csv',
)
```

```bash
streamlit run app.py
```

기본 조건에서 내려받은 파일에는 다섯 행이 있으며 `ordered_at,region,amount` 헤더가 포함됩니다. `utf-8-sig`는 스프레드시트 프로그램에서 한국어 열과 지역 이름을 열 때 인코딩을 인식하기 쉬운 선택입니다.

## pytest로 필터 계약 고정하기

마지막으로 화면을 열지 않고도 기간 경계와 지역 필터를 검사합니다. `test_sales.py`를 만들고 실행하면, CSV 형식이나 UI를 바꿔도 핵심 필터 규칙이 유지되는지 빠르게 알 수 있습니다.

```python
# test_sales.py
from datetime import date

import pandas as pd

from sales import filter_sales


def test_filters_region_and_includes_date_boundaries() -> None:
    sales = pd.DataFrame({
        'ordered_at': pd.to_datetime(['2026-07-01', '2026-07-02', '2026-07-03']),
        'region': ['서울', '부산', '서울'],
        'amount': [100, 200, 300],
    })

    result = filter_sales(
        sales,
        region='서울',
        start_date=date(2026, 7, 1),
        end_date=date(2026, 7, 3),
    )

    assert result['amount'].tolist() == [100, 300]
```

```bash
pytest -q
```

이 테스트는 시작일과 종료일을 모두 포함하는지와 지역 조건이 적용되는지를 고정합니다. 실제 앱에서는 취소 주문 제외, 환불 음수 금액, 권한별 지역 접근 같은 업무 규칙을 추가할 때 테스트와 입력 계약을 함께 확장합니다.
