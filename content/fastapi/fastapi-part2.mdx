## 고정 경로를 동적 경로보다 먼저 선언하기

FastAPI는 경로 연산을 등록한 순서대로 확인합니다. `/posts/search` 같은 고정 경로를 `/posts/{post_id}` 뒤에 두면 `search`가 `post_id` 값으로 해석되어 정수 검증에서 422가 될 수 있습니다. 더 구체적인 경로를 먼저 선언합니다.

```python
from fastapi import FastAPI

app = FastAPI()


@app.get("/posts/search")
def search_posts(q: str = "") -> dict[str, str]:
    return {"query": q}


@app.get("/posts/{post_id}")
def read_post(post_id: int) -> dict[str, int]:
    return {"post_id": post_id}
```

```bash
curl --silent "http://127.0.0.1:8000/posts/search?q=fastapi"
```

```json
{"query":"fastapi"}
```

같은 HTTP 메서드와 경로를 두 번 등록해도 자동으로 덮어쓰지 않습니다. 먼저 등록한 경로가 요청을 처리할 수 있으므로 경로 하나에는 책임 하나만 둡니다.

## Path와 Query로 숫자와 문자열 범위 제한하기

`Annotated`에 `Path`와 `Query` 메타데이터를 결합하면 Python 타입과 HTTP 검증 규칙을 함께 표현할 수 있습니다. 잘못된 값을 임의로 보정하기보다 422로 거절하면 클라이언트가 계약 위반을 바로 알 수 있습니다.

```python
from typing import Annotated

from fastapi import Path, Query


@app.get("/authors/{author_id}/posts")
def read_author_posts(
    author_id: Annotated[int, Path(ge=1)],
    offset: Annotated[int, Query(ge=0)] = 0,
    limit: Annotated[int, Query(ge=1, le=100)] = 20,
    tag: Annotated[str | None, Query(min_length=2, max_length=30)] = None,
) -> dict[str, int | str | None]:
    return {
        "author_id": author_id,
        "offset": offset,
        "limit": limit,
        "tag": tag,
    }
```

`/authors/0/posts?limit=101`은 `author_id >= 1`, `limit <= 100` 규칙을 모두 위반합니다. 응답의 `detail[].loc`을 보면 오류가 경로인지 쿼리인지 구분할 수 있습니다.

## Pydantic 모델로 JSON 요청 본문 검증하기

Pydantic 모델 타입을 경로 연산 매개변수로 선언하면 FastAPI는 `application/json` 본문을 읽어 모델로 변환합니다. `Field` 제약은 OpenAPI 문서와 런타임 검증에 함께 반영됩니다.

```python
from pydantic import BaseModel, Field


class PostCreate(BaseModel):
    title: str = Field(min_length=1, max_length=100)
    content: str = Field(min_length=1, max_length=10_000)
    tags: list[str] = Field(default_factory=list, max_length=10)


@app.post("/posts")
def create_post(post: PostCreate) -> dict[str, object]:
    return {"received": post.model_dump()}
```

```bash
curl --silent --request POST http://127.0.0.1:8000/posts \
  --header 'Content-Type: application/json' \
  --data '{"title":"타입 기반 API","content":"본문","tags":["fastapi"]}'
```

```json
{"received":{"title":"타입 기반 API","content":"본문","tags":["fastapi"]}}
```

필드 누락, 빈 제목, 너무 긴 본문처럼 모델을 만들 수 없는 입력은 경로 연산을 실행하지 않고 422로 응답합니다.

## 생략 가능한 값과 명시적인 null 구분하기

`str | None`은 `null`을 허용한다는 뜻이고, 기본값이 있어야 필드 생략도 허용됩니다. 두 개념을 구분하지 않으면 PATCH에서 “변경하지 않음”과 “값을 비움”을 처리하기 어렵습니다.

```python
from pydantic import BaseModel


class PostUpdate(BaseModel):
    title: str | None = None
    content: str | None = None


patch = PostUpdate.model_validate({"title": None})

print(patch.model_dump(exclude_unset=True))
print(PostUpdate().model_dump(exclude_unset=True))
```

```text
{'title': None}
{}
```

첫 번째 입력은 제목을 `null`로 보내려는 요청이고 두 번째 입력은 제목을 보내지 않은 요청입니다. Part 4에서는 이 차이를 이용해 수정 정책을 구현합니다.

## response_model과 상태 코드로 공개 계약 고정하기

입력 모델, 내부 모델, 공개 응답 모델을 분리하면 내부 필드가 실수로 노출되는 일을 막을 수 있습니다. 반환 객체에 `internal_note`가 있어도 `response_model`에 없는 필드는 응답에서 제거됩니다.

```python
from fastapi import status


class PostPublic(BaseModel):
    id: int
    title: str
    content: str


@app.post(
    "/public-posts",
    response_model=PostPublic,
    status_code=status.HTTP_201_CREATED,
)
def create_public_post(post: PostCreate) -> dict[str, object]:
    return {
        "id": 1,
        **post.model_dump(exclude={"tags"}),
        "internal_note": "응답에서 제외",
    }
```

정상 응답은 201과 공개 필드만 반환합니다. 반환값이 `PostPublic` 계약을 충족하지 못하면 서버 구현 오류이므로 500 응답이 발생합니다. 입력 오류와 출력 오류의 책임이 다르다는 점을 구분해야 합니다.
