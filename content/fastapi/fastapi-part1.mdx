## FastAPI의 역할과 이 섹션의 학습 범위 이해하기

FastAPI는 Python 타입 힌트를 이용해 HTTP 요청을 Python 값으로 변환하고, 입력을 검증하며, 응답과 OpenAPI 문서를 만드는 ASGI 웹 프레임워크입니다. 웹 서버는 Uvicorn이 담당하고 데이터 검증은 Pydantic이 담당합니다. 이 구분을 알면 문제가 생겼을 때 애플리케이션 코드, 서버 설정, 데이터 모델 중 어디를 살펴봐야 하는지 판단하기 쉽습니다.

이 섹션은 Python 3.12를 기준으로 경로 연산, 검증, 의존성 주입, 인증과 인가, 테스트, 배포를 차례로 다룹니다. 데이터베이스 모델링은 SQLModel 섹션에서, HTML 템플릿은 Jinja2 섹션에서 더 자세히 학습합니다.

## 가상환경에 FastAPI 표준 의존성 설치하기

프로젝트마다 독립된 가상환경을 만들면 패키지 버전 충돌을 피할 수 있습니다. `fastapi[standard]`에는 FastAPI CLI, Uvicorn, 폼 파싱 등 튜토리얼에서 사용할 표준 의존성이 포함됩니다.

```bash
mkdir fastapi-study
cd fastapi-study
python3.12 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install "fastapi[standard]"
python --version
```

```text
Python 3.12.x
```

Windows PowerShell에서는 활성화 명령만 `.venv\Scripts\Activate.ps1`로 바꿉니다. 설치 결과의 패키지 버전과 다운로드 로그는 실행 시점과 운영체제에 따라 달라집니다.

## 최소 애플리케이션을 만들고 개발 서버 실행하기

다음 코드를 `app/main.py`에 저장합니다. `app`은 FastAPI 애플리케이션 객체이며, 데코레이터와 함수의 조합을 경로 연산(path operation)이라고 부릅니다.

```python
# app/main.py
from fastapi import FastAPI

app = FastAPI(title="FastAPI Study")


@app.get("/")
def read_root() -> dict[str, str]:
    return {"message": "Hello, FastAPI!"}
```

개발 중에는 `fastapi dev`를 사용합니다. 파일 변경을 감지해 서버를 다시 시작하므로 운영 환경에서는 사용하지 않습니다.

```bash
fastapi dev app/main.py
```

서버가 준비된 뒤 다른 터미널에서 요청을 보냅니다.

```bash
curl --silent http://127.0.0.1:8000/
```

```json
{"message":"Hello, FastAPI!"}
```

## 경로 연산에서 경로 매개변수와 JSON 응답 사용하기

경로에 `{post_id}`를 선언하고 함수 매개변수에 `int` 타입을 지정하면 FastAPI가 URL 문자열을 정수로 변환합니다. 변환할 수 없는 값은 함수가 실행되기 전에 422 응답으로 거절됩니다.

```python
@app.get("/posts/{post_id}")
def read_post(post_id: int) -> dict[str, int | str]:
    return {"post_id": post_id, "title": f"Post {post_id}"}
```

```bash
curl --silent http://127.0.0.1:8000/posts/7
```

```json
{"post_id":7,"title":"Post 7"}
```

`/posts/not-a-number`를 요청하면 상태 코드가 422가 됩니다. 이 자동 변환과 검증은 함수 시그니처가 API 계약으로 사용된 결과입니다.

## 자동 생성된 API 문서와 OpenAPI 스키마 확인하기

FastAPI는 등록된 경로, 매개변수 타입, 응답 모델을 바탕으로 OpenAPI 스키마를 생성합니다. 서버를 실행한 상태에서 다음 URL을 확인합니다.

```text
Swagger UI: http://127.0.0.1:8000/docs
ReDoc:      http://127.0.0.1:8000/redoc
OpenAPI:    http://127.0.0.1:8000/openapi.json
```

스키마도 일반 HTTP 응답이므로 자동화 도구에서 직접 검사할 수 있습니다.

```bash
curl --silent http://127.0.0.1:8000/openapi.json | python -m json.tool
```

출력의 `paths`에는 `/`와 `/posts/{post_id}`가 포함됩니다. 문서에 기대한 경로나 입력 필드가 없다면 서버를 다시 시작하기 전에 데코레이터, 타입 힌트, 라우터 등록 여부부터 확인합니다.
