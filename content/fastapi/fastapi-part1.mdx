FastAPI는 HTTP 요청을 함수로 받고 응답을 JSON으로 돌려주는 Python 웹 API 프레임워크입니다. 이 노트에서는 SQLModel로 블로그 포스트를 저장하면서 제목·본문·썸네일 URL을 다루는 작은 API를 만듭니다. 첫 세 토픽의 준비 코드는 한 번만 실행하고, 뒤의 엔드포인트를 차례로 추가합니다.

## FastAPI와 SQLModel 설치하고 개발 서버 실행하기

블로그 API를 실행하고 자동 API 문서를 확인할 때 FastAPI와 SQLModel을 함께 설치합니다. SQLite는 Python에 포함되어 있어 로컬 학습용으로 별도 서버가 필요 없습니다.

```bash
python3.12 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install 'fastapi[standard]' sqlmodel
fastapi dev blog_api.py
```

개발 서버가 실행되면 `/docs`에서 요청 본문을 입력해 API를 시험할 수 있습니다. `fastapi dev`는 개발 중 변경을 감지해 서버를 다시 시작합니다.

## SQLModel 테이블로 BlogPost 저장 규칙 만들기

제목, 본문, 썸네일 URL을 데이터베이스에 저장할 때 `table=True`인 SQLModel 클래스를 선언합니다. 검색할 제목에는 인덱스를 두고, 새 행의 ID는 데이터베이스가 만들도록 `None`을 기본값으로 둡니다.

```python
from sqlmodel import Field, SQLModel


class BlogPost(SQLModel, table=True):
    id: int | None = Field(default=None, primary_key=True)
    title: str = Field(index=True)
    content: str
    thumbnail_url: str | None = None
```

`table=True`가 있는 `BlogPost`는 테이블과 행을 표현합니다. 다음 Part에서 작성자와 여러 사진 관계를 추가하므로, 이 단계에서는 포스트 자체의 필드만 둡니다.

## create_engine과 lifespan으로 테이블 준비하기

API가 시작될 때 테이블이 없으면 만들고, SQLite를 FastAPI 요청에서 사용할 엔진을 준비합니다. 파일 기반 SQLite를 FastAPI에서 쓸 때는 `check_same_thread=False`가 필요합니다.

```python
from contextlib import asynccontextmanager

from fastapi import FastAPI
from sqlmodel import SQLModel, create_engine


engine = create_engine(
    'sqlite:///blog.db',
    connect_args={'check_same_thread': False},
)


@asynccontextmanager
async def lifespan(app: FastAPI):
    SQLModel.metadata.create_all(engine)
    yield


app = FastAPI(lifespan=lifespan)
```

`create_engine()`은 데이터베이스와 통신할 설정을 만들고, `create_all()`은 등록된 테이블을 만듭니다. 이미 있는 테이블의 구조를 변경하지는 않으므로, 운영 환경의 스키마 변경은 마이그레이션으로 관리합니다.

## Depends와 Session으로 요청마다 작업 단위 만들기

각 HTTP 요청이 데이터베이스 작업을 수행할 때 독립적인 `Session`을 받도록 의존성을 선언합니다. `yield` 뒤에 세션이 닫히므로 엔드포인트가 전역 세션을 공유하지 않습니다.

```python
from typing import Annotated

from fastapi import Depends
from sqlmodel import Session


def get_session():
    with Session(engine) as session:
        yield session


SessionDep = Annotated[Session, Depends(get_session)]
```

이후 엔드포인트의 `session: SessionDep`은 FastAPI가 공급하는 요청 전용 세션입니다. 데이터베이스 연결, 인증처럼 반복되는 경계를 의존성으로 모을 수 있습니다.

## POST /posts로 새 블로그 포스트 저장하기

클라이언트가 보낸 포스트를 저장하고 생성된 ID를 포함해 돌려줄 때 요청 모델과 테이블 모델을 구분합니다. 요청에는 ID를 받지 않고, 응답에는 ID를 포함합니다.

```python
class BlogPostCreate(SQLModel):
    title: str
    content: str
    thumbnail_url: str | None = None


class BlogPostPublic(SQLModel):
    id: int
    title: str
    content: str
    thumbnail_url: str | None


@app.post('/posts', response_model=BlogPostPublic, status_code=201)
def create_post(post: BlogPostCreate, session: SessionDep) -> BlogPost:
    db_post = BlogPost.model_validate(post)
    session.add(db_post)
    session.commit()
    session.refresh(db_post)
    return db_post
```

`commit()`으로 저장을 확정하고 `refresh()`로 데이터베이스가 만든 `id`를 다시 읽습니다. `response_model`은 응답이 공개 모델의 필드만 갖도록 제한합니다.

```json
{
  "title": "SQLModel로 시작하는 블로그",
  "content": "첫 포스트입니다.",
  "thumbnail_url": "https://images.example.com/cover.png"
}
```

## select와 Query로 포스트 목록 페이지 나누기

블로그 홈에서 포스트 일부만 최신 ID 순으로 보여 줄 때 `offset`과 `limit`을 쿼리 매개변수로 받습니다. 너무 큰 요청을 막기 위해 `limit`의 최대값을 선언합니다.

```python
from fastapi import Query
from sqlmodel import select


@app.get('/posts', response_model=list[BlogPostPublic])
def list_posts(
    session: SessionDep,
    offset: int = 0,
    limit: int = Query(default=20, ge=1, le=100),
) -> list[BlogPost]:
    statement = select(BlogPost).order_by(BlogPost.id.desc()).offset(offset).limit(limit)
    return session.exec(statement).all()
```

`offset`은 건너뛸 행 수이고 `limit`은 가져올 최대 행 수입니다. 페이지를 안정적으로 표시하려면 이 예제처럼 `order_by()`로 순서를 고정합니다.

## HTTPException으로 없는 포스트를 404로 응답하기

상세 페이지가 존재하지 않는 포스트 ID를 요청했을 때 빈 성공 응답 대신 404를 돌려줍니다. `Session.get()`은 기본 키로 한 행을 찾고, 없으면 `None`을 반환합니다.

```python
from fastapi import HTTPException


@app.get('/posts/{post_id}', response_model=BlogPostPublic)
def get_post(post_id: int, session: SessionDep) -> BlogPost:
    post = session.get(BlogPost, post_id)
    if post is None:
        raise HTTPException(status_code=404, detail='포스트를 찾을 수 없습니다.')
    return post
```

```text
GET /posts/999
404 {"detail":"포스트를 찾을 수 없습니다."}
```

라우트 함수의 반환 타입은 성공 시의 `BlogPost`이고, 실패 흐름은 `HTTPException`으로 명확히 분리됩니다.

## DELETE `/posts/{post_id}`로 포스트 삭제하기

작성자가 자신의 포스트를 지울 때는 대상을 먼저 조회하고, 존재할 때만 삭제를 확정합니다. 권한 확인은 Part 4에서 현재 작성자 의존성과 함께 추가합니다.

```python
@app.delete('/posts/{post_id}', status_code=204)
def delete_post(post_id: int, session: SessionDep) -> None:
    post = session.get(BlogPost, post_id)
    if post is None:
        raise HTTPException(status_code=404, detail='포스트를 찾을 수 없습니다.')
    session.delete(post)
    session.commit()
```

`session.delete()`만으로는 영구 반영되지 않으며 `commit()`이 필요합니다. 실제 서비스에서는 먼저 해당 포스트를 수정할 권한이 있는지 확인해야 합니다.
