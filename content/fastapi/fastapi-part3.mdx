블로그 API가 커지면 부분 수정, 검색 결과의 일관성, 여러 행을 바꾸는 작업, 테스트 가능한 의존성 경계가 중요해집니다. 이 노트는 `Author`, `BlogPost`, `PostImage`, `SessionDep`가 준비된 상태에서 각 문제를 SQLModel 중심으로 확장합니다.

## BlogPostUpdate와 exclude_unset으로 보낸 필드만 수정하기

제목만 바꾸는 요청이 본문이나 썸네일을 기본값으로 덮어쓰지 않게 모든 수정 필드를 선택적으로 선언합니다. `exclude_unset=True`은 요청에 실제로 포함된 필드만 골라냅니다.

```python
class BlogPostUpdate(SQLModel):
    title: str | None = None
    content: str | None = None
    thumbnail_url: str | None = None


@app.patch('/posts/{post_id}', response_model=BlogPostPublic)
def update_post(
    post_id: int,
    patch: BlogPostUpdate,
    session: SessionDep,
) -> BlogPost:
    post = session.get(BlogPost, post_id)
    if post is None:
        raise HTTPException(status_code=404, detail='포스트를 찾을 수 없습니다.')

    post.sqlmodel_update(patch.model_dump(exclude_unset=True))
    session.add(post)
    session.commit()
    session.refresh(post)
    return post
```

`thumbnail_url: null`을 명시적으로 보내면 그 값은 포함되어 썸네일을 지울 수 있습니다. 반대로 해당 키 자체를 보내지 않으면 기존 값이 유지됩니다.

## 사진 교체를 Session.begin 트랜잭션으로 묶기

본문 사진 전체를 새 배열로 교체할 때 기존 사진 삭제와 새 사진 삽입이 함께 성공해야 합니다. `session.begin()` 블록에서 작업하면 예외가 발생한 경우 변경이 커밋되지 않습니다.

```python
class PostImagesReplace(SQLModel):
    images: list[PostImageCreate]


def replace_images(post_id: int, data: PostImagesReplace, session: Session) -> None:
    with session.begin():
        post = session.get(BlogPost, post_id)
        if post is None:
            raise HTTPException(status_code=404, detail='포스트를 찾을 수 없습니다.')
        old_images = session.exec(
            select(PostImage).where(PostImage.post_id == post_id)
        ).all()
        for image in old_images:
            session.delete(image)
        session.add_all([
            PostImage(post_id=post_id, url=image.url, alt_text=image.alt_text, position=index)
            for index, image in enumerate(data.images, start=1)
        ])
```

트랜잭션 안에서는 `commit()`을 직접 호출하지 않습니다. 블록이 정상 종료될 때 커밋되고 예외가 전파되면 롤백됩니다. 조회도 블록 안에서 시작해 이미 열린 트랜잭션과 충돌하지 않게 합니다.

## select와 ilike로 제목 검색 결과 만들기

블로그 검색창이 제목에 검색어를 포함한 포스트를 찾을 때 SQL 표현식으로 조건을 만듭니다. 빈 검색어는 전체 목록이 되지 않도록 요청 단계에서 거절합니다.

```python
from fastapi import Query


@app.get('/posts/search', response_model=list[BlogPostListItem])
def search_posts(
    session: SessionDep,
    q: str = Query(min_length=1, max_length=100),
) -> list[BlogPost]:
    pattern = f'%{q.strip()}%'
    statement = (
        select(BlogPost)
        .where(BlogPost.title.ilike(pattern))
        .order_by(BlogPost.id.desc())
        .limit(20)
    )
    return session.exec(statement).all()
```

`ilike()`의 대소문자 처리 방식은 데이터베이스마다 다를 수 있습니다. 검색 규모가 커지면 부분 문자열 검색 대신 데이터베이스의 전문 검색 기능과 인덱스를 검토합니다.

## 작성자 조건으로 내 포스트만 읽기

관리 화면처럼 특정 작성자의 포스트만 볼 때 작성자 ID를 URL에 맡기기보다 인증 결과에서 받은 ID를 쿼리 조건에 씁니다. 학습용으로 요청 헤더를 읽는 현재 작성자 의존성을 먼저 준비합니다. 실제 서비스에서는 검증된 세션이나 액세스 토큰에서 이 ID를 얻습니다.

```python
from typing import Annotated

from fastapi import Depends, Header


def get_current_author_id(
    x_author_id: Annotated[int, Header()],
    session: SessionDep,
) -> int:
    if session.get(Author, x_author_id) is None:
        raise HTTPException(status_code=401, detail='로그인한 작성자를 찾을 수 없습니다.')
    return x_author_id


CurrentAuthorId = Annotated[int, Depends(get_current_author_id)]
```

```python
def list_my_posts(current_author_id: int, session: Session) -> list[BlogPost]:
    statement = (
        select(BlogPost)
        .where(BlogPost.author_id == current_author_id)
        .order_by(BlogPost.id.desc())
    )
    return session.exec(statement).all()
```

이 경계는 클라이언트가 다른 작성자의 ID를 바꿔 요청해도 결과를 확대할 수 없게 합니다. 같은 원칙으로 수정·삭제 대상의 `author_id`도 현재 작성자와 비교합니다.

## 소유자 확인으로 다른 작성자의 수정 막기

포스트를 수정하거나 사진을 바꾸기 전에 현재 작성자가 소유자인지 확인합니다. 조회와 권한 검사를 서비스 함수에 모으면 PATCH, DELETE, 사진 API가 같은 규칙을 재사용합니다.

```python
def get_owned_post(post_id: int, author_id: int, session: Session) -> BlogPost:
    post = session.get(BlogPost, post_id)
    if post is None:
        raise HTTPException(status_code=404, detail='포스트를 찾을 수 없습니다.')
    if post.author_id != author_id:
        raise HTTPException(status_code=403, detail='이 포스트를 수정할 권한이 없습니다.')
    return post
```

앞의 부분 수정 코드는 이 확인 함수를 사용하도록 바꿉니다.

```python
@app.patch('/posts/{post_id}/owned', response_model=BlogPostPublic)
def update_owned_post(
    post_id: int,
    patch: BlogPostUpdate,
    current_author_id: CurrentAuthorId,
    session: SessionDep,
) -> BlogPost:
    post = get_owned_post(post_id, current_author_id, session)
    post.sqlmodel_update(patch.model_dump(exclude_unset=True))
    session.commit()
    session.refresh(post)
    return post
```

404와 403을 구분하면 클라이언트는 존재하지 않는 리소스와 권한 없는 리소스를 다르게 처리할 수 있습니다. 공개 범위를 감춰야 하는 제품은 두 경우의 응답 정책을 별도로 정합니다.

## dependency_overrides로 테스트 세션 바꾸기

API 테스트가 개발용 데이터베이스를 건드리지 않게 `get_session` 의존성을 테스트 전용 엔진으로 교체합니다. SQLite 메모리 데이터베이스는 테스트마다 빠르게 새 상태를 만들 수 있습니다.

```python
from fastapi.testclient import TestClient
from sqlalchemy.pool import StaticPool


test_engine = create_engine(
    'sqlite://',
    connect_args={'check_same_thread': False},
    poolclass=StaticPool,
)
SQLModel.metadata.create_all(test_engine)


def get_test_session():
    with Session(test_engine) as session:
        yield session


app.dependency_overrides[get_session] = get_test_session
client = TestClient(app)
```

`StaticPool`은 테스트 클라이언트와 요청 세션이 같은 메모리 SQLite 연결을 사용하게 합니다. 테스트가 끝나면 `app.dependency_overrides.clear()`로 교체를 해제합니다.

## TestClient로 생성·수정·권한 실패 흐름 확인하기

포스트 API가 성공 흐름과 실패 흐름 모두 올바른 상태 코드를 돌려주는지 `TestClient`로 확인합니다. 테스트 데이터의 작성자와 현재 작성자 의존성은 각 테스트에서 명시적으로 준비합니다.

```python
def test_update_rejects_another_authors_post() -> None:
    with Session(test_engine) as session:
        author = Author(name='민지')
        another_author = Author(id=999, name='도윤')
        post = BlogPost(title='비공개 초안', content='본문', author=author)
        session.add_all([another_author, post])
        session.commit()
        session.refresh(post)

    response = client.patch(
        f'/posts/{post.id}/owned',
        json={'title': '바꿀 수 없는 제목'},
        headers={'X-Author-Id': '999'},
    )

    assert response.status_code == 403
```

권한 실패를 테스트하면 단순한 모델 갱신 코드가 소유권 규칙을 우회하지 않는지 확인할 수 있습니다. 실제 인증을 연결한 뒤에도 이 테스트는 요청 헤더 대신 인증 토큰을 사용하도록 바꿔 같은 403 규칙을 유지합니다.
