## 인증·인가와 password flow의 범위 구분하기

인증은 요청자를 확인하고 인가는 그 요청자가 작업을 해도 되는지 판단합니다. 이 노트는 직접 관리하는 사용자명과 비밀번호로 Bearer 토큰을 발급하는 OAuth2 password flow를 학습 예제로 사용합니다. 브라우저 기반 공개 서비스라면 자체 로그인보다 외부 Identity Provider의 OpenID Connect와 Authorization Code flow를 먼저 검토합니다.

```text
POST /token             인증과 토큰 발급
POST /posts             인증된 사용자를 작성자로 사용
PATCH·DELETE /posts/{id} 작성자만 허용
```

JWT는 서명되지만 암호화되지는 않습니다. payload에 비밀번호나 개인정보를 넣지 않고 TLS로 전송합니다.

## 환경 설정과 Argon2 비밀번호 해시 준비하기

비밀키는 코드에 하드코딩하지 않고, 비밀번호는 복호화 가능한 값이 아니라 Argon2 해시로 저장합니다.

```python
# app/settings.py
from functools import lru_cache
from typing import Literal

from pydantic import Field
from pydantic_settings import BaseSettings, SettingsConfigDict


class Settings(BaseSettings):
    secret_key: str = Field(min_length=32)
    access_token_expire_minutes: int = Field(default=30, ge=1, le=1440)
    algorithm: Literal["HS256"] = "HS256"
    model_config = SettingsConfigDict(env_file=".env", extra="ignore")


@lru_cache
def get_settings() -> Settings:
    return Settings()
```

```python
# app/security.py
from pwdlib import PasswordHash

password_hash = PasswordHash.recommended()
DUMMY_HASH = password_hash.hash("not-a-real-password")


def hash_password(password: str) -> str:
    return password_hash.hash(password)
```

없는 사용자도 `password_hash.verify(password, DUMMY_HASH)`를 수행하고 실패 메시지를 통일하면 사용자 존재 여부에 따른 응답 차이를 줄일 수 있습니다. 운영에서는 `.env`를 이미지에 복사하지 않고 플랫폼 secret, 로그인 횟수 제한, 감사 로그를 함께 사용합니다.

## exp와 sub가 필수인 JWT 발급·검증하기

`sub`에는 고유한 사용자 식별자를, `exp`에는 UTC 만료 시각을 넣습니다. 검증할 때 허용 알고리즘을 고정하고 claim 자체도 필수로 요구합니다.

```python
# app/security.py
from datetime import datetime, timedelta, timezone

import jwt

from app.settings import Settings


def create_access_token(subject: str, settings: Settings) -> str:
    expires_at = datetime.now(timezone.utc) + timedelta(
        minutes=settings.access_token_expire_minutes
    )
    return jwt.encode(
        {"sub": subject, "exp": expires_at},
        settings.secret_key,
        algorithm=settings.algorithm,
    )


def decode_subject(token: str, settings: Settings) -> str:
    payload = jwt.decode(
        token,
        settings.secret_key,
        algorithms=[settings.algorithm],
        options={"require": ["exp", "sub"]},
    )
    subject = payload["sub"]
    if not isinstance(subject, str) or not subject:
        raise jwt.InvalidTokenError("invalid sub")
    return subject
```

PyJWT는 `exp`가 있으면 만료를 검사하지만 `options.require`가 없으면 claim이 빠진 토큰을 허용할 수 있습니다. 토큰 헤더가 제안하는 알고리즘을 검증 설정으로 사용하지 않습니다.

## 토큰 엔드포인트와 현재 사용자 의존성 연결하기

`OAuth2PasswordRequestForm`은 URL 인코딩된 `username`과 `password`를 읽고, `OAuth2PasswordBearer`는 이후 요청의 Bearer 헤더를 읽습니다.

```python
# app/auth.py
from dataclasses import dataclass
from typing import Annotated

import jwt
from fastapi import APIRouter, Depends, HTTPException, status
from fastapi.security import OAuth2PasswordBearer, OAuth2PasswordRequestForm
from pydantic import BaseModel

from app.security import (
    DUMMY_HASH,
    create_access_token,
    decode_subject,
    hash_password,
    password_hash,
)
from app.settings import Settings, get_settings


@dataclass
class UserRecord:
    username: str
    hashed_password: str
    disabled: bool = False


class Token(BaseModel):
    access_token: str
    token_type: str


users = {
    "alice": UserRecord("alice", hash_password("correct-password")),
}
router = APIRouter(tags=["auth"])
oauth2_scheme = OAuth2PasswordBearer(tokenUrl="token")


def credentials_exception() -> HTTPException:
    return HTTPException(
        status_code=status.HTTP_401_UNAUTHORIZED,
        detail="Could not validate credentials",
        headers={"WWW-Authenticate": "Bearer"},
    )


@router.post("/token", response_model=Token)
def login(
    form: Annotated[OAuth2PasswordRequestForm, Depends()],
    settings: Annotated[Settings, Depends(get_settings)],
) -> Token:
    user = users.get(form.username)
    candidate = user.hashed_password if user else DUMMY_HASH
    valid = password_hash.verify(form.password, candidate)
    if user is None or not valid or user.disabled:
        raise credentials_exception()
    return Token(
        access_token=create_access_token(user.username, settings),
        token_type="bearer",
    )


def get_current_user(
    token: Annotated[str, Depends(oauth2_scheme)],
    settings: Annotated[Settings, Depends(get_settings)],
) -> UserRecord:
    try:
        username = decode_subject(token, settings)
    except jwt.InvalidTokenError as exc:
        raise credentials_exception() from exc
    user = users.get(username)
    if user is None or user.disabled:
        raise credentials_exception()
    return user


CurrentUser = Annotated[UserRecord, Depends(get_current_user)]
```

예제의 `alice`는 Part 7을 바로 실행하기 위한 인메모리 계정입니다. 운영에서는 사용자 DB와 가입·관리 흐름으로 교체합니다. 비밀번호와 토큰은 로그에 남기지 않으며 사용자 없음, 잘못된 비밀번호, 만료 토큰은 모두 401로 응답합니다.

## 게시물 소유권으로 401과 403 구분하기

인증 정보가 없거나 유효하지 않으면 401, 인증은 되었지만 다른 사용자의 게시물을 변경하려 하면 403입니다. 작성자는 요청 본문이 아니라 `CurrentUser`에서 가져옵니다.

```python
def require_owner(post: PostRecord, current_user: UserRecord) -> None:
    if post.author != current_user.username:
        raise HTTPException(status_code=403, detail="Not enough permissions")


# 생성 경로
created = repository.add(post, author=current_user.username)

# PATCH와 DELETE에서 조회 직후 적용
stored = get_post_or_404(post_id, repository)
require_owner(stored, current_user)
```

관리자 역할이 필요하면 조건문을 경로마다 복사하지 않고 별도 정책 의존성이나 서비스 계층으로 모읍니다.

## 필수 claim과 소유권 정책을 테스트하기

보안 경계는 정상 로그인뿐 아니라 누락된 claim, 만료, 다른 작성자의 접근을 테스트합니다.

```python
from types import SimpleNamespace

import jwt
import pytest
from fastapi import HTTPException

from app.api.posts import require_owner
from app.auth import UserRecord
from app.security import decode_subject
from app.settings import Settings


def test_rejects_token_without_exp() -> None:
    settings = Settings(secret_key="t" * 32)
    token = jwt.encode(
        {"sub": "alice"},
        settings.secret_key,
        algorithm=settings.algorithm,
    )
    with pytest.raises(jwt.InvalidTokenError):
        decode_subject(token, settings)


def test_other_user_is_forbidden() -> None:
    post = SimpleNamespace(author="bob")
    alice = UserRecord("alice", "not-used")
    with pytest.raises(HTTPException) as exc_info:
        require_owner(post, alice)
    assert exc_info.value.status_code == 403
```

만료된 `exp`도 `ExpiredSignatureError`가 발생하는지 확인합니다. Part 7에서는 이 정책들이 실제 401·403 HTTP 응답으로 연결되는지 검증합니다.
