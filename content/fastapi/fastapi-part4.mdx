## 생성·수정·공개 모델의 책임 분리하기

클라이언트가 보내는 값, 부분 수정 값, 서버가 공개하는 값은 서로 다릅니다. 하나의 모델을 모두에 재사용하면 서버가 정해야 할 ID와 작성자를 클라이언트가 지정하거나 내부 필드가 응답에 노출될 수 있습니다.

```python
from datetime import datetime

from pydantic import BaseModel, ConfigDict, Field


class PostCreate(BaseModel):
    title: str = Field(min_length=1, max_length=100)
    content: str = Field(min_length=1, max_length=10_000)


class PostUpdate(BaseModel):
    title: str | None = Field(default=None, min_length=1, max_length=100)
    content: str | None = Field(default=None, min_length=1, max_length=10_000)


class PostPublic(BaseModel):
    model_config = ConfigDict(from_attributes=True)

    id: int
    title: str
    content: str
    author: str
    created_at: datetime
```

`PostCreate`에는 `id`, `author`, `created_at`이 없습니다. 이 값들은 인증 결과와 서버 상태에서 만들며, 클라이언트 입력을 그대로 신뢰하지 않습니다.

## 학습용 인메모리 저장소의 계약 만들기

라우터와 저장 방식을 분리하면 HTTP 처리와 데이터 저장을 각각 테스트할 수 있습니다. 다음 저장소는 FastAPI 학습을 위한 최소 구현이며 서버 재시작 시 데이터가 사라집니다.

```python
from dataclasses import dataclass
from datetime import datetime, timezone


@dataclass
class PostRecord:
    id: int
    title: str
    content: str
    author: str
    created_at: datetime


class DuplicateTitleError(Exception):
    pass


class PostRepository:
    def __init__(self) -> None:
        self._posts: dict[int, PostRecord] = {}
        self._next_id = 1

    def add(self, data: PostCreate, author: str) -> PostRecord:
        if any(
            post.title.casefold() == data.title.casefold()
            for post in self._posts.values()
        ):
            raise DuplicateTitleError(data.title)

        post = PostRecord(
            id=self._next_id,
            title=data.title,
            content=data.content,
            author=author,
            created_at=datetime.now(timezone.utc),
        )
        self._posts[post.id] = post
        self._next_id += 1
        return post

    def get(self, post_id: int) -> PostRecord | None:
        return self._posts.get(post_id)

    def all(self) -> list[PostRecord]:
        return sorted(self._posts.values(), key=lambda post: post.id)

    def delete(self, post_id: int) -> None:
        del self._posts[post_id]


repository = PostRepository()
```

여러 워커의 메모리는 공유되지 않으므로 이 저장소를 운영에 사용하면 요청을 처리한 워커에 따라 데이터가 달라집니다. 운영에서는 SQLModel 같은 데이터 계층과 공유 DB를 사용하되 라우터의 HTTP 계약은 유지합니다.

## 생성과 제한된 목록 조회 구현하기

생성 성공에는 201을 사용하고 목록에는 검증된 `offset`과 `limit`을 적용합니다. 이 단계에서는 인증을 배우기 전이므로 서버가 정한 `demo-user`를 작성자로 사용하며 Part 5에서 현재 사용자 의존성으로 교체합니다.

```python
from typing import Annotated

from fastapi import APIRouter, Depends, HTTPException, Query, status

router = APIRouter(prefix="/posts", tags=["posts"])


def get_repository() -> PostRepository:
    return repository


Repo = Annotated[PostRepository, Depends(get_repository)]


@router.post("", response_model=PostPublic, status_code=status.HTTP_201_CREATED)
def create_post(post: PostCreate, repository: Repo) -> PostRecord:
    try:
        return repository.add(post, author="demo-user")
    except DuplicateTitleError as exc:
        raise HTTPException(status_code=409, detail="Title already exists") from exc


@router.get("", response_model=list[PostPublic])
def read_posts(
    repository: Repo,
    offset: Annotated[int, Query(ge=0)] = 0,
    limit: Annotated[int, Query(ge=1, le=100)] = 20,
) -> list[PostRecord]:
    return repository.all()[offset : offset + limit]
```

```bash
curl --silent --request POST http://127.0.0.1:8000/posts \
  --header 'Content-Type: application/json' \
  --data '{"title":"첫 글","content":"FastAPI CRUD"}'
```

```json
{"id":1,"title":"첫 글","content":"FastAPI CRUD","author":"demo-user","created_at":"2026-07-13T12:00:00Z"}
```

`created_at`은 요청 시점에 따라 달라집니다. 같은 제목을 다시 생성하면 저장소의 `DuplicateTitleError`를 409 응답으로 바꿉니다. 운영에서는 요청 전 조회만 믿지 말고 DB의 UNIQUE 제약과 충돌 처리를 함께 사용합니다.

## 검색을 동적 상세 경로보다 먼저 등록하기

검색은 고정 경로이므로 `/{post_id}`보다 앞에 선언합니다. 검색어가 없거나 지나치게 길면 함수 실행 전에 422로 거절합니다.

```python
@router.get("/search", response_model=list[PostPublic])
def search_posts(
    repository: Repo,
    q: Annotated[str, Query(min_length=1, max_length=100)],
) -> list[PostRecord]:
    keyword = q.casefold()
    return [
        post
        for post in repository.all()
        if keyword in post.title.casefold() or keyword in post.content.casefold()
    ]


@router.get("/{post_id}", response_model=PostPublic)
def read_post(post_id: int, repository: Repo) -> PostRecord:
    post = repository.get(post_id)
    if post is None:
        raise HTTPException(status_code=404, detail="Post not found")
    return post
```

```bash
curl --silent "http://127.0.0.1:8000/posts/search?q=crud"
```

```json
[{"id":1,"title":"첫 글","content":"FastAPI CRUD","author":"demo-user","created_at":"2026-07-13T12:00:00Z"}]
```

`created_at`은 생성 시점에 따라 달라집니다. 목록에 일치하는 글이 없으면 오류가 아니라 빈 배열 `[]`을 반환합니다. 특정 ID 조회 결과가 없을 때만 404를 사용합니다.

## PATCH에서 생략과 null을 구분해 수정하기

`exclude_unset=True`는 클라이언트가 실제로 보낸 필드만 반환합니다. 이 API에서는 제목과 본문을 `null`로 지우는 것을 허용하지 않으므로 명시적인 `null`은 422로 거절합니다.

```python
@router.patch("/{post_id}", response_model=PostPublic)
def update_post(
    post_id: int,
    update: PostUpdate,
    repository: Repo,
) -> PostRecord:
    post = repository.get(post_id)
    if post is None:
        raise HTTPException(status_code=404, detail="Post not found")

    changes = update.model_dump(exclude_unset=True)
    if not changes:
        return post
    if any(value is None for value in changes.values()):
        raise HTTPException(
            status_code=422,
            detail="title and content cannot be null",
        )

    for field, value in changes.items():
        setattr(post, field, value)
    return post
```

빈 문자열은 `PostUpdate`의 `min_length`에서 먼저 거절됩니다. 필드가 생략되었는지, `null`인지, 빈 문자열인지 각각 다른 입력이라는 점을 테스트로 고정합니다.

## 삭제 결과와 오류 상태를 명확히 반환하기

삭제가 성공하면 본문 없는 204를 반환합니다. 이미 없어진 리소스를 다시 삭제하면 성공처럼 숨기지 않고 404를 반환하도록 계약을 정합니다.

```python
from fastapi import Response


@router.delete("/{post_id}", status_code=204)
def delete_post(post_id: int, repository: Repo) -> Response:
    if repository.get(post_id) is None:
        raise HTTPException(status_code=404, detail="Post not found")
    repository.delete(post_id)
    return Response(status_code=204)
```

201·204·404·422 같은 상태 코드는 클라이언트의 분기 기준입니다. 성공 응답 본문에 `{"ok": false}`를 넣는 방식보다 HTTP 상태와 응답 모델로 결과를 표현해야 자동 문서와 테스트도 같은 계약을 사용합니다.
