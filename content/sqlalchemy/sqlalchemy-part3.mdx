## selectinload로 목록의 컬렉션 관계 미리 읽기

작성자 목록과 각 작성자의 글 목록처럼 여러 부모 행의 컬렉션 관계를 함께 보여 줄 때 `selectinload()`를 선택합니다. 기본 지연 로딩에 맡기면 각 `author.books` 접근마다 추가 쿼리가 생겨 N+1 문제가 될 수 있는데, `selectinload()`는 부모 조회 뒤 필요한 자식들을 한 번 더 모아 읽습니다.

```python
from sqlalchemy import ForeignKey, String, create_engine, select
from sqlalchemy.orm import (
    DeclarativeBase,
    Mapped,
    Session,
    mapped_column,
    relationship,
    selectinload,
)


class Base(DeclarativeBase):
    pass


class Author(Base):
    __tablename__ = 'authors'

    id: Mapped[int] = mapped_column(primary_key=True)
    name: Mapped[str] = mapped_column(String(50))
    books: Mapped[list['Book']] = relationship(back_populates='author')


class Book(Base):
    __tablename__ = 'books'

    id: Mapped[int] = mapped_column(primary_key=True)
    title: Mapped[str] = mapped_column(String(100))
    author_id: Mapped[int] = mapped_column(ForeignKey('authors.id'))
    author: Mapped[Author] = relationship(back_populates='books')


engine = create_engine('sqlite://')
Base.metadata.create_all(engine)

with Session(engine) as session:
    session.add_all([
        Author(name='민지', books=[Book(title='SQL 입문'), Book(title='ORM 실습')]),
        Author(name='도윤', books=[Book(title='테스트 전략')]),
    ])
    session.commit()

with Session(engine) as session:
    statement = select(Author).options(selectinload(Author.books)).order_by(Author.id)
    authors = session.scalars(statement).all()
    lines = [f'{author.name}: {[book.title for book in author.books]}' for author in authors]

print('\n'.join(lines))
```

```text
민지: ['SQL 입문', 'ORM 실습']
도윤: ['테스트 전략']
```

목록 API에서 이 패턴을 쓴다면 부모 수와 관계 수가 큰 실제 데이터로 쿼리 횟수와 반환 행 수를 확인합니다. 한 행만 읽는 상세 화면에서는 기본 지연 로딩이 더 단순할 수 있으므로, 모든 관계에 로딩 옵션을 일괄 적용하지 않습니다.

## joinedload와 unique로 한 번의 조인 결과 중복 제거하기

부모와 컬렉션 관계를 한 SQL 조인으로 가져와야 할 때 `joinedload()`를 사용합니다. 부모 한 명이 자식 여러 명을 가지면 SQL 결과에는 부모 행이 반복되므로, 컬렉션 `joinedload()` 결과에서는 `unique()`로 ORM 객체를 중복 없이 만들도록 명시합니다.

```python
from sqlalchemy import ForeignKey, String, create_engine, select
from sqlalchemy.orm import (
    DeclarativeBase,
    Mapped,
    Session,
    joinedload,
    mapped_column,
    relationship,
)


class Base(DeclarativeBase):
    pass


class Author(Base):
    __tablename__ = 'authors'

    id: Mapped[int] = mapped_column(primary_key=True)
    name: Mapped[str] = mapped_column(String(50))
    books: Mapped[list['Book']] = relationship(back_populates='author')


class Book(Base):
    __tablename__ = 'books'

    id: Mapped[int] = mapped_column(primary_key=True)
    title: Mapped[str] = mapped_column(String(100))
    author_id: Mapped[int] = mapped_column(ForeignKey('authors.id'))
    author: Mapped[Author] = relationship(back_populates='books')


engine = create_engine('sqlite://')
Base.metadata.create_all(engine)

with Session(engine) as session:
    session.add(Author(name='민지', books=[Book(title='SQL 입문'), Book(title='ORM 실습')]))
    session.commit()

with Session(engine) as session:
    statement = select(Author).options(joinedload(Author.books))
    authors = session.scalars(statement).unique().all()

print([(author.name, len(author.books)) for author in authors])
```

```text
[('민지', 2)]
```

`joinedload()`가 항상 더 빠른 것은 아닙니다. 부모당 자식 수가 많으면 조인 결과의 행 수와 전송량이 크게 늘 수 있으므로, 컬렉션 목록에는 대체로 `selectinload()`가 예측하기 쉽습니다. 정렬·필터링을 위해 실제로 자식 테이블을 조인해야 하는 요구와, 관계를 미리 읽는 로딩 전략도 구분합니다.

## AsyncSession으로 비동기 요청의 데이터베이스 I/O 기다리기

async 웹 서버처럼 하나의 요청이 데이터베이스 응답을 기다리는 동안 다른 코루틴을 실행해야 할 때 `AsyncSession`을 사용합니다. 비동기 드라이버가 필요하므로 SQLite 학습 예제는 `aiosqlite`를 함께 설치합니다.

```bash
python -m pip install 'SQLAlchemy[asyncio]' aiosqlite
```

```python
import asyncio

from sqlalchemy import String, select
from sqlalchemy.ext.asyncio import async_sessionmaker, create_async_engine
from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column


class Base(DeclarativeBase):
    pass


class Task(Base):
    __tablename__ = 'tasks'

    id: Mapped[int] = mapped_column(primary_key=True)
    title: Mapped[str] = mapped_column(String(100))


async def main() -> None:
    engine = create_async_engine('sqlite+aiosqlite://')

    async with engine.begin() as connection:
        await connection.run_sync(Base.metadata.create_all)

    session_factory = async_sessionmaker(engine, expire_on_commit=False)

    async with session_factory.begin() as session:
        session.add_all([Task(title='비동기 조회'), Task(title='비동기 저장')])

    async with session_factory() as session:
        tasks = (await session.scalars(select(Task).order_by(Task.id))).all()

    print([task.title for task in tasks])
    await engine.dispose()


asyncio.run(main())
```

```text
['비동기 조회', '비동기 저장']
```

`AsyncSession` 하나를 여러 동시 작업이 공유하면 안 됩니다. 요청 또는 독립된 코루틴마다 새 세션을 만들고, 데이터베이스 작업에는 `await`를 붙입니다. CPU 계산이 오래 걸리는 문제는 `async`로 바꾼다고 빨라지지 않으므로, 비동기 I/O가 필요한지 먼저 판단합니다.

Python 3.12와 SQLAlchemy 2.0의 관계 로딩·`unique()` 규칙·비동기 세션은 [Relationship Loading Techniques 공식 문서](https://docs.sqlalchemy.org/en/20/orm/queryguide/relationships.html)와 [AsyncIO 공식 문서](https://docs.sqlalchemy.org/en/20/orm/extensions/asyncio.html)를 기준으로 확인합니다.
