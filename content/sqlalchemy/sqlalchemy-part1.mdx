## create_engine으로 SQLite 데이터베이스 연결하기

파이썬 프로그램에서 데이터베이스와 연결을 재사용할 출발점이 필요할 때 `create_engine()`으로 `Engine`을 만듭니다. `Engine`은 실제 쿼리마다 새 연결을 직접 만드는 객체가 아니라, 연결과 연결 풀을 관리하는 진입점입니다. 이 노트에서는 별도 서버 없이 실행할 수 있는 SQLite 메모리 데이터베이스로 시작합니다.

```python
from sqlalchemy import create_engine, text


engine = create_engine('sqlite://')

with engine.connect() as connection:
    answer = connection.scalar(text('SELECT 6 * 7'))

print(answer)
```

```text
42
```

`sqlite://`는 실행이 끝나면 사라지는 메모리 데이터베이스이고, `sqlite:///tasks.db`는 현재 디렉터리에 파일을 만듭니다. `text()` 안에 값을 문자열로 이어 붙이지 말고, 이후 토픽에서처럼 SQLAlchemy 표현식이나 바인딩 파라미터로 값을 전달합니다. SQLAlchemy는 SQL 표현식 중심의 Core와 객체 중심의 ORM을 함께 제공하며, 이 섹션은 일상적인 애플리케이션 모델에 적합한 ORM을 중심으로 다룹니다.

## DeclarativeBase와 Mapped로 테이블 모델 선언하기

파이썬 객체를 테이블 행으로 다루고 싶을 때 `DeclarativeBase`를 상속한 베이스와 `Mapped` 필드로 모델을 선언합니다. 클래스 이름과 타입 힌트가 파이썬 모델의 모양을, `__tablename__`과 `mapped_column()`이 데이터베이스 스키마를 함께 설명합니다.

```python
from sqlalchemy import String
from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column


class Base(DeclarativeBase):
    pass


class Task(Base):
    __tablename__ = 'tasks'

    id: Mapped[int] = mapped_column(primary_key=True)
    title: Mapped[str] = mapped_column(String(100))
    done: Mapped[bool] = mapped_column(default=False)


print(Task.__tablename__)
print(list(Task.__table__.columns.keys()))
```

```text
tasks
['id', 'title', 'done']
```

`Mapped[int]`와 `Mapped[str]`는 모델 속성의 파이썬 타입을 명확히 하며, SQLAlchemy가 적절한 기본 SQL 타입과 null 허용 여부를 추론하는 데도 사용합니다. 기본 키는 행을 하나씩 식별하므로 모든 ORM 모델에 하나 이상 선언합니다. 길이, 인덱스, 외래 키처럼 기본 추론만으로 표현되지 않는 규칙은 `mapped_column()` 인수로 추가합니다.

## Base.metadata.create_all로 개발용 테이블 만들기

방금 선언한 모델로 로컬 학습용 테이블을 만들 때 `Base.metadata.create_all(engine)`을 호출합니다. 모델들이 등록한 메타데이터를 읽어 아직 없는 테이블의 `CREATE TABLE` 문을 실행합니다.

```python
from sqlalchemy import String, create_engine, inspect
from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column


class Base(DeclarativeBase):
    pass


class Task(Base):
    __tablename__ = 'tasks'

    id: Mapped[int] = mapped_column(primary_key=True)
    title: Mapped[str] = mapped_column(String(100))
    done: Mapped[bool] = mapped_column(default=False)


engine = create_engine('sqlite://')
Base.metadata.create_all(engine)

print(inspect(engine).get_table_names())
```

```text
['tasks']
```

`create_all()`은 빠르게 시작하는 학습·프로토타입에는 편리하지만, 이미 운영 중인 테이블의 변경 이력을 관리하지는 않습니다. 운영 스키마 변경은 팀의 마이그레이션 도구와 검토 절차로 별도 관리하고, 앱을 시작할 때 무조건 `create_all()`로 구조를 바꾸지 않습니다.

## Session.add와 commit으로 새 행 저장하기

새 `Task` 객체를 데이터베이스에 저장할 때는 `Session`에 객체를 등록한 뒤 `commit()`합니다. `Session`은 객체 변경을 모아 SQL로 반영하고, `commit()`은 그 변경을 하나의 트랜잭션으로 확정합니다.

```python
from sqlalchemy import String, create_engine, select
from sqlalchemy.orm import DeclarativeBase, Mapped, Session, mapped_column


class Base(DeclarativeBase):
    pass


class Task(Base):
    __tablename__ = 'tasks'

    id: Mapped[int] = mapped_column(primary_key=True)
    title: Mapped[str] = mapped_column(String(100))
    done: Mapped[bool] = mapped_column(default=False)


engine = create_engine('sqlite://')
Base.metadata.create_all(engine)

with Session(engine) as session:
    session.add_all([
        Task(title='세탁하기'),
        Task(title='공과금 납부하기'),
    ])
    session.commit()

with Session(engine) as session:
    titles = session.scalars(select(Task.title).order_by(Task.id)).all()

print(titles)
```

```text
['세탁하기', '공과금 납부하기']
```

`with Session(...)`은 세션을 사용한 뒤 닫아 연결 자원을 반환합니다. `add()`와 `add_all()`은 저장 대상을 세션에 등록할 뿐이고, `commit()` 전에는 다른 트랜잭션에 영구 반영되지 않습니다. 여러 저장 작업이 반드시 함께 성공해야 하는 경우는 중급의 `Session.begin()` 패턴으로 경계를 더 분명히 만듭니다.

## select와 Session.scalars로 모델 목록 읽기

테이블 행 전체를 ORM 객체 목록으로 읽을 때는 `select(모델)`을 만들고 `Session.scalars()`로 실행합니다. `scalars()`는 결과 행의 첫 값을 꺼내므로, `Task`를 선택했다면 반복할 때마다 `Task` 객체를 받습니다.

```python
from sqlalchemy import String, create_engine, select
from sqlalchemy.orm import DeclarativeBase, Mapped, Session, mapped_column


class Base(DeclarativeBase):
    pass


class Task(Base):
    __tablename__ = 'tasks'

    id: Mapped[int] = mapped_column(primary_key=True)
    title: Mapped[str] = mapped_column(String(100))
    done: Mapped[bool] = mapped_column(default=False)


engine = create_engine('sqlite://')
Base.metadata.create_all(engine)

with Session(engine) as session:
    session.add_all([Task(title='보고서 쓰기'), Task(title='회의 준비', done=True)])
    session.commit()

with Session(engine) as session:
    tasks = session.scalars(select(Task).order_by(Task.id)).all()

print([(task.title, task.done) for task in tasks])
```

```text
[('보고서 쓰기', False), ('회의 준비', True)]
```

`select(Task)`은 SQL 문자열을 즉시 실행하지 않고 조합 가능한 문(statement)을 만듭니다. SQLAlchemy 2.0 스타일에서는 과거의 `session.query(Task)`보다 `select()`와 `Session.execute()` 또는 `Session.scalars()` 조합을 기본으로 사용합니다.

## select.where와 order_by로 필요한 행만 정렬하기

완료되지 않은 할 일처럼 조건에 맞는 행만 읽고 일정한 순서로 보여 줄 때 `where()`와 `order_by()`를 연결합니다. 파이썬의 비교 연산으로 작성한 조건은 SQL의 `WHERE` 조건으로 변환되고 값은 안전하게 바인딩됩니다.

```python
from sqlalchemy import String, create_engine, select
from sqlalchemy.orm import DeclarativeBase, Mapped, Session, mapped_column


class Base(DeclarativeBase):
    pass


class Task(Base):
    __tablename__ = 'tasks'

    id: Mapped[int] = mapped_column(primary_key=True)
    title: Mapped[str] = mapped_column(String(100))
    done: Mapped[bool] = mapped_column(default=False)


engine = create_engine('sqlite://')
Base.metadata.create_all(engine)

with Session(engine) as session:
    session.add_all([
        Task(title='배포하기'),
        Task(title='문서 읽기', done=True),
        Task(title='테스트 작성하기'),
    ])
    session.commit()

with Session(engine) as session:
    statement = select(Task).where(Task.done.is_(False)).order_by(Task.title)
    titles = [task.title for task in session.scalars(statement)]

print(titles)
```

```text
['배포하기', '테스트 작성하기']
```

불리언 비교는 `Task.done.is_(False)`처럼 의도를 드러내면 `NULL`을 다루는 SQL의 규칙과도 구분하기 쉽습니다. 조건값이 사용자 입력이라도 파이썬 문자열 결합으로 SQL을 만들지 말고, `Task.title == input_title`처럼 표현식에 전달합니다.

## Session.get으로 기본 키 행 하나 찾기

이미 알고 있는 기본 키로 행 하나를 가져올 때는 `Session.get(모델, 기본키)`를 사용합니다. 행이 없을 수 있는 조회에 자연스럽게 맞고, 같은 세션에서 이미 읽은 객체라면 세션의 식별 맵을 먼저 확인합니다.

```python
from sqlalchemy import String, create_engine
from sqlalchemy.orm import DeclarativeBase, Mapped, Session, mapped_column


class Base(DeclarativeBase):
    pass


class Task(Base):
    __tablename__ = 'tasks'

    id: Mapped[int] = mapped_column(primary_key=True)
    title: Mapped[str] = mapped_column(String(100))
    done: Mapped[bool] = mapped_column(default=False)


engine = create_engine('sqlite://')
Base.metadata.create_all(engine)

with Session(engine) as session:
    session.add(Task(id=10, title='릴리스 노트 작성'))
    session.commit()

with Session(engine) as session:
    found = session.get(Task, 10)
    missing = session.get(Task, 99)

print(found.title if found is not None else None)
print(missing)
```

```text
릴리스 노트 작성
None
```

`get()`은 기본 키 조회 전용입니다. 제목·이메일처럼 다른 조건으로 한 행을 찾을 때는 `select(...).where(...)`를 사용하고, 결과가 반드시 하나여야 하는지 여부도 중급의 결과 메서드로 명시합니다.

Python 3.12와 SQLAlchemy 2.0의 선언형 모델·엔진·세션·`select()` 사용법은 [SQLAlchemy ORM Quick Start 공식 문서](https://docs.sqlalchemy.org/en/20/orm/quickstart.html)와 [ORM Querying Guide 공식 문서](https://docs.sqlalchemy.org/en/20/orm/queryguide/index.html)를 기준으로 확인합니다.
