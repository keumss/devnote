## 재고 예약의 데이터와 트랜잭션 계약 정하기

이번에는 `reserve_stock.py` 하나로 SQLite 재고를 초기화하고 상품을 예약하는 CLI를 만듭니다. 예약은 양수 수량만 허용하고, 재고 차감과 예약 기록은 반드시 함께 확정합니다. 아래 코드를 새 파일의 처음에 작성합니다.

```python
# reserve_stock.py
from __future__ import annotations

from datetime import datetime

from sqlalchemy import CheckConstraint, DateTime, ForeignKey, String, create_engine, func
from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column, relationship, sessionmaker


class Base(DeclarativeBase):
    pass


class Product(Base):
    __tablename__ = 'products'
    __table_args__ = (CheckConstraint('stock >= 0', name='ck_products_stock_nonnegative'),)

    id: Mapped[int] = mapped_column(primary_key=True)
    sku: Mapped[str] = mapped_column(String(30), unique=True)
    stock: Mapped[int]
    reservations: Mapped[list[Reservation]] = relationship(back_populates='product')


class Reservation(Base):
    __tablename__ = 'reservations'
    __table_args__ = (CheckConstraint('quantity > 0', name='ck_reservations_quantity_positive'),)

    id: Mapped[int] = mapped_column(primary_key=True)
    product_id: Mapped[int] = mapped_column(ForeignKey('products.id'))
    quantity: Mapped[int]
    created_at: Mapped[datetime] = mapped_column(DateTime(timezone=True), server_default=func.now())
    product: Mapped[Product] = relationship(back_populates='reservations')


engine = create_engine('sqlite:///inventory.db')
SessionLocal = sessionmaker(engine)
```

`UNIQUE`, `CHECK`, 외래 키는 애플리케이션 코드 밖의 입력 경로도 지키는 마지막 경계입니다. `Reservation`을 만들기 전에 재고를 확인하더라도, 데이터베이스 제약을 제거하지 않습니다. 다음 토픽에서 모델의 제약만으로 해결되지 않는 재고 차감 경쟁도 트랜잭션 안에서 처리합니다.

## create_all과 초기 상품으로 실행 가능한 데이터 준비하기

파일을 처음 실행할 때 테이블을 만들고, 학습용 상품이 아직 없을 때만 넣는 `initialize()` 함수를 앞 코드 아래에 추가합니다. 초기화 명령을 여러 번 실행해도 같은 상품을 중복으로 만들지 않는 것이 목표입니다.

```python
from sqlalchemy import select


def initialize() -> None:
    Base.metadata.create_all(engine)

    with SessionLocal.begin() as session:
        has_product = session.scalar(select(Product.id).limit(1)) is not None
        if not has_product:
            session.add_all([
                Product(sku='KEYBOARD-1', stock=10),
                Product(sku='MOUSE-1', stock=5),
            ])
```

`SessionLocal.begin()`은 블록이 정상 종료될 때 초기 상품을 커밋하고, 예외가 나면 롤백합니다. 실서비스의 초기 데이터는 배포·마이그레이션 절차에 둘 수 있지만, 여기서는 CLI가 즉시 재현 가능하도록 파일 안에 최소 데이터만 둡니다.

## update.returning으로 재고 차감과 예약을 원자적으로 처리하기

동시에 두 예약이 같은 재고를 보고 모두 성공하면 안 될 때, 먼저 상품을 읽고 파이썬에서 수량을 빼는 대신 조건이 들어간 `UPDATE`를 실행합니다. `stock >= quantity`를 만족할 때만 재고를 줄이고, 같은 트랜잭션 안에 예약 기록도 추가합니다. 다음 함수를 앞 코드 아래에 추가합니다.

```python
from sqlalchemy import update


def reserve_stock(sku: str, quantity: int) -> tuple[int, int]:
    if quantity <= 0:
        raise ValueError('예약 수량은 1 이상이어야 합니다.')

    with SessionLocal.begin() as session:
        row = session.execute(
            update(Product)
            .where(Product.sku == sku, Product.stock >= quantity)
            .values(stock=Product.stock - quantity)
            .returning(Product.id, Product.stock)
        ).one_or_none()

        if row is None:
            raise ValueError('재고가 부족하거나 SKU가 없습니다.')

        product_id, remaining_stock = row
        reservation = Reservation(product_id=product_id, quantity=quantity)
        session.add(reservation)
        session.flush()

        return reservation.id, remaining_stock
```

`returning()`으로 수정된 상품 ID와 남은 재고를 바로 받으므로 별도 `SELECT`가 필요 없습니다. `session.flush()`는 아직 커밋하지 않고 예약 ID를 데이터베이스에서 받아오게 합니다. 예약 기록 삽입이 실패하면 예외로 블록을 나가며, 앞선 재고 차감도 같은 트랜잭션에서 롤백됩니다.

## select로 현재 재고를 확인하는 읽기 함수 만들기

예약 뒤 사용자가 확인할 수 있도록 SKU와 남은 수량을 정렬해 읽는 함수를 추가합니다. 읽기 전용 세션도 컨텍스트로 닫아 연결을 반환하고, 데이터베이스 행을 CLI 표현과 분리한 튜플 목록으로 돌려줍니다.

```python
def list_stock() -> list[tuple[str, int]]:
    with SessionLocal() as session:
        rows = session.execute(
            select(Product.sku, Product.stock).order_by(Product.sku)
        ).all()

    return [(sku, stock) for sku, stock in rows]
```

조회 함수는 트랜잭션을 확정하지 않으므로 `commit()`을 호출하지 않습니다. 화면·CLI가 커지면 `Product` 모델 자체를 여기저기 넘기기보다, 이처럼 필요한 공개 데이터만 반환하는 함수나 응답 모델을 두면 저장 구조와 출력 계약을 분리하기 좋습니다.

## argparse로 init, reserve, stock 명령 연결하기

이제 파일 맨 아래에 명령줄 입력과 종료 상태를 연결합니다. 잘못된 수량·없는 SKU·재고 부족은 예상한 업무 오류이므로 짧은 메시지를 표준 오류로 보내고 상태 코드 1을 반환합니다.

```python
import argparse
import sys


def main() -> int:
    parser = argparse.ArgumentParser(description='SQLite 재고 예약 도구')
    commands = parser.add_subparsers(dest='command', required=True)
    commands.add_parser('init')

    reserve_parser = commands.add_parser('reserve')
    reserve_parser.add_argument('sku')
    reserve_parser.add_argument('quantity', type=int)

    commands.add_parser('stock')
    args = parser.parse_args()

    if args.command == 'init':
        initialize()
        print('inventory initialized')
        return 0

    try:
        if args.command == 'reserve':
            reservation_id, remaining_stock = reserve_stock(args.sku, args.quantity)
            print(f'reservation_id={reservation_id} remaining_stock={remaining_stock}')
            return 0

        for sku, stock in list_stock():
            print(f'{sku}: {stock}')
        return 0
    except ValueError as error:
        print(f'reserve-stock: {error}', file=sys.stderr)
        return 1


if __name__ == '__main__':
    raise SystemExit(main())
```

`argparse`가 수량을 `int`로 바꾸고, 도메인 함수가 1 이상인지 다시 검사합니다. 명령줄 파싱·출력은 `main()`에 두고, `reserve_stock()`은 검증된 파이썬 값과 트랜잭션만 다루게 분리하면 HTTP API나 작업 큐에서도 같은 예약 규칙을 재사용할 수 있습니다.

## 실행 결과로 예약 성공과 재고 부족 확인하기

앞 토픽의 코드를 순서대로 합친 뒤, 새 디렉터리에서 초기화·예약·재고 조회를 실행합니다. 초기 재고가 10인 키보드 두 개를 예약하면 남은 재고는 8이고, 99개 예약은 아무 행도 수정하지 않은 채 실패합니다.

```bash
python reserve_stock.py init
python reserve_stock.py reserve KEYBOARD-1 2
python reserve_stock.py stock
python reserve_stock.py reserve KEYBOARD-1 99; echo $?
```

```text
inventory initialized
reservation_id=1 remaining_stock=8
KEYBOARD-1: 8
MOUSE-1: 5
reserve-stock: 재고가 부족하거나 SKU가 없습니다.
1
```

`inventory.db`를 지우고 다시 `init`을 실행하면 이 학습 시나리오를 처음부터 반복할 수 있습니다. 실제 서비스에서는 재고 부족, 중복 요청, 상품 비활성화 같은 실패 조건을 각각 테스트하고, 사용하는 데이터베이스의 격리 수준·잠금·재시도 정책도 업무 요구에 맞춰 검토합니다.

Python 3.12와 SQLAlchemy 2.0의 조건부 `UPDATE`, `RETURNING`, 세션 트랜잭션은 [ORM-enabled INSERT, UPDATE, and DELETE 공식 문서](https://docs.sqlalchemy.org/en/20/orm/queryguide/dml.html)와 [Session Basics 공식 문서](https://docs.sqlalchemy.org/en/20/orm/session_basics.html)를 기준으로 확인합니다.
