## 관계 속성으로 INNER JOIN과 LEFT JOIN 만들기

매핑된 관계를 조인 경로로 사용하면 외래 키 조건을 반복하지 않아도 됩니다. INNER JOIN은 게시물이 있는 사용자만 반환하고, LEFT JOIN은 게시물이 없는 Charlie도 보존합니다.

```python
from sqlalchemy import select

from setup import Post, SessionLocal, User, reset_database


reset_database()

with SessionLocal() as session:
    stmt = (
        select(User.name, Post.title)
        .join(User.posts)
        .order_by(User.id, Post.id)
    )
    for name, title in session.execute(stmt):
        print(name, '-', title)

    empty = session.execute(
        select(User.name, Post.title)
        .outerjoin(User.posts)
        .where(User.name == 'Charlie')
    ).one()
    print(empty)
```

```text
Alice - SQLAlchemy 시작
Alice - 관계 매핑
Bob - 쿼리 패턴
Bob - 로딩 성능
('Charlie', None)
```

조인으로 반환 행을 결정하는 것과 `selectinload()`·`joinedload()`로 관계 객체를 미리 읽는 것은 목적이 다릅니다.

## GROUP BY와 COUNT로 관계 개수 집계하기

게시물 수만 필요하다면 모든 Post 객체를 로드하지 말고 DB에서 집계합니다. 게시물이 없는 사용자도 포함하려면 LEFT JOIN과 `count(Post.id)`를 함께 사용합니다.

```python
from sqlalchemy import func, select

from setup import Post, SessionLocal, User


stmt = (
    select(User.name, func.count(Post.id))
    .outerjoin(User.posts)
    .group_by(User.id, User.name)
    .order_by(User.id)
)

with SessionLocal() as session:
    for name, count in session.execute(stmt):
        print(name, count)
```

```text
Alice 2
Bob 2
Charlie 0
```

`count(*)`는 LEFT JOIN이 만든 사용자 행도 세므로 게시물이 없어도 1이 될 수 있습니다. 오른쪽 테이블의 기본 키를 세면 실제 관계 수가 됩니다.

## 고유한 정렬 키로 커서 페이지네이션하기

OFFSET은 임의 페이지 이동에 편리하지만 값이 커질수록 건너뛰는 비용이 늘고, 조회 중 데이터가 추가되면 중복이나 누락이 생길 수 있습니다. 순방향 목록은 마지막으로 본 고유 키를 다음 조건에 사용하는 방식이 단순합니다.

```python
from sqlalchemy import select

from setup import Post, SessionLocal


with SessionLocal() as session:
    first_page = session.scalars(
        select(Post).order_by(Post.id).limit(2)
    ).all()
    cursor = first_page[-1].id

    next_page = session.scalars(
        select(Post)
        .where(Post.id > cursor)
        .order_by(Post.id)
        .limit(2)
    ).all()

    print([post.id for post in first_page])
    print([post.id for post in next_page])
```

```text
[1, 2]
[3, 4]
```

날짜처럼 중복 가능한 컬럼으로 정렬한다면 `(created_at, id)`처럼 고유한 보조 키를 커서에 포함합니다.

## scalar_subquery로 집계 결과를 조건에 사용하기

한 값만 반환하는 SELECT는 `scalar_subquery()`로 다른 WHERE 조건 안에 넣을 수 있습니다. 다음 쿼리는 전체 평균 나이보다 나이가 많은 사용자를 찾습니다.

```python
from sqlalchemy import func, select

from setup import SessionLocal, User


average_age = select(func.avg(User.age)).scalar_subquery()
stmt = (
    select(User.name, User.age)
    .where(User.age > average_age)
    .order_by(User.id)
)

with SessionLocal() as session:
    print(session.execute(stmt).all())
```

```text
[('Charlie', 35)]
```

중간 결과를 여러 곳에서 재사용하거나 복잡한 단계에 이름을 붙여야 할 때는 `.cte()`로 CTE를 만들 수 있습니다. CTE나 서브쿼리가 항상 더 빠른 것은 아니므로 실제 DB의 실행 계획으로 확인합니다.
