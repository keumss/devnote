## SQLAlchemy의 Core와 ORM 이해하기

SQLAlchemy 2.0은 SQL 표현식을 만드는 Core와 테이블을 Python 클래스로 매핑하는 ORM을 제공합니다. ORM도 내부에서는 SQL을 실행하므로 생성되는 쿼리와 트랜잭션 경계를 이해해야 예상하지 못한 쓰기와 성능 문제를 찾을 수 있습니다.

```sh
python -m pip install 'SQLAlchemy>=2.0,<2.1'
```

이 섹션은 Python 3.12와 SQLAlchemy 2.0 스타일을 기준으로 합니다. 설치 로그와 패치 버전은 실행 시점에 따라 달라질 수 있습니다.

## 공통 실습 모델과 Engine 준비하기

네 노트에서 사용할 공통 코드를 `setup.py`에 저장합니다. Engine은 연결과 SQL 실행 기반을 공유하고, Session은 한 작업 단위에서만 만들고 닫습니다.

```python
from __future__ import annotations

from sqlalchemy import ForeignKey, String, create_engine
from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column, relationship, sessionmaker


class Base(DeclarativeBase):
    pass


class User(Base):
    __tablename__ = 'users'

    id: Mapped[int] = mapped_column(primary_key=True)
    name: Mapped[str] = mapped_column(String(30), unique=True)
    age: Mapped[int | None]
    posts: Mapped[list[Post]] = relationship(
        back_populates='author',
        cascade='all, delete-orphan',
    )


class Post(Base):
    __tablename__ = 'posts'

    id: Mapped[int] = mapped_column(primary_key=True)
    title: Mapped[str] = mapped_column(String(100))
    user_id: Mapped[int] = mapped_column(ForeignKey('users.id'))
    author: Mapped[User] = relationship(back_populates='posts')


engine = create_engine('sqlite+pysqlite:///sqlalchemy_lab.db', echo=False)
SessionLocal = sessionmaker(engine)


def reset_database() -> None:
    Base.metadata.drop_all(engine)
    Base.metadata.create_all(engine)
    with SessionLocal.begin() as session:
        session.add_all([
            User(name='Alice', age=25, posts=[
                Post(title='SQLAlchemy 시작'),
                Post(title='관계 매핑'),
            ]),
            User(name='Bob', age=30, posts=[
                Post(title='쿼리 패턴'),
                Post(title='로딩 성능'),
            ]),
            User(name='Charlie', age=35),
        ])
```

`reset_database()`는 예상 결과를 동일하게 만들기 위한 학습 전용 함수입니다. `drop_all()`은 데이터를 모두 지우므로 운영에서는 사용하지 않고, 스키마 변경은 Alembic으로 관리합니다.

## Session.begin으로 객체 생성하기

`add()`는 객체를 Session에 등록하고 `flush()`는 INSERT를 보내 생성된 기본 키를 객체에 반영합니다. `SessionLocal.begin()`은 블록이 성공하면 commit하고 예외가 발생하면 rollback합니다.

```python
from setup import SessionLocal, User, reset_database


reset_database()

with SessionLocal.begin() as session:
    dana = User(name='Dana', age=28)
    session.add(dana)
    session.flush()
    print(dana.id, dana.name)
```

```text
4 Dana
```

flush는 SQL을 전송하지만 트랜잭션을 확정하지 않습니다. 블록 안에서 후속 작업이 실패하면 INSERT도 함께 취소됩니다.

## select와 scalars로 객체 조회하기

`select(User)`는 User 객체를 반환하는 문장을 만들고, `scalars()`는 결과 행에서 ORM 객체 열만 꺼냅니다. 결과 순서를 예측해야 하는 조회에는 `order_by()`를 지정합니다.

```python
from sqlalchemy import select

from setup import SessionLocal, User


with SessionLocal() as session:
    stmt = (
        select(User)
        .where(User.age >= 30)
        .order_by(User.age.desc(), User.id)
    )
    for user in session.scalars(stmt):
        print(user.name, user.age)
```

```text
Charlie 35
Bob 30
```

## get과 one_or_none으로 조회 결과의 의미 표현하기

기본 키 조회에는 Session의 identity map을 먼저 확인하는 `get()`을 사용합니다. 결과가 없을 수 있지만 둘 이상이면 데이터 전제 오류인 조회에는 `one_or_none()`이 적합합니다.

```python
from sqlalchemy import select

from setup import SessionLocal, User


with SessionLocal() as session:
    alice = session.get(User, 1)
    unknown = session.scalars(
        select(User).where(User.name == 'Unknown')
    ).one_or_none()

    print(alice.name)
    print(unknown)
```

```text
Alice
None
```

`first()`는 여러 결과 중 첫 행만 선택하므로 “정확히 하나”라는 조건을 검증하지 않습니다. 조회 의도에 맞는 결과 메서드를 선택하면 누락과 중복을 더 일찍 발견할 수 있습니다.
