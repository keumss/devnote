## SQLAlchemy란?

SQLAlchemy는 SQL을 구성하는 Core와 테이블을 Python 클래스로 매핑하는 ORM을 제공하는 데이터 접근 도구입니다. ORM은 SQL을 없애는 기술이 아니라 객체 변경을 INSERT·UPDATE로 변환하고 결과 행을 객체로 조립하는 계층입니다. 생성되는 SQL과 트랜잭션 경계를 이해해야 성능 문제와 예상하지 못한 쓰기를 찾을 수 있습니다.

## 엔진 및 세션 (Engine & Session)

데이터베이스 통신을 담당하는 Engine과 쿼리를 실행하는 Session을 설정합니다.

```python
import os

from sqlalchemy import create_engine
from sqlalchemy.orm import Session

database_url = os.environ.get('DATABASE_URL', 'sqlite:///app.db')
engine = create_engine(database_url, echo=False)

# 세션 컨텍스트 매니저를 통해 안전하게 연결
with Session(engine) as session:
    # 데이터베이스 작업 수행
    pass
```

로컬 기본값은 바로 실행 가능한 SQLite 파일이고, 운영에서는 `DATABASE_URL`로 PostgreSQL 같은 실제 연결을 주입합니다. 비밀번호가 든 URL을 코드에 커밋하지 않습니다. `echo=True`는 학습 중 SQL을 확인하는 데 유용하지만 바인딩 값에 민감 정보가 포함될 수 있어 운영 로그에서는 주의합니다.

## 테이블 모델 선언 (Declarative Base)

타입 힌트(Mapped)를 활용하여 파이썬 클래스를 데이터베이스 테이블에 매핑합니다.

```python
from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column
from sqlalchemy import String

class Base(DeclarativeBase):
    pass

class User(Base):
    __tablename__ = 'users'

    id: Mapped[int] = mapped_column(primary_key=True)
    name: Mapped[str] = mapped_column(String(30), nullable=False)
    age: Mapped[int | None] = mapped_column(nullable=True)

# DB에 테이블 생성 (개발 환경에서 주로 사용)
Base.metadata.create_all(engine)
```

`create_all()`은 아직 없는 테이블을 만들지만 기존 컬럼의 이름·타입을 변경하지 않습니다. 학습과 짧은 테스트에는 편리하고, 운영 스키마 변경에는 Alembic 리비전을 사용합니다.

## 데이터 생성 (Insert & Bulk Insert)

단일 객체를 추가하거나, add_all()을 사용하여 여러 개의 객체를 한 번에 삽입합니다.

```python
with Session(engine) as session:
    # 단일 객체 추가
    user1 = User(name='Alice', age=25)
    session.add(user1)

    # 여러 객체 일괄 추가
    users = [
        User(name='Bob', age=30),
        User(name='Charlie', age=35),
    ]
    session.add_all(users)

    session.commit()  # 트랜잭션 확정
    session.refresh(user1)
    print(user1.id, user1.name)
```

새 DB에서 처음 실행했다면 DB가 만든 기본 키가 객체에 반영됩니다.

```text
1 Alice
```

`add_all()`은 한 트랜잭션에 객체를 추가하지만 반드시 하나의 SQL 문으로 합쳐진다는 뜻은 아닙니다. 중간에 하나라도 실패하면 `rollback()`하고 실패 원인을 처리한 뒤 새 트랜잭션을 시작합니다.

## 기본 데이터 조회 (Select, Where, Order By)

select() 구문에 조건을 추가하고, 특정 기준에 따라 정렬된 결과를 가져옵니다.

```python
from sqlalchemy import select

with Session(engine) as session:
    # 20살 이상인 유저를 나이 내림차순으로 정렬
    stmt = select(User).where(User.age >= 20).order_by(User.age.desc())

    # scalars()를 통해 ORM 객체 스트림 획득 후 리스트 변환
    users = session.scalars(stmt).all()

    for u in users:
        print(u.name, u.age)
```

```text
Charlie 35
Bob 30
Alice 25
```

`scalars()`는 `select(User)` 결과에서 ORM 객체 열만 꺼냅니다. 결과 전체가 필요하지 않다면 `.all()`로 모두 메모리에 올리지 말고 페이지네이션이나 스트리밍을 검토합니다.
