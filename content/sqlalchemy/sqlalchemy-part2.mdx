## select.where와 in_로 여러 검색 조건 조합하기

목록 화면의 완료 상태·제목 검색·선택한 ID처럼 선택적인 필터를 하나의 조회에 조합할 때 `where()`를 이어 붙입니다. 각 조건은 SQL의 `AND`로 연결되며, 목록 값은 `in_()`에 전달해 `IN` 조건으로 만듭니다.

```python
from sqlalchemy import String, create_engine, select
from sqlalchemy.orm import DeclarativeBase, Mapped, Session, mapped_column


class Base(DeclarativeBase):
    pass


class Task(Base):
    __tablename__ = 'tasks'

    id: Mapped[int] = mapped_column(primary_key=True)
    title: Mapped[str] = mapped_column(String(100))
    done: Mapped[bool] = mapped_column(default=False)


engine = create_engine('sqlite://')
Base.metadata.create_all(engine)

with Session(engine) as session:
    session.add_all([
        Task(id=1, title='배포 점검'),
        Task(id=2, title='회의록 정리', done=True),
        Task(id=3, title='배포 공지'),
    ])
    session.commit()

with Session(engine) as session:
    statement = (
        select(Task)
        .where(Task.done.is_(False))
        .where(Task.id.in_([1, 3]))
        .order_by(Task.id.desc())
    )
    titles = [task.title for task in session.scalars(statement)]

print(titles)
```

```text
['배포 공지', '배포 점검']
```

필터가 선택 사항이면 값이 있을 때만 `statement = statement.where(...)`로 조건을 추가합니다. 빈 목록에 `in_([])`을 넘겼을 때의 의미는 제품 요구에 따라 다르므로, 모든 결과를 뜻하는지 결과 없음을 뜻하는지 호출 전에 명확히 정합니다.

## ScalarResult.one_or_none으로 0개 또는 1개 결과 강제하기

이메일처럼 고유해야 하는 값으로 사용자를 찾을 때, 결과가 없을 수는 있지만 두 개 이상이면 데이터 문제인 경우 `one_or_none()`을 사용합니다. 반환 타입은 ORM 객체 또는 `None`이고, 여러 행이면 조용히 첫 행을 고르지 않고 예외를 냅니다.

```python
from sqlalchemy import String, create_engine, select
from sqlalchemy.orm import DeclarativeBase, Mapped, Session, mapped_column


class Base(DeclarativeBase):
    pass


class Member(Base):
    __tablename__ = 'members'

    id: Mapped[int] = mapped_column(primary_key=True)
    email: Mapped[str] = mapped_column(String(200), unique=True)


engine = create_engine('sqlite://')
Base.metadata.create_all(engine)

with Session(engine) as session:
    session.add(Member(email='minji@example.com'))
    session.commit()

with Session(engine) as session:
    statement = select(Member).where(Member.email == 'nobody@example.com')
    member = session.scalars(statement).one_or_none()

print(member)
```

```text
None
```

반드시 정확히 한 행이어야 한다면 `one()`, 첫 행만 필요하고 여러 행을 허용한다면 `first()`를 고릅니다. 애플리케이션의 기대와 데이터베이스의 `UNIQUE` 제약을 함께 두어, 조회 코드와 저장 경계가 같은 규칙을 지키게 합니다.

## ForeignKey와 relationship으로 주문-고객 관계 연결하기

한 고객이 여러 주문을 갖는 것처럼 객체 사이의 관계를 탐색하고 싶을 때 외래 키와 양방향 `relationship()`을 함께 선언합니다. 외래 키는 데이터베이스의 연결 규칙이고, `relationship()`은 ORM 객체에서 `customer.orders`와 `order.customer`로 이동하는 경로입니다.

```python
from sqlalchemy import ForeignKey, String, create_engine, select
from sqlalchemy.orm import DeclarativeBase, Mapped, Session, mapped_column, relationship


class Base(DeclarativeBase):
    pass


class Customer(Base):
    __tablename__ = 'customers'

    id: Mapped[int] = mapped_column(primary_key=True)
    name: Mapped[str] = mapped_column(String(50))
    orders: Mapped[list['Order']] = relationship(back_populates='customer')


class Order(Base):
    __tablename__ = 'orders'

    id: Mapped[int] = mapped_column(primary_key=True)
    number: Mapped[str] = mapped_column(String(30))
    customer_id: Mapped[int] = mapped_column(ForeignKey('customers.id'))
    customer: Mapped[Customer] = relationship(back_populates='orders')


engine = create_engine('sqlite://')
Base.metadata.create_all(engine)

with Session(engine) as session:
    session.add(Customer(name='민지', orders=[Order(number='ORD-100'), Order(number='ORD-101')]))
    session.commit()

with Session(engine) as session:
    customer = session.scalars(select(Customer).where(Customer.name == '민지')).one()
    print([order.number for order in customer.orders])
```

```text
['ORD-100', 'ORD-101']
```

`back_populates`는 두 속성이 같은 관계의 양쪽임을 명시합니다. 이 예제처럼 컬렉션을 처음 접근할 때 추가 조회가 일어날 수 있으며, 많은 부모 행의 관계를 목록에서 읽는 상황은 고급 노트의 로딩 전략으로 제어합니다.

## select.join으로 관계 조건에 맞는 행 조회하기

고객 이름으로 주문을 찾는 것처럼 한 테이블의 조건으로 다른 테이블 행을 고를 때 `select().join()`을 사용합니다. 모델 관계를 통해 조인하면 외래 키 조건을 문자열로 다시 쓰지 않아도 됩니다.

```python
from sqlalchemy import ForeignKey, String, create_engine, select
from sqlalchemy.orm import DeclarativeBase, Mapped, Session, mapped_column, relationship


class Base(DeclarativeBase):
    pass


class Customer(Base):
    __tablename__ = 'customers'

    id: Mapped[int] = mapped_column(primary_key=True)
    name: Mapped[str] = mapped_column(String(50))
    orders: Mapped[list['Order']] = relationship(back_populates='customer')


class Order(Base):
    __tablename__ = 'orders'

    id: Mapped[int] = mapped_column(primary_key=True)
    number: Mapped[str] = mapped_column(String(30))
    customer_id: Mapped[int] = mapped_column(ForeignKey('customers.id'))
    customer: Mapped[Customer] = relationship(back_populates='orders')


engine = create_engine('sqlite://')
Base.metadata.create_all(engine)

with Session(engine) as session:
    session.add_all([
        Customer(name='민지', orders=[Order(number='ORD-100')]),
        Customer(name='도윤', orders=[Order(number='ORD-200')]),
    ])
    session.commit()

with Session(engine) as session:
    statement = select(Order.number).join(Order.customer).where(Customer.name == '민지')
    numbers = session.scalars(statement).all()

print(numbers)
```

```text
['ORD-100']
```

`join(Order.customer)`는 `Order.customer_id`와 `Customer.id`의 관계를 사용합니다. 관계가 여럿이거나 조인 대상이 모호한 복잡한 조회는 별칭과 명시적인 조인 조건을 사용해 의도를 드러내고, 생성된 SQL을 테스트·로그에서 확인합니다.

## 모델 속성 변경으로 UPDATE 반영하기

세션에서 읽은 ORM 객체의 일반 속성을 수정한 뒤 저장할 때 별도 `UPDATE` 문을 직접 만들 필요가 없습니다. `Session`은 연결된 객체의 변경을 추적하고 `commit()` 시점에 필요한 `UPDATE`를 보냅니다.

```python
from sqlalchemy import String, create_engine
from sqlalchemy.orm import DeclarativeBase, Mapped, Session, mapped_column


class Base(DeclarativeBase):
    pass


class Task(Base):
    __tablename__ = 'tasks'

    id: Mapped[int] = mapped_column(primary_key=True)
    title: Mapped[str] = mapped_column(String(100))
    done: Mapped[bool] = mapped_column(default=False)


engine = create_engine('sqlite://')
Base.metadata.create_all(engine)

with Session(engine) as session:
    session.add(Task(id=1, title='장애 공지 작성'))
    session.commit()

with Session(engine) as session:
    task = session.get(Task, 1)
    assert task is not None
    task.done = True
    session.commit()

with Session(engine) as session:
    updated = session.get(Task, 1)

print(updated.done if updated is not None else None)
```

```text
True
```

이 방식은 한 행을 읽어 업무 규칙을 적용한 뒤 변경할 때 읽기 쉽습니다. 많은 행을 같은 조건으로 바꾸는 배치 작업은 고급 SQLAlchemy DML 문을 검토하되, 세션에 이미 로드된 객체와 상태를 어떻게 동기화할지 함께 결정합니다.

## Session.delete로 삭제할 객체 표시하기

한 행을 확인한 뒤 삭제할 때 `Session.delete()`에 ORM 객체를 전달합니다. 삭제는 호출 순간이 아니라 flush 또는 commit 시점에 SQL로 실행되므로, 같은 트랜잭션 안의 다른 변경과 함께 확정하거나 취소할 수 있습니다.

```python
from sqlalchemy import String, create_engine, select
from sqlalchemy.orm import DeclarativeBase, Mapped, Session, mapped_column


class Base(DeclarativeBase):
    pass


class Task(Base):
    __tablename__ = 'tasks'

    id: Mapped[int] = mapped_column(primary_key=True)
    title: Mapped[str] = mapped_column(String(100))


engine = create_engine('sqlite://')
Base.metadata.create_all(engine)

with Session(engine) as session:
    session.add_all([Task(id=1, title='보관'), Task(id=2, title='삭제')])
    session.commit()

with Session(engine) as session:
    task = session.get(Task, 2)
    assert task is not None
    session.delete(task)
    session.commit()

with Session(engine) as session:
    titles = session.scalars(select(Task.title).order_by(Task.id)).all()

print(titles)
```

```text
['보관']
```

관계가 있는 모델을 삭제하면 외래 키 제약과 ORM cascade 설정이 영향을 줍니다. 연결된 주문까지 함께 지울지, 삭제를 막을지, 소유자만 비울지는 업무 규칙이므로 모델 관계와 데이터베이스 제약을 같은 정책으로 설계합니다.

## Session.begin으로 함께 성공해야 하는 저장 묶기

두 행을 함께 저장해야 할 때 `Session.begin()` 컨텍스트 안에 작업을 둡니다. 블록이 정상 종료되면 커밋하고, 무결성 오류처럼 예외가 나오면 롤백하므로 부분 저장을 남기지 않습니다.

```python
from sqlalchemy import String, create_engine, select
from sqlalchemy.exc import IntegrityError
from sqlalchemy.orm import DeclarativeBase, Mapped, Session, mapped_column


class Base(DeclarativeBase):
    pass


class Task(Base):
    __tablename__ = 'tasks'

    id: Mapped[int] = mapped_column(primary_key=True)
    title: Mapped[str] = mapped_column(String(100), unique=True)


engine = create_engine('sqlite://')
Base.metadata.create_all(engine)

try:
    with Session(engine) as session:
        with session.begin():
            session.add_all([Task(title='중복 제목'), Task(title='중복 제목')])
except IntegrityError:
    pass

with Session(engine) as session:
    titles = session.scalars(select(Task.title)).all()

print(titles)
```

```text
[]
```

예외를 잡은 뒤에는 실패한 세션을 계속 쓰지 말고 이 예제처럼 컨텍스트를 벗어나 새 작업 단위를 만듭니다. 트랜잭션은 보통 요청 하나 또는 명령 하나처럼 짧게 잡고, 사용자 입력·네트워크 호출·긴 계산을 열린 트랜잭션 안에 오래 두지 않습니다.

Python 3.12와 SQLAlchemy 2.0의 관계 매핑·조회 결과·변경 추적·트랜잭션은 [Basic Relationship Patterns 공식 문서](https://docs.sqlalchemy.org/en/20/orm/basic_relationships.html), [ORM Querying Guide 공식 문서](https://docs.sqlalchemy.org/en/20/orm/queryguide/index.html), [Session Basics 공식 문서](https://docs.sqlalchemy.org/en/20/orm/session_basics.html)를 기준으로 확인합니다.
