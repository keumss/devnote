이 노트의 예제는 Part 1의 `setup.py`를 사용하며 위에서 아래로 실행합니다.

## 외래 키와 relationship 함께 사용하기

외래 키는 DB의 참조 무결성을 지키고, `relationship()`은 `user.posts`와 `post.author`로 객체 사이를 탐색하게 합니다. `selectinload()`를 사용하면 필요한 관계를 별도 SELECT로 미리 읽을 수 있습니다.

```python
from sqlalchemy import select
from sqlalchemy.orm import selectinload

from setup import SessionLocal, User, reset_database


reset_database()

with SessionLocal() as session:
    alice = session.scalars(
        select(User)
        .where(User.name == 'Alice')
        .options(selectinload(User.posts))
    ).one()

    for post in alice.posts:
        print(post.title, '-', post.author.name)
```

```text
SQLAlchemy 시작 - Alice
관계 매핑 - Alice
```

`back_populates`는 한쪽 관계가 바뀔 때 반대쪽 객체 속성도 동기화합니다. 여러 사용자의 관계를 미리 로드하지 않고 반복 접근하면 Part 4에서 다룰 N+1 쿼리가 생길 수 있습니다.

## Unit of Work로 한 객체 수정하기

Session에서 조회한 객체의 속성을 바꾸면 Unit of Work가 변경을 추적합니다. 다시 `add()`하지 않아도 commit 직전 flush에서 UPDATE가 실행됩니다.

```python
from sqlalchemy import select

from setup import SessionLocal, User


with SessionLocal.begin() as session:
    alice = session.scalars(
        select(User).where(User.name == 'Alice')
    ).one()
    alice.age = 26

with SessionLocal() as session:
    print(session.get(User, 1).age)
```

```text
26
```

DB 트리거나 ORM 밖의 SQL로 바뀐 값을 즉시 다시 읽어야 할 때는 `session.refresh(object)`를 사용합니다. flush가 실패한 Session을 계속 사용하려면 먼저 rollback해야 합니다.

## delete-orphan으로 관계에서 제거된 객체 삭제하기

Part 1의 `User.posts`에는 `cascade='all, delete-orphan'`이 있습니다. 부모 컬렉션에서 제거되어 소유자가 없어진 Post는 flush 때 삭제됩니다.

```python
from sqlalchemy import select
from sqlalchemy.orm import selectinload

from setup import Post, SessionLocal, User


with SessionLocal.begin() as session:
    alice = session.scalars(
        select(User)
        .where(User.name == 'Alice')
        .options(selectinload(User.posts))
    ).one()
    removed_post = alice.posts.pop()
    removed_id = removed_post.id

with SessionLocal() as session:
    print(session.get(Post, removed_id) is None)
```

```text
True
```

삭제 cascade는 데이터 수명 정책입니다. DB의 `ON DELETE CASCADE`도 사용한다면 ORM의 `cascade`와 `passive_deletes`가 같은 정책을 표현하도록 함께 설정합니다.

## bulk UPDATE로 여러 행 수정하기

여러 객체를 읽지 않고 같은 규칙으로 바꾸려면 UPDATE 문을 직접 실행할 수 있습니다. 개별 객체 이벤트와 Python 수준 검증을 거치지 않으므로 단순한 변경에 사용합니다.

```python
from sqlalchemy import select, update

from setup import SessionLocal, User


with SessionLocal.begin() as session:
    result = session.execute(
        update(User)
        .where(User.age >= 30)
        .values(age=User.age + 1)
    )
    print('매칭된 행:', result.rowcount)

with SessionLocal() as session:
    rows = session.execute(
        select(User.name, User.age)
        .where(User.age >= 30)
        .order_by(User.id)
    )
    for name, age in rows:
        print(name, age)
```

```text
매칭된 행: 2
Bob 31
Charlie 36
```

`rowcount`는 실제 값이 달라진 행이 아니라 WHERE에 매칭된 행 수이며 드라이버에 따라 `-1`일 수 있습니다. 중요한 변경은 `RETURNING`이나 변경 후 SELECT로 확인합니다.
