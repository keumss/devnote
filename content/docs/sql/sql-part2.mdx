## 안전한 제약 조건 (Constraints)

잘못된 데이터가 들어오지 못하도록 컬럼 규칙을 정합니다. (NOT NULL, UNIQUE, CHECK 등)

```sql
CREATE TABLE products (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100) UNIQUE NOT NULL,
  price INT CHECK (price >= 0),
  status VARCHAR(20) DEFAULT 'active'
);
```

## 관계 설정 (Foreign Key)

두 테이블을 연결하여 데이터 정합성을 유지합니다. (보통 3정규형 적용 시 분리된 테이블을 연결할 때 사용)

```sql
CREATE TABLE orders (
  order_id SERIAL PRIMARY KEY,
  user_id INT REFERENCES users(id) ON DELETE CASCADE,
  amount INT
);
```

## 테이블 결합 (INNER JOIN)

두 테이블에서 공통으로 존재하는 데이터만 연결하여 가져옵니다.

```sql
SELECT users.username, orders.amount
FROM users
INNER JOIN orders ON users.id = orders.user_id;
```

## 외부 결합 (LEFT / RIGHT JOIN)

LEFT JOIN은 왼쪽 테이블의 모든 데이터를 유지하며, 우측에 매칭되는 값이 없으면 NULL을 반환합니다.

```sql
-- 구매 이력이 없는 유저도 포함하여 목록 출력
SELECT users.username, orders.order_id
FROM users
LEFT JOIN orders ON users.id = orders.user_id;
```

## 데이터 입력 (INSERT)

테이블에 새로운 데이터를 추가합니다. 여러 행을 한 번에 삽입할 수도 있습니다.

```sql
INSERT INTO users (username, age)
VALUES
  ('John', 25),
  ('Jane', 28)
RETURNING *; -- Postgres 전용: 입력된 결과 즉시 확인
```

## 수정 / 삭제 (UPDATE / DELETE)

데이터를 변경하거나 지웁니다. WHERE 절을 꼭 써서 전체 데이터가 변경되는 것을 방지하세요.

```sql
-- 수정
UPDATE users SET age = 30 WHERE username = 'John';

-- 삭제
DELETE FROM users WHERE age < 10;
```

## 결과 합치기 (UNION)

두 개 이상의 SELECT 결과를 위아래로 이어 붙입니다. UNION은 중복을 제거하고, UNION ALL은 모두 포함합니다.

```sql
SELECT email FROM customers
UNION ALL
SELECT email FROM employees;
```

## 가상 테이블 (VIEW)

복잡한 쿼리를 하나의 가상 테이블로 만들어 재사용성을 높입니다.

```sql
-- 뷰 생성
CREATE VIEW active_adult_users AS
SELECT * FROM users WHERE status = 'active' AND age >= 20;

-- 뷰 사용 (일반 테이블처럼)
SELECT * FROM active_adult_users;
```
