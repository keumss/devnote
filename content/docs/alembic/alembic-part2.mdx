## 리비전 자동 생성 (Auto Generate)

DB 모델 코드와 실제 DB 상태를 비교하여 변경된 사항을 적용할 파이썬 마이그레이션 스크립트를 만들어냅니다.

```python
# 모델 클래스를 수정한 뒤 아래 명령어로 변경사항 스크립트(.py) 추가
# -m 플래그로 해당 작업의 이름을 붙입니다.
alembic revision --autogenerate -m "Add email column to users table"

# 성공하면 alembic/versions/ 폴더 내에 임의의 해시값이 붙은 파일이 새로 생성됩니다.
```

## 데이터베이스 적용 (Upgrade)

생성된 최신 마이그레이션 스크립트를 실제로 데이터베이스에 반영하여 구조를 업데이트합니다.

```python
# 최신 버전(head)까지 누락된 모든 마이그레이션을 데이터베이스에 반영
alembic upgrade head

# 특정 리비전(해시) 또는 한 단계만 반영하려면 (+1)
alembic upgrade +1
alembic upgrade 8b3c4d5e6f7g
```

## 이전 상태로 롤백 (Downgrade)

구조를 잘못 바꿨거나 오류가 발생했을 때, 이전 리비전의 상태로 되돌리고 싶을 때 역재생(downgrade 함수)을 실행합니다.

```python
# 한 단계 안전하게 이전 버전으로 롤백
alembic downgrade -1

# 특정 단일 리비전(해시) 시점으로 롤백 (해당 시점 이후 적용분이 취소됨)
alembic downgrade 8b3c4d5e6f7g
```

## 히스토리 이력 및 상태 추적

프로젝트에 누적된 리비전 목록이나, 현재 적용 중인 데이터베이스의 작업 버전을 확인할 수 있습니다.

```python
# 모든 마이그레이션 이력(위부터 최신) 나열 및 확인
alembic history --verbose

# 현재 데이터베이스에 반영되어 적용된 최신 리비전 해시값 확인
alembic current
```
