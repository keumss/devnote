## 외래 키 추가 구조 (Foreign Keys)

Field(foreign_key=...) 를 사용하여 다른 테이블의 기본 키를 참조합니다.

```python
class Team(SQLModel, table=True):
    id: Optional[int] = Field(default=None, primary_key=True)
    name: str = Field(index=True)
    headquarters: str

# Hero에 Team의 ID를 외래키로 추가
class Hero(SQLModel, table=True):
    id: Optional[int] = Field(default=None, primary_key=True)
    name: str = Field(index=True)

    # ForeignKey 설정
    team_id: Optional[int] = Field(default=None, foreign_key="team.id")
```

## 양방향 속성 정의 (Relationship)

Relationship() 을 통해 DB 쿼리 없이도 객체 안에서 서로 양방향(1:N) 탐색을 할 수 있도록 만듭니다.

```python
from sqlmodel import Relationship

class Team(SQLModel, table=True):
    # ... (생략)
    # 하나의 팀이 다수의 영웅(Hero)을 가지게 됨 (List 타입 명시)
    heroes: list["Hero"] = Relationship(back_populates="team")

class Hero(SQLModel, table=True):
    # ... (생략)
    team_id: Optional[int] = Field(default=None, foreign_key="team.id")
    # Team 객체로 바로 접근 가능
    team: Optional[Team] = Relationship(back_populates="heroes")
```

## 내부 조인 (Inner Join)

join()을 이용해 두 개 이상의 모델을 연결(교집합)합니다. 두 테이블 모두에 데이터가 있는 경우만 조회됩니다.

```python
with Session(engine) as session:
    # Hero와 Team 데이터를 한 번에 가져오기 (Team에 소속되지 않은 Hero는 리스트에서 제외됨)
    statement = select(Hero, Team).join(Team)

    # (Hero, Team) 튜플 형태로 결과가 반환됩니다.
    results = session.exec(statement)
    for hero, team in results:
        print(f"Hero: {hero.name}, belongs to Team: {team.name}")
```

## 외부 조인 (Left Outer Join)

isouter=True 속성을 추가하여, 매칭되는 대상이 없더라도 왼쪽 테이블의 데이터는 모두 가져옵니다.

```python
with Session(engine) as session:
    # Team이 없는 Hero도 모두 조회하기 위함 (Left Join)
    statement = select(Hero, Team).join(Team, isouter=True)

    results = session.exec(statement)
    for hero, team in results:
        # team은 None일 수 있습니다. (히어로가 팀에 속하지 않은 경우)
        team_name = team.name if team else "No Team"
        print(f"Hero: {hero.name}, Team: {team_name}")
```

## Eager Loading (selectinload / joinedload)

N+1 쿼리 문제를 해결하기 위해, 연관된 데이터들을 사전에 쿼리하여 메모리에 함께 로딩해둡니다.

```python
from sqlalchemy.orm import selectinload, joinedload

with Session(engine) as session:
    # selectinload: 1:N 관계에서 주로 사용 (별도의 SELECT 쿼리를 1번 더 실행하여 매핑함. 리스트에 적합)
    # joinedload: 1:1 또는 N:1 관계에서 주로 사용 (JOIN을 통해 한 번의 쿼리로 다 가져옴)
    statement = select(Team).options(selectinload(Team.heroes))

    teams = session.exec(statement).all()
    for team in teams:
        # 관계형 데이터(team.heroes)에 접근할 때 DB 조회가 또 발생하지 않습니다.
        print(f"Team: {team.name}, Heroes Count: {len(team.heroes)}")
```
