## C: 포스트 생성 (POST)

새로운 블로그 글을 작성하는 엔드포인트입니다. 클라이언트가 넘긴 데이터를 받아 데이터베이스에 저장합니다.

```python
@app.post("/posts/", response_model=Post)
def create_post(*, session: Session = Depends(get_session), post: PostCreate):
    # Pydantic v2 방식 (SQLModel 0.0.14+ 권장)
    db_post = Post.model_validate(post)
    session.add(db_post)
    session.commit()
    session.refresh(db_post)
    return db_post
```

## 응답 모델(Response Model)과 데이터 은닉

데이터베이스 모델(Post, User 등)을 그대로 반환하면 비밀번호와 같은 민감한 정보가 클라이언트에 노출될 수 있습니다. 응답 전용 모델(예: PostPublic, UserPublic)을 만들어 response_model 속성에 지정하면, FastAPI가 반환 데이터에서 자동으로 민감한 정보를 필터링해 줍니다.

```python
from sqlmodel import SQLModel
from datetime import datetime

# 외부에 노출할 공개용 응답 모델 (비밀번호나 내부 ID 제외)
class UserPublic(SQLModel):
    username: str

class PostPublic(SQLModel):
    id: int
    title: str
    content: str
    created_at: datetime
    # 작성자 정보도 공개용 모델로 포맷팅
    author: UserPublic

# 라우터 반환 시 자동으로 필터링 및 데이터 변환이 적용됨
@app.get("/posts/{post_id}", response_model=PostPublic)
def read_post_public(post_id: int, session: Session = Depends(get_session)):
    post = session.get(Post, post_id)
    return post
```

## R: 포스트 목록 조회 (GET)

오프셋(offset)과 리밋(limit)을 활용하여 페이지네이션 처리가 된 게시글 목록을 불러옵니다.

```python
from typing import List
from sqlmodel import select

@app.get("/posts/", response_model=List[Post])
def read_posts(offset: int = 0, limit: int = 20, session: Session = Depends(get_session)):
    posts = session.exec(select(Post).offset(offset).limit(limit)).all()
    return posts
```

## 블로그 포스트 검색 (Query Parameters)

사용자가 입력한 키워드를 기반으로 포스트 제목이나 내용에서 검색을 수행하는 방법입니다. FastAPI에서는 경로 매개변수 외의 함수 인자가 자동으로 쿼리 매개변수(Query Params)가 됩니다.

```python
from typing import List, Optional
from sqlmodel import select, or_

@app.get("/posts/search/", response_model=List[Post])
def search_posts(q: Optional[str] = None, session: Session = Depends(get_session)):
    query = select(Post)
    if q:
        # 검색어(q)가 포함된 제목 또는 내용을 필터링합니다.
        query = query.where(or_(Post.title.contains(q), Post.content.contains(q)))

    # 최신순 정렬 후 반환합니다.
    posts = session.exec(query.order_by(Post.created_at.desc())).all()
    return posts
```

## R: 특정 포스트 상세 조회 (GET)

포스트 ID를 경로 매개변수(Path parameter)로 받아 특정 글의 상세 내용을 조회합니다. 없을 경우 404 예외를 발생시킵니다.

```python
from fastapi import HTTPException

@app.get("/posts/{post_id}", response_model=Post)
def read_post(post_id: int, session: Session = Depends(get_session)):
    post = session.get(Post, post_id)
    if not post:
        raise HTTPException(status_code=404, detail="Post not found")
    return post
```

## U: 포스트 수정 (PATCH)

일부 내용만 수정하는 PATCH 라우트입니다. 클라이언트가 보낸 데이터 중 None이 아닌 값만 업데이트합니다.

```python
@app.patch("/posts/{post_id}", response_model=Post)
def update_post(*, session: Session = Depends(get_session), post_id: int, post_update: PostUpdate):
    db_post = session.get(Post, post_id)
    if not db_post:
        raise HTTPException(status_code=404, detail="Post not found")

    # Pydantic v2 방식
    update_data = post_update.model_dump(exclude_unset=True)
    for key, value in update_data.items():
        setattr(db_post, key, value)

    session.add(db_post)
    session.commit()
    session.refresh(db_post)
    return db_post
```

## D: 포스트 삭제 (DELETE)

특정 포스트를 삭제합니다.

```python
@app.delete("/posts/{post_id}")
def delete_post(*, session: Session = Depends(get_session), post_id: int):
    post = session.get(Post, post_id)
    if not post:
        raise HTTPException(status_code=404, detail="Post not found")

    session.delete(post)
    session.commit()
    return {"ok": True}
```
