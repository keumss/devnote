## 입력과 결과 계약 정하기

이번 예제는 `feedbacks.json`의 고객 피드백을 읽어 모델로 분류하고, 사람이 검토할지 자동 답변 후보로 둘지를 `review_queue.jsonl`에 저장하는 `feedback_triage.py`를 만듭니다. 모델에는 분류에 필요한 `message`만 보내고, 이메일 주소처럼 결과 생성에 필요하지 않은 데이터는 보내지 않습니다.

```json
[
  {
    "feedback_id": "fb-1001",
    "message": "결제는 됐는데 영수증을 받지 못했어요.",
    "customer_email": "mina@example.com"
  },
  {
    "feedback_id": "fb-1002",
    "message": "검색 결과가 비어 있어요."
  }
]
```

```python
# feedback_triage.py
from __future__ import annotations

import json
from dataclasses import asdict, dataclass
from pathlib import Path
from typing import Literal

import openai
from openai import OpenAI
from pydantic import BaseModel

MODEL = "gpt-5.6"

@dataclass(frozen=True)
class Feedback:
    feedback_id: str
    message: str
    customer_email: str | None = None

class Triage(BaseModel):
    category: Literal["billing", "bug", "question"]
    summary: str
    needs_human: bool

@dataclass(frozen=True)
class QueueItem:
    feedback_id: str
    category: Literal["billing", "bug", "question"]
    action: Literal["human_review", "auto_reply", "retry"]
    summary: str
```

`feedback_id`는 작업을 추적하는 고유 키이고 `customer_email`은 외부 경계에서만 필요할 수 있는 값입니다. 모델의 분류 결과에는 실행 명령이나 이메일 본문 대신 카테고리·요약·사람 검토 여부만 허용합니다.

## responses.parse로 피드백을 검증된 분류 결과로 바꾸기

분류 함수는 텍스트 대신 `Triage` 객체를 반환하게 만듭니다. 호출부는 프롬프트와 SDK 세부 사항을 알 필요 없이 `category`와 `needs_human`만 사용합니다.

```python
def classify_feedback(client: OpenAI, message: str) -> Triage:
    response = client.responses.parse(
        model=MODEL,
        input=[
            {
                "role": "developer",
                "content": (
                    "고객 피드백을 billing, bug, question 중 하나로 분류한다. "
                    "긴급하거나 불확실한 경우 needs_human을 true로 둔다."
                ),
            },
            {"role": "user", "content": message},
        ],
        text_format=Triage,
    )

    if response.output_parsed is None:
        raise RuntimeError("피드백 분류 결과가 비어 있습니다.")
    return response.output_parsed
```

`message`만 함수 인수로 받으므로 이후에 프롬프트를 고쳐도 이메일을 실수로 전달하기 어렵습니다. 이 함수는 외부 API 호출을 하므로 다음 단계에서 실패를 작업 상태로 바꿉니다.

## 분류 결과와 일시적 실패를 대기열 항목으로 만들기

사람 검토 여부는 모델의 구조화된 필드를 기준으로 정하고, 요청 한도나 연결 실패는 `retry`로 기록합니다. 인증·권한·잘못된 요청 같은 오류는 조용히 재시도하지 않고 요청 ID와 함께 호출자에게 전달합니다.

```python
def make_queue_item(feedback: Feedback, triage: Triage) -> QueueItem:
    action: Literal["human_review", "auto_reply", "retry"]
    action = "human_review" if triage.needs_human else "auto_reply"
    return QueueItem(
        feedback_id=feedback.feedback_id,
        category=triage.category,
        action=action,
        summary=triage.summary,
    )


def triage_one(client: OpenAI, feedback: Feedback) -> QueueItem:
    try:
        triage = classify_feedback(client, feedback.message)
    except (openai.APIConnectionError, openai.RateLimitError):
        return QueueItem(
            feedback_id=feedback.feedback_id,
            category="question",
            action="retry",
            summary="일시적 API 오류로 재시도 필요",
        )
    except openai.APIStatusError as error:
        raise RuntimeError(
            f"피드백 {feedback.feedback_id} 분류 실패: request_id={error.request_id}"
        ) from error

    return make_queue_item(feedback, triage)
```

이 예제의 `auto_reply`는 답장을 전송하는 명령이 아니라 자동 답변 후보라는 대기열 상태입니다. 실제 발송 기능을 붙일 때는 사용자 동의, 수신자 검증, 중복 전송을 막는 멱등성 키를 추가합니다.

## JSON 입력을 읽고 JSONL 대기열로 저장하기

입력 파일을 한 번 읽어 `Feedback`으로 바꾸고, 결과는 줄마다 독립된 JSON 객체인 JSONL로 저장합니다. 실패 항목도 결과에 남으므로 운영자가 재시도 대상을 잃지 않습니다.

```python
def load_feedbacks(path: Path) -> list[Feedback]:
    rows = json.loads(path.read_text(encoding="utf-8"))
    return [Feedback(**row) for row in rows]


def save_queue_items(path: Path, items: list[QueueItem]) -> None:
    lines = [
        json.dumps(asdict(item), ensure_ascii=False)
        for item in items
    ]
    path.write_text("\n".join(lines) + "\n", encoding="utf-8")


def main() -> None:
    client = OpenAI(timeout=20.0)
    feedbacks = load_feedbacks(Path("feedbacks.json"))
    queue_items = [triage_one(client, feedback) for feedback in feedbacks]
    save_queue_items(Path("review_queue.jsonl"), queue_items)
    print(f"{len(queue_items)}건을 review_queue.jsonl에 저장했습니다.")


if __name__ == "__main__":
    main()
```

## 가짜 클라이언트로 대기열 계약 테스트하기

네트워크와 API 키 없이도 분류 결과를 대기열 상태로 바꾸는 계약은 테스트할 수 있습니다. 가짜 응답 객체가 `output_parsed`를 제공하도록 만들어 실제 SDK 호출 대신 고정된 `Triage`를 반환합니다.

```python
# test_feedback_triage.py
from types import SimpleNamespace

from feedback_triage import Feedback, Triage, classify_feedback, make_queue_item

class FakeResponses:
    def __init__(self, triage: Triage) -> None:
        self.triage = triage
        self.kwargs: dict[str, object] = {}

    def parse(self, **kwargs: object) -> SimpleNamespace:
        self.kwargs = kwargs
        return SimpleNamespace(output_parsed=self.triage)

class FakeClient:
    def __init__(self, triage: Triage) -> None:
        self.responses = FakeResponses(triage)

def test_feedback_becomes_human_review_item() -> None:
    fake_client = FakeClient(
        Triage(
            category="bug",
            summary="검색 결과가 비어 있음",
            needs_human=True,
        )
    )
    feedback = Feedback("fb-1002", "검색 결과가 비어 있어요.")

    triage = classify_feedback(fake_client, feedback.message)
    item = make_queue_item(feedback, triage)

    assert item.action == "human_review"
    assert item.category == "bug"
    assert fake_client.responses.kwargs["text_format"] is Triage
```

실제 API를 호출하는 통합 테스트는 별도 환경과 비용 한도를 두고 실행합니다. 단위 테스트는 분류 결과를 받아도 상태 변경 없이 올바른 대기열 항목을 만드는지에 집중합니다.

구조화된 결과 형식은 [Structured Outputs 가이드](https://developers.openai.com/api/docs/guides/structured-outputs), Python SDK의 오류와 요청 ID·재시도 동작은 [OpenAI Python 라이브러리 README](https://github.com/openai/openai-python)를 기준으로 확인합니다.
