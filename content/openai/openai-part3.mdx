## strict 함수 스키마로 모델이 요청할 입력 제한하기

모델이 주문 조회나 사내 검색처럼 애플리케이션의 기능을 선택하게 할 때는 `tools`에 함수의 이름·설명·JSON Schema를 선언합니다. 도구 설명은 모델에게 가능한 동작을 알려 줄 뿐이고, 실제 Python 함수 실행은 애플리케이션이 결정합니다.

```python
LOOKUP_ORDER_TOOL = {
    "type": "function",
    "name": "lookup_order",
    "description": "주문 ID로 현재 주문 상태를 읽기 전용으로 조회한다.",
    "parameters": {
        "type": "object",
        "properties": {
            "order_id": {
                "type": "string",
                "description": "예: ORD-1001",
            },
        },
        "required": ["order_id"],
        "additionalProperties": False,
    },
    "strict": True,
}
```

`strict=True`를 사용하면 함수 인수가 스키마를 따르도록 요청합니다. 이 모드에서는 각 객체의 `additionalProperties`가 `False`여야 하고, `properties`의 모든 필드는 `required`에 있어야 합니다. 선택 필드는 생략하는 대신 `null`을 허용하는 타입으로 표현합니다.

## 허용 목록과 검증으로 실제 함수를 선택하기

함수 호출의 이름과 인수는 모델 출력이므로 `globals()`나 `getattr()`로 곧바로 실행하지 않습니다. 허용한 도구 이름만 분기하고, 도구의 범위와 인수를 코드에서도 다시 검사합니다.

```python
orders = {
    "ORD-1001": {"status": "paid", "total": 12000},
    "ORD-1002": {"status": "shipped", "total": 25000},
}

def lookup_order(order_id: str) -> dict[str, str | int | bool]:
    order = orders.get(order_id)
    if order is None:
        return {"found": False}
    return {"found": True, "order_id": order_id, **order}


def call_tool(name: str, arguments: dict[str, object]) -> dict[str, str | int | bool]:
    if name != "lookup_order":
        raise ValueError(f"허용하지 않는 도구입니다: {name}")

    order_id = arguments.get("order_id")
    if not isinstance(order_id, str) or not order_id.startswith("ORD-"):
        raise ValueError("올바른 주문 ID가 필요합니다.")

    return lookup_order(order_id)


print(call_tool("lookup_order", {"order_id": "ORD-1002"}))
```

```text
{'found': True, 'order_id': 'ORD-1002', 'status': 'shipped', 'total': 25000}
```

조회처럼 읽기 전용 도구부터 연결합니다. 환불, 이메일 전송, 파일 삭제처럼 상태를 바꾸는 도구에는 사용자 권한 확인, 실행 전 확인 화면, 멱등성 키를 별도로 두며 모델의 함수 호출만으로 실행하지 않습니다.

## function_call 결과를 다시 모델에 전달해 답변 완성하기

Responses API가 함수 호출을 반환하면 `response.output`에서 `type == "function_call"`인 항목을 모두 처리합니다. 함수 결과를 `function_call_output`으로 만들어 다시 보내면, 모델이 실제 조회 결과를 바탕으로 최종 사용자 답변을 작성합니다.

```python
import json

from openai import OpenAI

MODEL = "gpt-5.6"

def answer_order_question(client: OpenAI, question: str) -> str:
    response = client.responses.create(
        model=MODEL,
        input=[{"role": "user", "content": question}],
        tools=[LOOKUP_ORDER_TOOL],
    )

    tool_outputs = []
    for item in response.output:
        if item.type != "function_call":
            continue

        arguments = json.loads(item.arguments)
        result = call_tool(item.name, arguments)
        tool_outputs.append(
            {
                "type": "function_call_output",
                "call_id": item.call_id,
                "output": json.dumps(result, ensure_ascii=False),
            }
        )

    if not tool_outputs:
        return response.output_text

    final_response = client.responses.create(
        model=MODEL,
        input=tool_outputs,
        tools=[LOOKUP_ORDER_TOOL],
    )
    return final_response.output_text


client = OpenAI()
print(answer_order_question(client, "ORD-1002 주문 상태를 알려 줘."))
```

한 응답에서 함수가 여러 번 호출될 수 있으므로 첫 항목만 처리하지 않습니다. 함수 결과는 문자열, 보통은 JSON으로 전달하고, 실패한 조회도 성공한 것처럼 꾸미지 말고 오류 코드나 `found: false` 같은 명확한 결과로 전달합니다.

함수 호출의 도구 정의·반환 루프·도구 선택과 strict 모드는 [OpenAI 함수 호출 가이드](https://developers.openai.com/api/docs/guides/function-calling)를 기준으로 확인합니다.
