## 컬럼 이름과 타입 변경의 전제조건 명시하기

이름 변경은 autogenerate가 삭제와 추가로 오해할 수 있으므로 의도를 직접 작성합니다. 기존 타입과 nullable 정보를 함께 적으면 리비전만 읽어도 변경 전 상태를 알 수 있고, 전체 컬럼 정의가 필요한 MySQL 같은 DB에서도 안전하게 SQL을 생성할 수 있습니다.

```python
from alembic import op
import sqlalchemy as sa


def upgrade() -> None:
    op.alter_column(
        'users',
        'display_name',
        new_column_name='name',
        existing_type=sa.String(length=100),
        existing_nullable=False,
    )


def downgrade() -> None:
    op.alter_column(
        'users',
        'name',
        new_column_name='display_name',
        existing_type=sa.String(length=100),
        existing_nullable=False,
    )
```

컬럼 타입 변경은 저장된 모든 값이 새 타입으로 변환 가능한지 먼저 검증합니다. 예를 들어 문자열 나이에 숫자가 아닌 값이 들어간 뒤 다시 정수로 downgrade하면 실패합니다. PostgreSQL의 `postgresql_using`, MySQL의 전체 기존 정의처럼 DB별 요구사항도 있으므로 대상 DB에서 생성 SQL과 실제 실행을 모두 시험합니다.

## 임시 서버 기본값으로 기존 행을 채운 뒤 제거하기

기존 행이 있는 테이블에 NOT NULL 컬럼을 추가할 때 서버 기본값을 임시로 사용하면 한 문장으로 추가할 수 있는 DB도 있습니다. 애플리케이션이 앞으로 값을 반드시 전달하게 만들 목적이라면 기존 행을 채운 뒤 기본값을 제거해 의도를 분명히 합니다.

```python
from alembic import op
import sqlalchemy as sa


def upgrade() -> None:
    op.add_column(
        'users',
        sa.Column(
            'is_active',
            sa.Boolean(),
            nullable=False,
            server_default=sa.true(),
        ),
    )
    op.alter_column(
        'users',
        'is_active',
        existing_type=sa.Boolean(),
        existing_nullable=False,
        server_default=None,
    )


def downgrade() -> None:
    op.drop_column('users', 'is_active')
```

`server_default`는 DB가 INSERT 시 적용하는 기본값이고 Python 모델의 `default`와 다릅니다. 제품 요구사항이 “값을 생략하면 항상 활성화”라면 제거하지 않고 모델 메타데이터에도 같은 서버 기본값을 선언합니다. 어느 정책이든 모델과 실제 DB가 일치해야 `alembic check`에서 반복 차이가 생기지 않습니다.

제약 조건은 나중에 안정적으로 변경할 수 있도록 SQLAlchemy `MetaData`의 `naming_convention`으로 이름을 부여합니다.

```python
import sqlalchemy as sa

naming_convention = {
    'ix': 'ix_%(column_0_label)s',
    'uq': 'uq_%(table_name)s_%(column_0_name)s',
    'ck': 'ck_%(table_name)s_%(column_0_name)s',
    'fk': 'fk_%(table_name)s_%(column_0_name)s_%(referred_table_name)s',
    'pk': 'pk_%(table_name)s',
}

metadata = sa.MetaData(naming_convention=naming_convention)
```

## 데이터 크기에 따라 마이그레이션 방식을 선택하기

코드 테이블의 초기 행처럼 작고 고정된 데이터는 리비전 안에서 `bulk_insert()`로 함께 관리할 수 있습니다. ORM 모델은 시간이 지나며 바뀌므로 과거 리비전에서는 최소한의 임시 테이블 정의를 사용합니다.

```python
from alembic import op
import sqlalchemy as sa

roles = sa.table(
    'roles',
    sa.column('code', sa.String(length=20)),
    sa.column('label', sa.String(length=50)),
)


def upgrade() -> None:
    op.bulk_insert(
        roles,
        [
            {'code': 'admin', 'label': '관리자'},
            {'code': 'member', 'label': '회원'},
        ],
    )


def downgrade() -> None:
    op.execute(roles.delete().where(roles.c.code.in_(['admin', 'member'])))
```

운영 데이터 전체를 변환하는 작업은 처리 시간이 길고 재시도·관찰·속도 제한이 필요합니다. 이 경우 다음처럼 스키마 변경과 데이터 이동을 분리합니다.

```text
1차 배포: 새 nullable 컬럼 추가 → 구·신 컬럼을 모두 읽고 쓰는 코드 배포
백그라운드 작업: 기본 키 범위로 나눠 백필 → 처리량과 실패 건수 관찰
2차 배포: 누락 검증 → NOT NULL·인덱스 추가 → 읽기를 새 컬럼으로 전환
3차 배포: 이전 버전이 모두 종료된 뒤 구 컬럼 삭제
```

이 흐름은 즉시 downgrade에 의존하지 않고 이전 애플리케이션과 새 스키마가 공존할 시간을 만듭니다. 데이터 삭제가 포함된 마지막 단계는 백업과 보존 정책을 확인한 뒤 실행합니다.

## SQLite batch mode의 재생성 동작과 제약 이해하기

SQLite에서 직접 지원하지 않는 형태의 테이블 변경은 `batch_alter_table()`로 표현합니다. Alembic은 기존 테이블을 반영해 임시 테이블을 만들고, 데이터를 복사한 뒤 기존 테이블을 교체합니다.

```python
from alembic import op
import sqlalchemy as sa


def upgrade() -> None:
    with op.batch_alter_table('users') as batch_op:
        batch_op.drop_column('legacy_name')


def downgrade() -> None:
    with op.batch_alter_table('users') as batch_op:
        batch_op.add_column(
            sa.Column('legacy_name', sa.String(length=50), nullable=True)
        )
```

autogenerate도 batch 문맥으로 출력하려면 `env.py`에 옵션을 추가합니다. 다른 DB에서는 기본적으로 일반 ALTER처럼 동작합니다.

```python
context.configure(
    connection=connection,
    target_metadata=target_metadata,
    render_as_batch=True,
)
```

테이블 재생성은 외래 키와 CHECK 제약의 영향을 받습니다. 이름 없는 CHECK 제약은 반영 과정에서 누락될 수 있고, 다른 테이블이 참조하는 동안 대상 테이블을 삭제하지 못할 수도 있습니다. 제약에 이름을 붙이고 실제 SQLite 설정에서 upgrade와 downgrade를 실행해 재생성된 테이블 구조와 행 수를 확인합니다.
