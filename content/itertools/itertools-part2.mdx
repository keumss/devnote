## accumulate()로 누적 합 계산하기

매 단계의 중간 합계가 필요할 때 `accumulate()`를 사용합니다. 기본 연산은 덧셈이고, `func` 인수로 다른 이진 함수를 지정할 수 있습니다.

```python
from itertools import accumulate

daily_sales = [12000, 8500, 15000, 9300, 11200]
running_total = list(accumulate(daily_sales))
print(running_total)
```

```text
[12000, 20500, 35500, 44800, 56000]
```

`accumulate()`는 누적 중간값을 모두 내보냅니다. 최종 합계만 필요하면 `sum()`을 씁니다.

## accumulate()에 func 지정해 누적 최대값 구하기

판매 최고 기록처럼 매 시점의 최대값을 누적할 때 `func=max`를 지정합니다. 이전 최대값과 현재 값 중 큰 것을 계속 전달합니다.

```python
from itertools import accumulate
import operator

scores = [73, 91, 85, 98, 76, 88]
running_max = list(accumulate(scores, func=max))
print(running_max)
```

```text
[73, 91, 91, 98, 98, 98]
```

`operator.add`, `operator.mul` 등 표준 연산자 함수도 쓸 수 있습니다. `initial` 인수로 시작값을 따로 줄 수 있으며, 이 경우 출력 길이가 입력보다 하나 더 깁니다.

## groupby()로 연속된 같은 키 묶기

같은 값이 연속된 항목을 묶을 때 `groupby()`를 사용합니다. 각 그룹은 `(key, sub_iterator)` 쌍으로 반환됩니다.

```python
from itertools import groupby

events = ['login', 'login', 'view', 'view', 'view', 'logout']
for key, group in groupby(events):
    print(key, list(group))
```

```text
login ['login', 'login']
view ['view', 'view', 'view']
logout ['logout']
```

`sub_iterator`는 다음 그룹으로 넘어가면 소진됩니다. 루프 안에서 `list(group)`으로 먼저 소비해야 안전합니다.

## groupby() 전에 정렬이 필요한 이유

`groupby()`는 인접한 항목만 묶습니다. 같은 키가 떨어져 있으면 별개 그룹이 됩니다. 전체 키 기준으로 묶으려면 반드시 미리 정렬합니다.

```python
from itertools import groupby

orders = [
    {'product': 'A', 'qty': 2},
    {'product': 'B', 'qty': 1},
    {'product': 'A', 'qty': 3},
]

# 정렬 없이 groupby
for key, group in groupby(orders, key=lambda o: o['product']):
    print(key, [o['qty'] for o in group])
```

```text
A [2]
B [1]
A [3]
```

`'A'` 그룹이 두 번 나타납니다. 상품별 전체 합계가 필요하면 `sorted(orders, key=lambda o: o['product'])` 후에 `groupby()`를 적용합니다.

## filterfalse()로 조건 불만족 항목 고르기

조건을 만족하지 않는 항목만 남길 때 `filterfalse()`를 사용합니다. 내장 `filter()`와 반대로 동작합니다.

```python
from itertools import filterfalse

records = [
    {'name': 'Alice', 'active': True},
    {'name': 'Bob', 'active': False},
    {'name': 'Carol', 'active': True},
    {'name': 'Dave', 'active': False},
]

inactive = list(filterfalse(lambda r: r['active'], records))
print([r['name'] for r in inactive])
```

```text
['Bob', 'Dave']
```

`filter(lambda r: not r['active'], records)`와 결과가 같지만, 부정 조건을 명시하지 않아 의도를 더 직접적으로 표현합니다.

## compress()로 마스크 기반 선택하기

불리언 마스크로 데이터를 선택할 때 `compress()`를 사용합니다. 두 번째 인수의 참 위치에 해당하는 항목만 남깁니다.

```python
from itertools import compress

products = ['apple', 'banana', 'cherry', 'date', 'elderberry']
in_stock = [True, False, True, True, False]

available = list(compress(products, in_stock))
print(available)
```

```text
['apple', 'cherry', 'date']
```

두 이터러블의 길이가 다르면 짧은 쪽에서 끝납니다. 마스크가 동적으로 생성되는 경우에 유용합니다.

## pairwise()로 인접 쌍 만들기

연속된 두 항목을 비교하거나 차이를 계산할 때 `pairwise()`를 사용합니다. 길이 n의 입력에서 n-1개의 `(a, b)` 쌍을 만듭니다.

```python
from itertools import pairwise

checkpoints = [0, 15, 42, 78, 100]
for start, end in pairwise(checkpoints):
    print(f'{start} → {end}: {end - start}km')
```

```text
0 → 15: 15km
15 → 42: 27km
42 → 78: 36km
78 → 100: 22km
```

`pairwise()`는 Python 3.10에서 추가되었습니다. 슬라이딩 윈도우가 2보다 크면 `zip(it, islice(it, n, None))`으로 직접 만듭니다.

## starmap()으로 튜플 시퀀스에 함수 적용하기

인수가 튜플로 묶인 시퀀스에 함수를 적용할 때 `starmap()`을 사용합니다. `map(func, *iterables)`와 달리, 하나의 이터러블에 담긴 튜플을 언패킹해 함수에 넘깁니다.

```python
from itertools import starmap

pairs = [(3, 2), (10, 3), (2, 8)]
results = list(starmap(pow, pairs))
print(results)
```

```text
[9, 1000, 256]
```

`starmap(func, pairs)`는 `[func(*pair) for pair in pairs]`와 같습니다. 데이터가 이미 튜플 형태로 저장되어 있을 때 언패킹 없이 바로 함수에 넣을 수 있습니다.

공식 Python [itertools 문서](https://docs.python.org/3.12/library/itertools.html)의 `accumulate`, `groupby`, `filterfalse`, `compress`, `pairwise`, `starmap` 동작을 기준으로 Python 3.12 예제를 구성했습니다.
