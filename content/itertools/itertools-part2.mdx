## batched로 두 개씩, 세 개씩 나누기

한 번에 처리할 수 있는 개수가 정해져 있을 때 `batched()`를 사용합니다. 예를 들어 API가 사용자 ID를 한 요청에 두 개까지만 받는다면, 목록을 두 개씩 나누어 보낼 수 있습니다. `batched()`는 Python 3.12에서 추가되었습니다.

```python
from itertools import batched

user_ids = [101, 102, 103, 104, 105]

for ids in batched(user_ids, 2):
    print(ids)
```

```text
(101, 102)
(103, 104)
(105,)
```

마지막 묶음은 입력이 남은 만큼만 들어가므로 짧을 수 있습니다. 마지막도 정확히 두 개여야 한다면, 짧은 묶음을 어떻게 처리할지 호출 전에 정합니다.

## accumulate로 누계를 한 번에 만들기

`sum()`은 최종 합계 하나만 반환합니다. 매일의 누계처럼 중간 합계도 모두 필요할 때는 `accumulate()`를 사용합니다.

```python
from itertools import accumulate

daily_sales = [120, 80, 150]
running_totals = accumulate(daily_sales)

print(list(running_totals))
```

```text
[120, 200, 350]
```

두 번째 인수에 계산 함수를 넣으면 덧셈 대신 다른 누적도 만들 수 있습니다. 지금까지 가장 느린 응답 시간을 알고 싶다면 `max`를 사용합니다.

```python
from itertools import accumulate

latencies = [120, 98, 135, 110]
print(list(accumulate(latencies, max)))
```

```text
[120, 120, 135, 135]
```

## groupby는 붙어 있는 같은 값만 묶습니다

`groupby()`는 같은 값이 **연속해서** 나오는 구간을 묶습니다. 아래 입력에서 `'paid'`는 떨어져 있으므로 두 그룹으로 나뉩니다.

```python
from itertools import groupby

statuses = ['paid', 'paid', 'pending', 'paid']

for status, group in groupby(statuses):
    print(status, list(group))
```

```text
paid ['paid', 'paid']
pending ['pending']
paid ['paid']
```

상태별로 한 그룹만 만들고 싶다면 먼저 정렬합니다. SQL의 `GROUP BY`와 달리 `groupby()`는 자동으로 멀리 떨어진 값을 모으지 않는다는 점을 기억합니다.

```python
from itertools import groupby

statuses = ['paid', 'paid', 'pending', 'paid']
for status, group in groupby(sorted(statuses)):
    print(status, list(group))
```

```text
paid ['paid', 'paid', 'paid']
pending ['pending']
```

## groupby의 결과는 그 자리에서 사용하기

`groupby()`가 주는 `group`은 목록이 아니라 원본을 함께 읽는 이터레이터입니다. 다음 그룹으로 넘어가기 전에 현재 그룹을 목록으로 만들거나 합계처럼 필요한 계산을 끝냅니다.

```python
from itertools import groupby

orders = [('paid', 12000), ('paid', 8000), ('refunded', -8000)]

for status, group in groupby(orders, key=lambda order: order[0]):
    amount = sum(price for _, price in group)
    print(status, amount)
```

```text
paid 20000
refunded -8000
```

위처럼 합계만 필요하면 목록을 만들 필요가 없습니다. 나중에 각 주문도 다시 써야 한다면 `items = list(group)`으로 현재 반복 안에서 저장합니다.

## compress로 선택된 값만 남기기

`compress()`는 데이터와 같은 위치의 참·거짓 값을 보고, `True`인 데이터만 꺼냅니다. 표시할 열을 선택하거나 권한이 있는 항목만 남길 때 쓸 수 있습니다.

```python
from itertools import compress

fields = ['id', 'email', 'token', 'name']
visible = [True, True, False, True]

print(list(compress(fields, visible)))
```

```text
['id', 'email', 'name']
```

데이터와 선택값 중 하나가 먼저 끝나면 `compress()`도 멈춥니다. 선택값의 개수가 정확히 맞아야 하는 중요한 작업이라면 길이를 먼저 확인합니다.

## takewhile과 dropwhile로 앞부분 건너뛰기

두 함수는 조건에 맞는 값을 전체에서 찾는 필터가 아니라, **앞부분의 경계**를 찾습니다. `takewhile()`은 조건이 참인 동안만 꺼내고, `dropwhile()`은 조건이 참인 앞부분을 버린 뒤 나머지를 모두 꺼냅니다.

```python
from itertools import dropwhile, takewhile

levels = [1, 2, 3, 0, 4]
print(list(takewhile(lambda level: level > 0, levels)))

header_lines = ['# 제목', '# 설명', '본문', '# 결론']
print(list(dropwhile(lambda line: line.startswith('#'), header_lines)))
```

```text
[1, 2, 3]
['본문', '# 결론']
```

`takewhile()`은 `0`을 만나는 즉시 멈추므로 뒤의 `4`는 반환하지 않습니다. `dropwhile()`은 처음 조건이 거짓인 `'본문'`부터는 조건을 다시 검사하지 않고 전부 반환합니다.

## starmap으로 튜플을 함수 인자로 넘기기

입력이 `(가로, 세로)`처럼 함수에 전달할 인자 묶음일 때 `starmap()`을 씁니다. 각 튜플 앞에 `*`를 붙여 호출하는 일을 대신합니다.

```python
from itertools import starmap

rectangles = [(3, 4), (5, 2), (10, 1)]
areas = starmap(lambda width, height: width * height, rectangles)

print(list(areas))
```

```text
[12, 10, 10]
```

복잡한 계산이라면 람다 대신 이름 있는 함수를 사용합니다. `starmap()`은 한 항목씩 결과를 만들므로, 큰 입력도 `for area in areas:`처럼 바로 처리할 수 있습니다.
