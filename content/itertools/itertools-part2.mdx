## accumulate로 누적 상태 계산하기

`accumulate(iterable, function)`은 최종값 하나가 아니라 각 단계의 누적 결과를 반환합니다. 함수 기본값은 덧셈이고, `max`, `min`, `operator.mul`이나 두 인자를 받는 사용자 함수를 전달할 수 있습니다. `initial`을 지정하면 그 값이 첫 출력이 되어 출력 길이가 입력보다 하나 늘어납니다.

```python
from itertools import accumulate

daily_changes = [10, -3, 5]
balances = accumulate(daily_changes, initial=100)
running_peak = accumulate([3, 1, 6, 4], max)

print(list(balances))
print(list(running_peak))
```

```text
[100, 110, 107, 112]
[3, 3, 6, 6]
```

마지막 누적값만 필요하면 `sum()`이나 `functools.reduce()`가 의도를 더 잘 드러낼 수 있습니다. 반대로 잔액 변화, 실행 최고값, 단계별 상태처럼 중간 결과가 필요할 때 `accumulate()`를 선택합니다.

## starmap으로 튜플을 함수 인자로 펼치기

`map(function, iterable)`이 각 원소를 인자 하나로 전달한다면 `starmap(function, iterable)`은 각 원소를 `function(*arguments)` 형태로 펼칩니다. CSV에서 읽은 좌표나 이미 묶인 함수 인자 목록을 처리할 때 불필요한 람다를 줄일 수 있습니다.

```python
from itertools import starmap

def price_with_tax(price: int, tax_rate: float) -> float:
    return round(price * (1 + tax_rate), 2)

orders = [(10_000, 0.1), (25_000, 0.08)]
totals = starmap(price_with_tax, orders)

print(list(totals))
```

```text
[11000.0, 27000.0]
```

각 튜플의 길이와 함수 매개변수 개수가 맞지 않으면 값을 꺼내는 시점에 `TypeError`가 발생합니다. 지연 평가 때문에 `starmap()`을 만든 줄이 아니라 반복하는 줄에서 오류가 나타날 수 있으므로 외부 입력은 변환·검증한 뒤 연결합니다.

## compress와 filterfalse로 조건에 맞는 원소 선택하기

`compress(data, selectors)`는 같은 위치의 선택자가 참인 데이터만 반환합니다. 두 입력 중 짧은 쪽이 끝나면 멈추므로 선택자 개수를 점검해야 데이터가 조용히 누락되지 않습니다. `filterfalse(predicate, iterable)`는 내장 `filter()`와 반대로 조건이 거짓인 원소를 반환합니다.

```python
from itertools import compress, filterfalse

features = ['search', 'export', 'billing', 'audit']
enabled = [True, False, True, False]
disabled_statuses = filterfalse(lambda status: status == 'ok', ['ok', 'error', 'ok', 'timeout'])

print(list(compress(features, enabled)))
print(list(disabled_statuses))
```

```text
['search', 'billing']
['error', 'timeout']
```

데이터 자체로 조건을 계산할 수 있다면 제너레이터 표현식이 더 읽기 쉬울 수 있습니다. 이미 별도의 불리언 마스크가 있으면 `compress()`, 거짓인 항목만 골라야 하면 `filterfalse()`가 목적을 분명히 합니다.

## takewhile과 dropwhile의 경계 동작 구분하기

`takewhile(predicate, iterable)`은 처음 조건이 거짓이 되는 원소 직전까지만 반환합니다. `dropwhile()`은 처음 조건이 거짓인 원소까지 버린 뒤에는 조건을 다시 적용하지 않고 나머지를 모두 반환합니다. 둘 다 전체 필터가 아니라 입력 앞부분의 경계를 찾는 도구입니다.

```python
from itertools import dropwhile, takewhile

latencies = [80, 95, 120, 70, 60]

print(list(takewhile(lambda value: value < 100, latencies)))
print(list(dropwhile(lambda value: value < 100, latencies)))
```

```text
[80, 95]
[120, 70, 60]
```

`takewhile()`이 멈추게 만든 첫 실패 원소는 이미 소비되어 결과에 포함되지 않습니다. 그 원소를 이후 처리에 보존해야 한다면 `before_and_after` 같은 별도 분할 함수를 구현하거나 명시적인 반복문으로 경계를 관리합니다. 입력 전체에서 조건에 맞는 항목을 고르려면 `filter()`나 제너레이터 표현식을 사용합니다.

## pairwise와 zip_longest로 이웃과 불균형 입력 다루기

`pairwise(iterable)`은 겹치는 이웃 쌍을 만들어 변화량과 구간을 계산할 때 유용합니다. 입력 원소가 2개보다 적으면 결과는 비어 있습니다. `zip_longest()`는 가장 긴 입력이 끝날 때까지 짝을 만들고, 짧은 입력의 빈자리를 `fillvalue`로 채웁니다.

```python
from itertools import pairwise, zip_longest

readings = [10, 13, 12, 18]
changes = (current - previous for previous, current in pairwise(readings))

names = ['민수', '지수', '서연']
scores = [90, 85]

print(list(changes))
print(list(zip_longest(names, scores, fillvalue='미제출')))
```

```text
[3, -1, 6]
[('민수', 90), ('지수', 85), ('서연', '미제출')]
```

길이가 같아야 하는 입력이라면 빈자리를 채우는 대신 `zip(..., strict=True)`로 불일치를 오류로 만드는 편이 안전합니다. `zip_longest()`에 무한 이터레이터가 포함되면 결과도 끝나지 않으므로 `islice()`나 `takewhile()`로 반드시 제한합니다.
