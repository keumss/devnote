## product, permutations, combinations 중 알맞은 선택 고르기

테스트 입력이나 팀 편성처럼 가능한 선택을 만들 때는 순서가 중요한지, 서로 다른 목록에서 하나씩 고르는지를 먼저 정합니다. `product()`는 각 입력에서 하나씩 고르는 모든 경우이고, `permutations()`는 순서가 있는 선택, `combinations()`는 순서가 없는 선택입니다.

```python
from itertools import combinations, permutations, product

members = ['A', 'B', 'C']

print(list(product(['S', 'M'], ['black', 'white'])))
print(list(permutations(members, 2)))
print(list(combinations(members, 2)))
```

```text
[('S', 'black'), ('S', 'white'), ('M', 'black'), ('M', 'white')]
[('A', 'B'), ('A', 'C'), ('B', 'A'), ('B', 'C'), ('C', 'A'), ('C', 'B')]
[('A', 'B'), ('A', 'C'), ('B', 'C')]
```

`('A', 'B')`와 `('B', 'A')`를 다르게 봐야 하면 `permutations()`, 같은 팀으로 봐야 하면 `combinations()`을 선택합니다. 같은 항목을 여러 번 고를 수 있는 메뉴 조합에는 `combinations_with_replacement()`을 사용합니다.

```python
from itertools import combinations_with_replacement

print(list(combinations_with_replacement(['tea', 'coffee'], 2)))
```

```text
[('tea', 'tea'), ('tea', 'coffee'), ('coffee', 'coffee')]
```

## math.comb와 islice로 조합 폭발 제한하기

조합 함수로 가능한 경우를 탐색하기 전에 결과 수가 감당할 만한지 확인합니다. 결과를 한 개씩 만드는 이터레이터라도 `list()`로 모두 모으면 결과 수만큼 메모리와 시간이 듭니다.

```python
from itertools import combinations, islice
from math import comb

members = list(range(30))

print(comb(len(members), 5))
print(list(islice(combinations(members, 5), 3)))
```

```text
142506
[(0, 1, 2, 3, 4), (0, 1, 2, 3, 5), (0, 1, 2, 3, 6)]
```

미리 보기처럼 일부만 필요하면 `islice()`로 개수를 제한합니다. 순서 있는 선택은 `math.perm()`으로 개수를 확인하고, 실제 탐색에서는 원하는 결과를 찾는 즉시 반복을 끝냅니다.

## tee와 cycle의 저장 비용 피하기

하나의 입력을 두 곳에서 읽어야 할 때 `tee()`를 사용할 수 있지만, 한쪽이 먼저 많이 읽으면 느린 쪽이 따라올 때까지 지난 값이 내부에 저장됩니다. 큰 스트림에서는 한 번의 반복문으로 필요한 결과를 함께 계산하는 편이 메모리를 예측하기 쉽습니다.

```python
from itertools import tee

source = iter([10, 20, 30])
left, right = tee(source)

print(next(left))
print(list(right))
print(list(left))
```

```text
10
[10, 20, 30]
[20, 30]
```

짧은 상태 목록을 반복할 때 `cycle()`도 편리하지만, 첫 번째 순회에서 입력값을 저장한 뒤 되풀이합니다. 큰 파일이나 끝없는 생성기를 `cycle()`에 넘기면 저장 공간이 계속 늘어날 수 있습니다.

```python
from itertools import cycle, islice

statuses = cycle(['대기', '진행', '완료'])
print(list(islice(statuses, 7)))
```

```text
['대기', '진행', '완료', '대기', '진행', '완료', '대기']
```

하나의 값만 되풀이한다면 입력을 저장하지 않는 `repeat()`이 더 알맞습니다. `cycle()`과 `repeat()`은 모두 끝나지 않으므로 `islice()`나 정해진 반복 횟수로 종료 조건을 둡니다.

Python 3.12의 조합 이터레이터와 `tee()`·`cycle()`의 메모리 특성은 [itertools 공식 문서](https://docs.python.org/3.12/library/itertools.html)를 기준으로 확인합니다.
