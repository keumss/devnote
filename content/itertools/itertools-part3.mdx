## groupby는 연속한 같은 키만 묶는다는 점 이해하기

`groupby(iterable, key)`는 키가 바뀔 때마다 새 그룹을 만듭니다. SQL의 `GROUP BY`처럼 떨어져 있는 같은 키를 자동으로 합치지 않으므로, 전체 키별 집계가 목적이면 먼저 같은 `key` 함수로 정렬해야 합니다. 이미 시간순으로 정렬된 로그에서 연속 상태 구간을 찾을 때는 원래 순서 그대로 쓰는 동작이 오히려 유용합니다.

```python
from itertools import groupby
from operator import itemgetter

events = [
    ('u-2', 'login'),
    ('u-1', 'view'),
    ('u-2', 'purchase'),
    ('u-1', 'logout'),
]

for user_id, group in groupby(sorted(events, key=itemgetter(0)), key=itemgetter(0)):
    print(user_id, [kind for _, kind in group])
```

```text
u-1 ['view', 'logout']
u-2 ['login', 'purchase']
```

정렬은 입력 전체를 메모리에 올리고 O(n log n) 시간이 듭니다. 단순 개수 집계라면 `collections.Counter`나 `defaultdict`가 더 직접적입니다. 데이터베이스나 파일이 이미 같은 키 순서로 제공될 때 `groupby()`는 한 그룹씩 처리해 메모리 사용을 제한할 수 있습니다.

## groupby의 그룹 이터레이터를 제때 소비하기

`groupby()`가 반환하는 각 그룹은 바깥 이터레이터와 같은 원본을 공유합니다. 바깥 반복을 다음 그룹으로 진행하면 이전 그룹의 아직 읽지 않은 원소에 다시 접근할 수 없습니다. 그룹을 나중에 써야 한다면 바깥 반복 안에서 리스트처럼 별도 저장소로 구체화합니다.

```python
from itertools import groupby

grouped = groupby('AAABB')
first_key, first_group = next(grouped)
second_key, second_group = next(grouped)

print(first_key, list(first_group))
print(second_key, list(second_group))
```

```text
A []
B ['B', 'B']
```

첫 그룹을 저장하려면 `first_values = list(first_group)`을 호출한 뒤 `next(grouped)`로 이동해야 합니다. 큰 그룹을 모두 리스트로 만들면 지연 처리의 장점이 사라지므로, 가능하면 그룹별 집계나 파일 출력을 그룹 반복문 안에서 끝냅니다.

## tee의 버퍼 비용과 안전 범위 판단하기

`tee(iterable, n=2)`는 하나의 입력에서 독립적으로 진행하는 것처럼 보이는 이터레이터를 만듭니다. 실제로는 앞서가는 복사본이 읽은 값을 뒤처진 복사본을 위해 임시 저장합니다. 두 소비 위치의 차이가 커지면 버퍼도 커지므로 한 복사본을 전부 읽고 다른 복사본을 나중에 읽는 패턴에는 원본을 리스트로 만드는 편이 나을 수 있습니다.

```python
from itertools import tee

source = (number * number for number in range(4))
left, right = tee(source)

print(next(left))
print(next(left))
print(list(right))
print(list(left))
```

```text
0
1
[0, 1, 4, 9]
[4, 9]
```

`tee()`를 만든 뒤에는 원본 이터레이터를 직접 사용하지 않습니다. 또한 같은 `tee` 결과를 여러 스레드에서 동시에 소비하는 것은 안전하지 않아 `RuntimeError`가 날 수 있습니다. 병렬 소비에는 스레드 안전한 큐나 각 소비자용 독립 입력을 설계합니다.

## product와 순열·조합의 결과 크기 예측하기

`product()`는 데카르트 곱, `permutations()`는 순서를 구분하는 배열, `combinations()`는 순서를 구분하지 않는 선택, `combinations_with_replacement()`는 같은 위치를 다시 고를 수 있는 조합을 만듭니다. 이 함수들은 입력 값을 기준이 아니라 위치를 기준으로 고유하게 취급하므로 입력에 중복 값이 있으면 같은 결과 튜플이 반복될 수 있습니다.

```python
from itertools import combinations, combinations_with_replacement, permutations, product

print(list(product('AB', repeat=2)))
print(list(permutations('ABC', 2)))
print(list(combinations('ABC', 2)))
print(list(combinations_with_replacement('AB', 2)))
```

```text
[('A', 'A'), ('A', 'B'), ('B', 'A'), ('B', 'B')]
[('A', 'B'), ('A', 'C'), ('B', 'A'), ('B', 'C'), ('C', 'A'), ('C', 'B')]
[('A', 'B'), ('A', 'C'), ('B', 'C')]
[('A', 'A'), ('A', 'B'), ('B', 'B')]
```

입력 크기가 `n`, 선택 길이가 `r`일 때 결과 수는 각각 `n ** r`, `math.perm(n, r)`, `math.comb(n, r)`, `math.comb(n + r - 1, r)`입니다. 이터레이터라서 결과 리스트를 미리 저장하지는 않지만 모든 경우를 처리하는 시간 자체는 줄지 않습니다. 실행 전 결과 수에 상한을 두고, 필요한 첫 결과만 있다면 조건을 만족하는 순간 반복을 중단합니다.

## 지연 처리도 메모리를 사용할 수 있음을 구분하기

`itertools` 결과가 지연 평가된다고 해서 항상 추가 메모리가 O(1)인 것은 아닙니다. `cycle()`은 반복할 원소를 저장하고, `tee()`는 소비 속도 차이를 저장합니다. `product()`, `permutations()`, `combinations()` 계열은 반복 재사용을 위해 유한한 입력을 내부 튜플로 소비합니다. 반면 `chain()`, `islice()`, `pairwise()` 같은 도구는 대체로 작은 상태만 유지합니다.

```python
from itertools import islice, product

def numbers():
    for number in range(3):
        print(f'입력 소비: {number}')
        yield number

pairs = product(numbers(), ['x', 'y'])
print('product 생성 완료')
print(list(islice(pairs, 2)))
```

```text
입력 소비: 0
입력 소비: 1
입력 소비: 2
product 생성 완료
[(0, 'x'), (0, 'y')]
```

`product()`는 첫 결과를 내기 전에 입력 풀을 완전히 소비하므로 무한 입력에는 사용할 수 없습니다. 성능을 평가할 때는 결과 객체가 이터레이터인지뿐 아니라 입력 선소비, 내부 버퍼, 전체 경우의 수, 하류 단계의 처리 속도를 함께 측정합니다.
