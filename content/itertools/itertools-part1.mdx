## itertools로 반복자를 조합하는 이유

`itertools`는 리스트를 복사하지 않고 이터레이터를 조합해 데이터를 처리할 때 사용합니다. 여러 컬렉션을 하나처럼 순회하거나, 이터레이터 일부만 꺼내거나, 고정 크기 묶음으로 나눌 때 루프와 슬라이싱 대신 쓸 수 있습니다.

```python
import itertools

pages = [['a', 'b'], ['c', 'd'], ['e']]
for item in itertools.chain.from_iterable(pages):
    print(item, end=' ')
```

```text
a b c d e
```

반환값은 이터레이터여서 `list()`로 감싸거나 `for` 루프에 바로 쓸 수 있습니다. 모든 예제는 Python 3.12 기준입니다.

## chain()으로 여러 시퀀스 이어 붙이기

서로 다른 컬렉션을 복사 없이 순서대로 순회할 때 `chain()`을 사용합니다. 왼쪽에서 오른쪽 순서로 각 이터러블의 항목을 하나씩 넘깁니다.

```python
from itertools import chain

headers = ['Content-Type', 'Authorization']
custom = ['X-Request-Id']

for h in chain(headers, custom):
    print(h)
```

```text
Content-Type
Authorization
X-Request-Id
```

전달한 이터러블이 모두 소진되면 반복이 끝납니다. `list(chain(a, b))`는 `a + b`와 같은 결과지만, 원본을 복사하지 않습니다.

## chain.from_iterable()로 중첩 목록 펼치기

이터러블을 담은 이터러블을 한 단계 평탄화할 때 `chain.from_iterable()`을 사용합니다. 개수를 미리 알 수 없는 묶음을 하나의 흐름으로 처리할 때 유용합니다.

```python
from itertools import chain

pages = [['apple', 'banana'], ['cherry'], ['date', 'elderberry']]
items = list(chain.from_iterable(pages))
print(items)
```

```text
['apple', 'banana', 'cherry', 'date', 'elderberry']
```

`chain(*pages)`와 같은 결과지만, `pages`가 이터레이터일 때도 동작합니다. 두 단계 이상의 중첩은 별도 루프나 재귀가 필요합니다.

## islice()로 이터레이터 일부만 꺼내기

인덱스 접근이 없는 이터레이터에서 특정 범위만 꺼낼 때 `islice()`를 사용합니다. 리스트 슬라이싱처럼 `start`, `stop`, `step`을 지정하되, 음수 인덱스는 지원하지 않습니다.

```python
from itertools import islice

def log_lines():
    for i in range(1, 1001):
        yield f'line {i}'

for line in islice(log_lines(), 3, 8):
    print(line)
```

```text
line 4
line 5
line 6
line 7
line 8
```

`islice(it, n)`은 처음 n개만, `islice(it, start, stop)`은 start번째부터 stop 직전까지 꺼냅니다. 이터레이터를 소진하지 않으므로 이후에도 이어서 읽을 수 있습니다.

## takewhile()로 조건이 참인 동안 꺼내기

조건을 만족하는 앞부분만 처리하고 나머지는 버릴 때 `takewhile()`을 사용합니다. 조건이 처음 거짓이 되는 순간 멈추며 이후 항목은 조건과 무관하게 건너뜁니다.

```python
from itertools import takewhile

prices = [980, 750, 1200, 600]
affordable = list(takewhile(lambda p: p < 1000, prices))
print(affordable)
```

```text
[980, 750]
```

`1200`을 만나 조건이 거짓이 되는 순간 멈추므로 이후의 `600`은 조건(`600 < 1000`)을 만족하더라도 결과에 포함되지 않습니다. 만약 첫 번째 항목부터 조건에 맞지 않으면 빈 리스트 `[]`를 반환합니다. 정렬된 데이터에서 범위를 자를 때 효과적이며, 조건에 맞는 모든 항목을 거르려면 `filter()`를 사용합니다.

## dropwhile()로 조건이 참인 동안 건너뛰기

앞부분의 헤더나 초기 상태를 건너뛰고 본문부터 처리할 때 `dropwhile()`을 사용합니다. 조건이 처음 거짓이 되는 항목부터 끝까지 모두 포함합니다.

```python
from itertools import dropwhile

log = ['INFO start', 'INFO loading', 'ERROR timeout', 'INFO retry', 'ERROR fail']
errors_and_after = list(dropwhile(lambda l: not l.startswith('ERROR'), log))
print(errors_and_after)
```

```text
['ERROR timeout', 'INFO retry', 'ERROR fail']
```

조건이 한 번 거짓이 되면 이후 항목은 다시 검사하지 않습니다. `'INFO retry'`가 중간에 나와도 건너뛰지 않습니다.

## batched()로 고정 크기 묶음으로 나누기

데이터를 n개씩 묶어 처리할 때 `batched()`를 사용합니다. Python 3.12에서 추가되었고, 마지막 묶음은 n개보다 적을 수 있습니다.

```python
from itertools import batched

ids = [101, 102, 103, 104, 105, 106, 107]
for batch in batched(ids, 3):
    print(list(batch))
```

```text
[101, 102, 103]
[104, 105, 106]
[107]
```

각 묶음은 튜플로 반환됩니다. API 요청 페이지네이션이나 DB 일괄 삽입처럼 한 번에 처리할 크기를 제한할 때 씁니다.

공식 Python [itertools 문서](https://docs.python.org/3.12/library/itertools.html)의 `chain`, `islice`, `takewhile`, `dropwhile`, `batched` 동작을 기준으로 Python 3.12 예제를 구성했습니다.
