## itertools는 언제 쓰는가?

여러 목록을 차례로 읽거나, 앞의 몇 개만 꺼내거나, 이웃한 값을 비교하는 반복 흐름이 필요할 때 `itertools`를 사용합니다. Python 표준 라이브러리인 이 모듈의 함수는 보통 값을 모두 담은 목록 대신, 필요할 때 다음 값을 만드는 이터레이터를 반환합니다.

```python
from itertools import chain

names = chain(['Mina', 'Jun'], ['Sora'])
print(list(names))
```

```text
['Mina', 'Jun', 'Sora']
```

`list()`는 여기서 결과를 확인하려고만 사용했습니다. 큰 파일이나 생성기 입력은 `for` 문으로 하나씩 처리하면 중간 목록을 만들지 않아도 됩니다.

## iter와 next로 한 번 읽는 입력 이해하기

파일·생성기·`itertools` 함수의 결과처럼 한 번만 읽는 입력을 처리할 때는 이터레이터가 값을 꺼낼수록 진행한다는 점을 알아야 합니다. 같은 이터레이터를 두 번 목록으로 만들면 두 번째 결과는 비어 있습니다.

```python
numbers = iter([10, 20, 30])

print(next(numbers))
print(list(numbers))
print(list(numbers))
```

```text
10
[20, 30]
[]
```

여러 번 순회할 데이터라면 처음부터 리스트를 사용합니다. 한 번만 읽는 입력은 필요한 계산을 한 번의 반복문 안에서 끝내는 편이 안전합니다.

## chain으로 여러 입력을 차례로 읽기

여러 목록을 한 순서로 처리하되 새 목록을 만들고 싶지 않을 때 `chain()`을 사용합니다. 첫 번째 입력이 끝나면 다음 입력으로 자연스럽게 이어집니다.

```python
from itertools import chain

morning = ['로그인', '검색']
afternoon = ['결제']

for event in chain(morning, afternoon):
    print(event)
```

```text
로그인
검색
결제
```

입력이 여러 목록으로 들어 있는 경우에는 `chain.from_iterable()`이 알맞습니다. 바깥 이터러블에서 안쪽 이터러블을 하나씩 받아 평평하게 만듭니다.

```python
from itertools import chain

tags_by_day = [['python', 'web'], ['database'], ['testing']]
print(list(chain.from_iterable(tags_by_day)))
```

```text
['python', 'web', 'database', 'testing']
```

## islice로 이터레이터의 앞부분 읽기

로그처럼 끝이 크거나 알 수 없는 입력에서 앞의 몇 건만 확인할 때 `islice()`를 사용합니다. 시퀀스 슬라이싱과 비슷하지만, 결과를 만들면서 입력도 그만큼 소비합니다.

```python
from itertools import islice

logs = (f'log-{number}' for number in range(1, 1000))
first_three = islice(logs, 3)

print(list(first_three))
```

```text
['log-1', 'log-2', 'log-3']
```

`islice(logs, 2, 5)`처럼 시작과 끝을, `islice(logs, 0, 10, 2)`처럼 건너뛸 간격을 줄 수도 있습니다. 이터레이터는 끝에서 거꾸로 읽을 수 없으므로 음수 인덱스는 지원하지 않습니다.

## pairwise로 이웃한 값 비교하기

오늘과 어제의 온도 차이처럼 연속한 두 값의 관계를 계산할 때 `pairwise()`를 사용합니다. 인덱스를 직접 다루지 않고 `(첫째, 둘째)`, `(둘째, 셋째)`처럼 겹치는 쌍을 만듭니다.

```python
from itertools import pairwise

temperatures = [18, 21, 20, 24]

for yesterday, today in pairwise(temperatures):
    print(today - yesterday)
```

```text
3
-1
4
```

입력이 하나 이하이면 비교할 쌍이 없으므로 아무것도 만들지 않습니다. 짧은 입력을 위한 별도 `if` 문이 필요 없는 이유입니다.

## zip_longest로 누락된 짝 표시하기

이름은 세 명인데 점수는 두 개처럼, 서로 길이가 다른 입력의 빠진 값을 보여야 할 때 `zip_longest()`를 사용합니다. 기본 `zip()`은 짧은 입력이 끝나는 즉시 멈춥니다.

```python
from itertools import zip_longest

users = ['Mina', 'Jun', 'Sora']
scores = [92, 88]

for user, score in zip_longest(users, scores, fillvalue='미입력'):
    print(f'{user}: {score}')
```

```text
Mina: 92
Jun: 88
Sora: 미입력
```

두 입력의 길이가 반드시 같아야 하는 작업에는 `zip(..., strict=True)`로 길이 차이를 오류로 처리합니다. `zip_longest()`는 누락 자체를 결과에 포함할 때 선택합니다.

## count와 repeat로 제한된 반복 패턴 만들기

작업에 번호를 붙이거나 일정한 기본값을 함께 흘려보낼 때 `count()`와 `repeat()`를 사용합니다. 두 함수는 끝없이 값을 만들 수 있으므로, 다른 입력의 끝이나 횟수 제한을 종료 조건으로 함께 둡니다.

```python
from itertools import count, repeat

for number, task in zip(count(1), ['메일', '배포', '점검']):
    print(number, task)

print(list(repeat('retry', 3)))
```

```text
1 메일
2 배포
3 점검
['retry', 'retry', 'retry']
```

`list(count())`처럼 무한 이터레이터를 그대로 목록으로 만들면 프로그램이 끝나지 않습니다. `count()`와 `repeat()`에는 `zip()`, `islice()`, 정해진 반복 횟수 중 하나를 항상 함께 둡니다.

Python 3.12의 `chain()`, `islice()`, `pairwise()`, 무한 이터레이터 규칙은 [itertools 공식 문서](https://docs.python.org/3.12/library/itertools.html)를 기준으로 확인합니다.
