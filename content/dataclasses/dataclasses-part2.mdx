## __post_init__으로 생성 직후 값을 검증하기

여러 필드를 함께 검사하거나 입력을 정규화해야 할 때 `__post_init__()`을 사용합니다. 자동 생성된 `__init__()`이 필드를 넣은 직후 이 메서드를 호출하므로, 유효하지 않은 객체가 만들어지는 일을 초기에 막을 수 있습니다.

```python
from dataclasses import dataclass

@dataclass
class Discount:
    name: str
    rate: float

    def __post_init__(self) -> None:
        self.name = self.name.strip()
        if not self.name:
            raise ValueError('할인 이름은 비어 있을 수 없습니다.')
        if not 0 <= self.rate <= 1:
            raise ValueError('할인율은 0과 1 사이여야 합니다.')

discount = Discount(' 신규 가입 ', 0.15)
print(discount)
```

```text
Discount(name='신규 가입', rate=0.15)
```

검증 실패는 생성자에서 `ValueError`로 바로 알리는 편이 좋습니다. 이후 코드가 매번 `rate` 범위를 다시 확인하는 대신, `Discount` 인스턴스는 유효하다는 계약을 사용할 수 있습니다.

## field(init=False)로 계산 필드 채우기

다른 필드에서 항상 계산되는 값은 호출자가 넣지 못하게 `field(init=False)`로 선언합니다. `__post_init__()`에서 계산하면 생성자 입력과 파생 결과를 분리할 수 있습니다.

```python
from dataclasses import dataclass, field

@dataclass
class LineItem:
    unit_price: int
    quantity: int
    subtotal: int = field(init=False)

    def __post_init__(self) -> None:
        if self.unit_price < 0 or self.quantity < 1:
            raise ValueError('가격은 0 이상이고 수량은 1 이상이어야 합니다.')
        self.subtotal = self.unit_price * self.quantity

item = LineItem(12000, 3)
print(item)
```

```text
LineItem(unit_price=12000, quantity=3, subtotal=36000)
```

`subtotal`은 `LineItem(12000, 3, 1)`처럼 임의 값으로 만들 수 없습니다. 원본 가격이나 수량을 나중에 바꾸는 설계라면 파생 값을 다시 계산하는 규칙까지 함께 정하거나, 불변 객체와 `replace()`를 선택합니다.

## InitVar로 초기화에만 필요한 입력 받기

생성 중에만 쓰고 인스턴스 상태로 보관하지 않을 값은 `InitVar`로 받습니다. 예를 들어 원본 비밀번호를 받아 해시만 보관하면, 평문 입력이 데이터 모델의 필드가 되지 않습니다.

```python
from dataclasses import InitVar, dataclass, field
import hashlib

@dataclass
class Account:
    email: str
    password: InitVar[str]
    password_digest: str = field(init=False)

    def __post_init__(self, password: str) -> None:
        if len(password) < 8:
            raise ValueError('비밀번호는 8자 이상이어야 합니다.')
        self.password_digest = hashlib.sha256(password.encode()).hexdigest()

account = Account('user@example.com', 'safe-pass-1')
print(account.email)
print(len(account.password_digest))
```

```text
user@example.com
64
```

실제 비밀번호 저장에는 목적에 맞는 느린 비밀번호 해시 알고리즘과 검증 라이브러리를 사용해야 합니다. 여기서의 핵심은 `InitVar`가 `__init__()`과 `__post_init__()` 사이의 입력 통로이며, 일반 dataclass 필드가 아니라는 점입니다.

## ClassVar로 인스턴스 데이터와 상수를 구분하기

모든 인스턴스가 공유하는 상수는 `ClassVar`로 표시합니다. dataclass는 이를 생성자·표현·비교에서 제외하므로, 인스턴스에 넣을 데이터와 클래스 전체의 정책을 구분할 수 있습니다.

```python
from dataclasses import dataclass
from typing import ClassVar

@dataclass
class Coupon:
    code: str
    rate: int

    maximum_rate: ClassVar[int] = 50

coupon = Coupon('WELCOME', 10)
print(coupon)
print(Coupon.maximum_rate)
```

```text
Coupon(code='WELCOME', rate=10)
50
```

`maximum_rate`는 `Coupon('WELCOME', 10, 50)`의 인자가 아닙니다. 인스턴스마다 다른 값이어야 한다면 `ClassVar`가 아니라 일반 필드로 선언하고 생성·검증 규칙을 정합니다.

## asdict로 중첩된 데이터를 경계 밖으로 내보내기

dataclass 데이터를 JSON 응답이나 직렬화 함수에 넘길 딕셔너리 형태로 바꿔야 할 때 `asdict()`를 사용합니다. 중첩 dataclass와 목록·딕셔너리·튜플도 재귀적으로 변환합니다.

```python
from dataclasses import asdict, dataclass

@dataclass
class Address:
    city: str
    street: str

@dataclass
class Order:
    order_id: str
    address: Address
    tags: list[str]

order = Order('ORD-9', Address('서울', '세종대로'), ['gift'])
print(asdict(order))
```

```text
{'order_id': 'ORD-9', 'address': {'city': '서울', 'street': '세종대로'}, 'tags': ['gift']}
```

`asdict()`는 단순한 얕은 복사가 아닙니다. 다른 객체에는 `copy.deepcopy()`를 사용하므로, 큰 객체 그래프나 파일 핸들처럼 복사하면 안 되는 값을 처리하는 일반 변환기로 쓰지 않습니다. JSON으로 보내려면 `datetime`, `Decimal`, `Path` 같은 남은 값의 표현 규칙도 경계에서 정합니다.

## replace로 변경된 새 인스턴스 만들기

이전 값을 보존하면서 일부 필드만 바꾼 결과가 필요할 때 `replace()`를 사용합니다. 특히 불변 데이터 모델에서 상태 전이를 표현하기 좋고, 새 객체도 일반 생성 과정을 거치므로 `__post_init__()` 검증이 다시 실행됩니다.

```python
from dataclasses import dataclass, replace

@dataclass(frozen=True)
class Reservation:
    code: str
    status: str

    def __post_init__(self) -> None:
        if self.status not in {'pending', 'confirmed', 'cancelled'}:
            raise ValueError('알 수 없는 예약 상태입니다.')

pending = Reservation('R-12', 'pending')
confirmed = replace(pending, status='confirmed')

print(pending)
print(confirmed)
```

```text
Reservation(code='R-12', status='pending')
Reservation(code='R-12', status='confirmed')
```

`replace()`는 원본을 수정하지 않습니다. 다만 `init=False` 필드는 원본에서 복사되지 않고 초기화 과정에서 다시 만들어질 수 있으므로, 그런 필드를 가진 모델의 복사 규칙은 별도 생성자로 명확히 하는 편이 안전합니다.

## kw_only로 확장 가능한 생성자 만들기

위치 인자 두 개는 짧고 안정적인 입력으로 유지하면서, 나중에 늘어날 옵션은 이름으로만 받게 하려면 `field(kw_only=True)`를 사용합니다. 호출 지점에서 값의 뜻을 드러내고 필드 추가에 더 안전한 생성자 계약을 만듭니다.

```python
from dataclasses import dataclass, field

@dataclass
class Payment:
    amount: int
    method: str
    currency: str = field(default='KRW', kw_only=True)

print(Payment(12000, 'card'))
print(Payment(12000, 'card', currency='USD'))
```

```text
Payment(amount=12000, method='card', currency='KRW')
Payment(amount=12000, method='card', currency='USD')
```

`Payment(12000, 'card', 'USD')`는 실패합니다. 모든 필드를 키워드 전용으로 만들고 싶다면 `@dataclass(kw_only=True)`를 사용합니다. 키워드 전용 필드는 위치 기반 패턴 매칭 인자에도 포함되지 않습니다.

Python 3.12의 `__post_init__()`, `InitVar`, `ClassVar`, `asdict()`, `replace()` 동작은 [dataclasses 공식 문서](https://docs.python.org/3.12/library/dataclasses.html)를 기준으로 확인합니다.
