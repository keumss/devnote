## 입력과 결과의 계약을 dataclass로 표현하기

이번 예제는 JSON 주문 항목을 받아 배송비 포함 견적을 출력하는 `shipping_quote.py`를 만듭니다. 항목은 SKU·단가·수량을, 결과는 소계·배송비·합계·무료 배송 여부를 가집니다. 금액은 부동소수점 오차를 피하기 위해 `Decimal`로 다룹니다.

먼저 이 파일의 데이터 모델을 작성합니다. `LineItem`은 만들어지는 시점에 유효성을 검사하고, 한 줄의 금액을 스스로 계산합니다.

```python
# shipping_quote.py
from dataclasses import dataclass
from decimal import Decimal

@dataclass(frozen=True, slots=True)
class LineItem:
    sku: str
    unit_price: Decimal
    quantity: int

    def __post_init__(self) -> None:
        if not self.sku:
            raise ValueError('SKU는 비어 있을 수 없습니다.')
        if not self.unit_price.is_finite() or self.unit_price < 0:
            raise ValueError('단가는 0 이상의 유한한 금액이어야 합니다.')
        if self.quantity < 1:
            raise ValueError('수량은 1 이상이어야 합니다.')

    @property
    def amount(self) -> Decimal:
        return self.unit_price * self.quantity

@dataclass(frozen=True, slots=True)
class ShippingQuote:
    subtotal: Decimal
    shipping: Decimal
    total: Decimal
    free_shipping: bool
```

모델은 `dict`나 JSON을 모르고 이미 검증된 값을 받습니다. 이 분리 덕분에 계산 함수는 외부 입력의 누락·타입 문제 대신 `LineItem`이라는 명확한 계약에 집중할 수 있습니다.

## 외부 dict를 LineItem으로 변환하기

JSON에서 읽은 값은 신뢰하지 않고 파일 경계에서 검사합니다. 다음 코드를 같은 `shipping_quote.py`에 이어 추가합니다. 목록 형식, 필수 키, 문자열 SKU, 정수 수량, `Decimal`로 해석할 수 있는 단가를 확인한 뒤 모델을 만듭니다.

```python
from collections.abc import Mapping
from decimal import InvalidOperation

def parse_price(value: object) -> Decimal:
    if isinstance(value, bool) or not isinstance(value, (int, str)):
        raise ValueError('unit_price는 정수 또는 문자열 금액이어야 합니다.')
    try:
        price = Decimal(str(value))
    except InvalidOperation as error:
        raise ValueError('unit_price는 금액으로 변환할 수 있어야 합니다.') from error
    if not price.is_finite():
        raise ValueError('unit_price는 유한한 금액이어야 합니다.')
    return price

def parse_item(raw: object) -> LineItem:
    if not isinstance(raw, Mapping):
        raise ValueError('각 항목은 객체여야 합니다.')

    try:
        sku = raw['sku']
        unit_price = raw['unit_price']
        quantity = raw['quantity']
    except KeyError as error:
        raise ValueError(f'필수 항목이 없습니다: {error.args[0]}') from error

    if not isinstance(sku, str):
        raise ValueError('sku는 문자열이어야 합니다.')
    if isinstance(quantity, bool) or not isinstance(quantity, int):
        raise ValueError('quantity는 정수여야 합니다.')
    return LineItem(sku.strip(), parse_price(unit_price), quantity)

def parse_items(raw: object) -> tuple[LineItem, ...]:
    if not isinstance(raw, Mapping):
        raise ValueError('주문은 객체여야 합니다.')
    items = raw.get('items')
    if not isinstance(items, list) or not items:
        raise ValueError('items는 비어 있지 않은 목록이어야 합니다.')
    return tuple(parse_item(item) for item in items)
```

`parse_item()`은 경계에서만 `Mapping`을 다룹니다. 여기서 `LineItem`을 반환한 뒤에는 다른 함수가 딕셔너리 키의 오타나 JSON 타입을 다시 걱정하지 않아도 됩니다. 금액은 JSON 숫자 `float`를 허용하지 않아, 이미 반올림된 이진 부동소수점 값이 조용히 들어오는 일을 막습니다.

## 견적 규칙을 순수 함수로 계산하기

다음으로 같은 파일에 배송 규칙을 더합니다. 50,000원 이상이면 무료 배송, 그 미만이면 3,000원을 부과하며, 이 함수는 파일·표준 입출력·JSON에 의존하지 않습니다.

```python
FREE_SHIPPING_THRESHOLD = Decimal('50000')
SHIPPING_FEE = Decimal('3000')

def calculate_quote(items: tuple[LineItem, ...]) -> ShippingQuote:
    subtotal = sum((item.amount for item in items), start=Decimal('0'))
    free_shipping = subtotal >= FREE_SHIPPING_THRESHOLD
    shipping = Decimal('0') if free_shipping else SHIPPING_FEE

    return ShippingQuote(
        subtotal=subtotal,
        shipping=shipping,
        total=subtotal + shipping,
        free_shipping=free_shipping,
    )
```

`sum()`의 `start=Decimal('0')`은 항목이 없을 때도 금액 타입을 유지하게 합니다. 이 예제에서는 `parse_items()`가 빈 주문을 막지만, 계산 함수가 독립적으로 읽힐 수 있도록 시작값도 계약에 맞춥니다.

## JSON 입출력과 실패를 경계에서 처리하기

이제 프로그램의 맨 아래에 입력·출력과 종료 정책을 추가합니다. 성공한 견적은 표준 출력에 JSON으로 쓰고, 잘못된 JSON이나 검증 오류는 표준 오류에 짧게 쓰며 상태 코드 1을 반환합니다.

```python
import json
import sys

def render_quote(quote: ShippingQuote) -> dict[str, str | bool]:
    return {
        'subtotal': str(quote.subtotal),
        'shipping': str(quote.shipping),
        'total': str(quote.total),
        'free_shipping': quote.free_shipping,
    }

def main() -> int:
    try:
        raw = json.load(sys.stdin)
        quote = calculate_quote(parse_items(raw))
    except (OSError, TypeError, ValueError) as error:
        print(f'shipping-quote: {error}', file=sys.stderr)
        return 1

    print(json.dumps(render_quote(quote), ensure_ascii=False, sort_keys=True))
    return 0

if __name__ == '__main__':
    raise SystemExit(main())
```

`Decimal`은 기본 JSON 인코더가 직접 처리할 수 없으므로, 외부 표현을 문자열로 고정했습니다. 내부 계산의 `Decimal` 계약과 HTTP·파일·CLI 같은 외부 경계의 JSON 계약을 같은 타입으로 섞지 않는 것이 핵심입니다.

## 실행 결과로 무료 배송 규칙 확인하기

앞의 코드 조각을 순서대로 합친 `shipping_quote.py`에 주문 JSON을 표준 입력으로 전달합니다. 소계가 54,000원이므로 배송비는 0이고 합계도 54,000원입니다.

```bash
printf '%s\n' '{"items":[{"sku":"A-1","unit_price":"12000","quantity":2},{"sku":"B-2","unit_price":"30000","quantity":1}]}' | python shipping_quote.py
```

```text
{"free_shipping": true, "shipping": "0", "subtotal": "54000", "total": "54000"}
```

입력이 `{"items": []}`이면 정상 결과를 만들지 않고 `shipping-quote: items는 비어 있지 않은 목록이어야 합니다.`를 표준 오류에 쓴 뒤 상태 코드 1로 끝납니다. 이처럼 정상 출력과 실패 진단을 분리하면 다음 프로그램이 성공한 JSON만 안정적으로 읽을 수 있습니다.

## pytest로 계산과 경계 계약 고정하기

마지막으로 계산 규칙과 외부 입력 검증을 작은 테스트로 고정합니다. 파일을 새로 읽지 않는 `calculate_quote()`는 객체만 만들어 바로 검사하고, 입력 경계는 `parse_items()`에 잘못된 데이터를 넣어 확인합니다.

```python
# test_shipping_quote.py
from decimal import Decimal

import pytest

from shipping_quote import LineItem, calculate_quote, parse_items

def test_applies_shipping_below_threshold() -> None:
    quote = calculate_quote((LineItem('A-1', Decimal('12000'), 2),))

    assert quote.subtotal == Decimal('24000')
    assert quote.shipping == Decimal('3000')
    assert quote.total == Decimal('27000')
    assert quote.free_shipping is False

def test_rejects_missing_item_key() -> None:
    with pytest.raises(ValueError, match='quantity'):
        parse_items({'items': [{'sku': 'A-1', 'unit_price': '12000'}]})

def test_rejects_float_price() -> None:
    with pytest.raises(ValueError, match='정수 또는 문자열'):
        parse_items({'items': [{'sku': 'A-1', 'unit_price': 12.5, 'quantity': 1}]})
```

테스트는 무료 배송 경계 바로 아래의 결과와, 외부 입력이 모델까지 들어오기 전에 거절되는 규칙을 확인합니다. 임계값 이상, 음수 단가, 0 수량, 잘못된 JSON과 표준 오류·상태 코드도 제품의 요구에 맞춰 추가합니다.

Python 3.12의 `@dataclass(frozen=True, slots=True)`, `__post_init__()`, `Decimal` API는 [dataclasses 공식 문서](https://docs.python.org/3.12/library/dataclasses.html)와 [decimal 공식 문서](https://docs.python.org/3.12/library/decimal.html)를 기준으로 확인합니다.
