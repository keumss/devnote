## 문서 생성 규칙을 함수로 분리하기

프로젝트 구조를 README에 반영하는 코드는 출력 규칙과 파일 쓰기를 분리하면 테스트하기 쉽습니다. 아래 `make_tree()`는 입력 경로를 받아 문자열만 만들고, 호출자가 어디에 기록할지 결정합니다. 기본 제외 목록과 정렬을 한곳에 모아 팀의 문서 형식을 일관되게 유지합니다.

```python
from pathlib import Path

import seedir as sd

EXCLUDED_FOLDERS = ['.git', '.venv', '__pycache__', 'node_modules']
EXCLUDED_FILES = ['.env', '.env.local', 'STRUCTURE.md', 'STRUCTURE.md.tmp']

def make_tree(root: Path) -> str:
    return sd.seedir(
        root,
        printout=False,
        sort=True,
        first='folders',
        depthlimit=3,
        beyond='content',
        exclude_folders=EXCLUDED_FOLDERS,
        exclude_files=EXCLUDED_FILES,
    )

print(make_tree(Path('sample-project')))
```

```text
sample-project/
├─docs/
│ ├─guide.md
│ └─logo.png
├─src/
│ ├─data/
│ │ ├─products.csv
│ │ └─users.csv
│ ├─__init__.py
│ └─api.py
├─tests/
│ └─test_api.py
├─README.md
└─pyproject.toml
```

Part 1의 원본과 비교하면 `.git/`과 `.env`가 빠졌고 나머지 구조는 정렬되어 있습니다. 생성 대상인 `STRUCTURE.md`와 임시 파일도 제외해야 두 번째 실행부터 결과가 자기 자신을 포함하는 문제를 막을 수 있습니다. 제외 목록은 프로젝트의 비밀 관리 정책에 맞춰 인증서나 고객 데이터 경로까지 보완합니다.

## 원자적으로 문서 스냅샷 갱신하기

출력 파일을 바로 덮어쓰다가 중단되면 빈 문서나 일부 결과가 남을 수 있습니다. 같은 디렉터리에 임시 파일을 쓴 뒤 `replace()`로 교체하면 대부분의 로컬 파일 시스템에서 독자가 완성본 또는 이전 파일 중 하나를 보게 할 수 있습니다.

```python
from pathlib import Path

def write_snapshot(root: Path, destination: Path) -> None:
    tree = make_tree(root)
    content = f'# 프로젝트 구조\n\n```text\n{tree}\n```\n'
    temporary = destination.with_suffix(destination.suffix + '.tmp')

    try:
        temporary.write_text(content, encoding='utf-8')
        temporary.replace(destination)
    except OSError as error:
        temporary.unlink(missing_ok=True)
        raise RuntimeError(f'구조 스냅샷을 쓸 수 없습니다: {error}') from error
```

대상 파일과 임시 파일은 같은 파일 시스템에 두어야 `replace()`의 동작이 예측 가능합니다. CI에서 이 함수를 실행할 때는 결과가 의도치 않게 커밋되지 않도록 변경 사항을 검사하고, 오류 메시지에 트리 전체나 비밀 값을 포함하지 않습니다.

## 임시 디렉터리로 출력 계약 테스트하기

테스트는 실제 저장소 대신 `pytest`의 `tmp_path`에 작은 트리를 만들면 독립적입니다. 정렬, 제외 규칙, 깊이 제한처럼 문서의 공개 계약인 부분을 검사합니다. `tmp_path`는 `pathlib.Path`이므로 `seedir`의 Path 지원도 함께 사용합니다.

```python
from pathlib import Path

from structure_snapshot import make_tree

def test_make_tree_excludes_private_and_sorts(tmp_path: Path) -> None:
    (tmp_path / 'z-last.py').write_text('', encoding='utf-8')
    (tmp_path / 'a-first.py').write_text('', encoding='utf-8')
    (tmp_path / '.git').mkdir()
    (tmp_path / '.git' / 'config').write_text('private', encoding='utf-8')

    tree = make_tree(tmp_path)

    assert '.git' not in tree
    assert tree.index('a-first.py') < tree.index('z-last.py')
```

정확한 공백과 선 문자 전체를 비교하는 스냅샷 테스트는 스타일 변경에도 실패할 수 있습니다. 그 형식이 계약이라면 전체 비교가 맞지만, 핵심 정책만 지키면 되는 경우에는 위처럼 포함·제외와 순서를 검사해 테스트를 덜 취약하게 만듭니다.

## CLI와 CI에서 실패를 명확히 처리하기

명령줄 진입점은 경로 존재 여부와 디렉터리 여부를 먼저 확인하고, 실패하면 표준 오류에 짧은 원인을 남긴 뒤 0이 아닌 상태 코드로 끝냅니다. 저장소 루트를 인자로 받게 하면 개발자 컴퓨터와 CI에서 같은 도구를 재사용할 수 있습니다.

```python
import sys
from pathlib import Path

def main(argv: list[str] | None = None) -> int:
    args = sys.argv[1:] if argv is None else argv
    root = Path(args[0]) if args else Path('.')

    if not root.is_dir():
        print(f'디렉터리가 아닙니다: {root}', file=sys.stderr)
        return 2

    destination = root / 'STRUCTURE.md'
    try:
        write_snapshot(root, destination)
    except RuntimeError as error:
        print(error, file=sys.stderr)
        return 1

    print(f'구조 스냅샷 생성: {destination}')
    return 0

if __name__ == '__main__':
    raise SystemExit(main())
```

정상 경로와 잘못된 경로를 각각 실행하면 성공 메시지와 오류를 구분할 수 있습니다.

```bash
python structure_snapshot.py sample-project
python structure_snapshot.py missing-project
```

```text
구조 스냅샷 생성: sample-project/STRUCTURE.md
디렉터리가 아닙니다: missing-project
```

첫 명령은 상태 코드 0으로 끝나고, 두 번째 명령은 오류를 표준 오류에 출력한 뒤 상태 코드 2로 끝납니다. CI에서는 생성 후 `git diff --exit-code -- STRUCTURE.md`로 문서 갱신 누락을 발견할 수 있습니다. 자동 커밋·푸시는 팀 정책에 따라 별도 승인합니다.
