## FakeDir로 파일을 만들지 않고 구조 설계하기

`FakeDir`는 디스크에 쓰지 않는 폴더 객체입니다. 아직 만들지 않은 프로젝트 구조를 설계 문서에 보이거나, 테스트용 트리를 준비할 때 실제 임시 파일보다 간단합니다. `create_file()`로 파일 항목을 추가하고 `seedir()` 메서드로 같은 형식의 트리를 얻습니다.

```python
import seedir as sd

root = sd.FakeDir('inventory-api')
root.create_file(['pyproject.toml', 'README.md'])

src = sd.FakeDir('src', parent=root)
src.create_file(['main.py', 'settings.py'])

print(root.seedir(sort=True, first='folders', printout=False))
```

출력은 실제 파일을 만들지 않고도 설계한 구조를 보여 줍니다.

```text
inventory-api/
├─src/
│ ├─main.py
│ └─settings.py
├─README.md
└─pyproject.toml
```

이 작업은 실제 `inventory-api` 폴더를 생성하지 않습니다. 따라서 설계 초안과 단위 테스트에 적합하지만, 배포 전에 실제 파일 시스템의 권한·대소문자 구분·경로 길이까지 검증하려면 별도의 통합 테스트가 필요합니다.

## 부모 관계와 경로 인덱스로 트리 수정하기

가짜 항목은 `parent`를 바꿔 트리 안에서 옮길 수 있고, 루트는 경로 같은 문자열로 자식에 접근할 수 있습니다. `delete()`에는 항목 이름 또는 객체를 넘겨 제거합니다. 이런 조작은 문서의 예시 구조를 단계적으로 발전시키기에 좋습니다.

```python
import seedir as sd

root = sd.FakeDir('app')
root.create_file('config.py')
package = sd.FakeDir('app', parent=root)
package.create_file('main.py')

print('[변경 전]')
print(root.seedir(printout=False, sort=True))

root['config.py'].parent = package
package.delete('main.py')

print('\n[변경 후]')
print(root.seedir(printout=False, sort=True))
```

```text
[변경 전]
app/
├─app/
│ └─main.py
└─config.py

[변경 후]
app/
└─app/
  └─config.py
```

변경 전에는 `config.py`가 루트에 있고 `main.py`가 패키지 안에 있습니다. 부모를 바꾸고 삭제한 뒤에는 `config.py`만 안쪽 `app/`으로 이동했습니다. 동일한 이름이 여러 위치에 있으면 `root['app/config.py']`처럼 루트 기준 경로로 접근하고, 변경 전후를 출력해 부모 관계를 검증합니다.

## 실제 디렉터리를 fakedir로 읽어 독립적으로 편집하기

`sd.fakedir()`는 현재 디렉터리를 `FakeDir` 트리로 읽어 옵니다. 읽어 온 뒤의 `delete()`나 부모 변경은 메모리의 가짜 트리에만 적용되므로, 원본 파일 시스템을 바꾸지 않고 문서용 구조를 다듬을 수 있습니다.

```python
from pathlib import Path

import seedir as sd

snapshot = sd.fakedir(
    Path('sample-project'),
    exclude_folders=['.git', '.venv'],
    exclude_files='.env',
)

print('[메모리로 읽은 직후]')
print(snapshot.seedir(printout=False, sort=True, first='folders', depthlimit=1))

snapshot.delete('README.md')

print('\n[README.md 삭제 후]')
print(snapshot.seedir(printout=False, sort=True, first='folders', depthlimit=1))
```

```text
[메모리로 읽은 직후]
sample-project/
├─docs/
├─src/
├─tests/
├─README.md
└─pyproject.toml

[README.md 삭제 후]
sample-project/
├─docs/
├─src/
├─tests/
└─pyproject.toml
```

`.git/`과 `.env`는 읽는 단계에서 제외되어 처음부터 없고, `delete()` 뒤에는 메모리 트리의 `README.md`만 사라집니다. 디스크의 실제 `README.md`는 그대로입니다. `fakedir()`도 원본을 읽을 때는 권한 오류와 순회 비용의 영향을 받으므로, 민감한 이름은 읽기 단계부터 제외하고 대형 프로젝트에는 깊이·항목 한도를 적용합니다.

## 텍스트 트리를 읽고 형식을 통일하기

기존 문서에 있는 들여쓰기 기반 트리는 `fakedir_fromstring()`으로 `FakeDir`로 바꿀 수 있습니다. 파싱한 뒤 `seedir()`를 호출하면 팀에서 정한 스타일과 정렬 규칙으로 다시 렌더링할 수 있습니다.

```python
import seedir as sd

outline = '''service/
  - src/
    - main.py
  - tests/
    - test_main.py'''

tree = sd.fakedir_fromstring(outline)
print(tree.seedir(style='spaces', indent=2, sort=True, printout=False))
```

변환 뒤에는 들여쓰기 기반 문자열을 같은 공백 스타일로 다시 출력할 수 있습니다.

```text
service/
  src/
    main.py
  tests/
    test_main.py
```

텍스트 다이어그램 파서는 모든 인터넷 예제 형식을 완벽하게 해석한다고 보장되지 않습니다. 주석을 가진 입력에는 `parse_comments=True`를 고려하고, 중요한 구조라면 변환 결과와 원문을 비교하는 테스트를 둡니다. 신뢰할 수 없는 텍스트를 실제 경로로 실행하거나 파일 생성에 바로 연결하지 않는 것도 원칙입니다.
