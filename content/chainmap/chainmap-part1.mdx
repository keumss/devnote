## ChainMap으로 설정 우선순위 만들기

기본 설정 위에 사용자 설정을 덮어 읽을 때 `ChainMap`을 사용합니다. 여러 매핑을 복사해 합치지 않고 하나의 사전처럼 조회하며, 앞에 둔 매핑의 값이 우선합니다.

```python
from collections import ChainMap


defaults = {'theme': 'light', 'page_size': 20}
user_settings = {'theme': 'dark'}
settings = ChainMap(user_settings, defaults)

print(settings['theme'])
print(settings['page_size'])
```

```text
dark
20
```

생성자에 전달한 순서가 조회 우선순위입니다. `user_settings`를 앞에 두었으므로 같은 `theme` 키에서는 사용자 값이 선택됩니다.

## get으로 없는 설정에 기본값 주기

선택 설정이 없을 수 있을 때는 사전과 같이 `get()`으로 대체값을 줍니다. 어느 매핑에도 키가 없을 때만 두 번째 인수가 반환됩니다.

```python
from collections import ChainMap


settings = ChainMap({'theme': 'dark'}, {'page_size': 20})

print(settings.get('language', 'ko'))
print(settings.get('page_size', 10))
```

```text
ko
20
```

필수 설정이라면 `settings['key']`로 읽어 누락 시 `KeyError`를 받습니다. 조용히 대체해도 되는 값에만 `get()`의 기본값을 둡니다.

## maps로 실제 설정 사전 확인하기

어떤 계층에서 값이 왔는지 확인하거나 원본 사전을 다뤄야 할 때 `maps`를 사용합니다. `maps[0]`은 가장 먼저 조회되는 현재 계층이고, 뒤로 갈수록 우선순위가 낮습니다.

```python
from collections import ChainMap


runtime = {'debug': True}
defaults = {'debug': False, 'page_size': 20}
settings = ChainMap(runtime, defaults)

print(settings.maps[0])
print(settings.maps[1])
```

```text
{'debug': True}
{'debug': False, 'page_size': 20}
```

`maps`는 공개된 목록이지만, 조회 순서를 바꾸는 수정은 설정 해석 전체에 영향을 줍니다. 보통은 읽기·진단 용도로 사용합니다.

## 원본 사전 변경을 즉시 반영하기

실행 중에 바뀌는 설정을 복사본 없이 반영해야 할 때 `ChainMap`은 원본 매핑을 참조합니다. 원본 사전을 바꾸면 이미 만든 `ChainMap` 조회 결과도 바뀝니다.

```python
from collections import ChainMap


runtime = {}
defaults = {'timeout': 3}
settings = ChainMap(runtime, defaults)

runtime['timeout'] = 10
print(settings['timeout'])
```

```text
10
```

스냅샷이 필요하다면 `dict(settings)`로 평탄한 새 사전을 만듭니다. 그 뒤 원본을 바꿔도 새 사전은 변하지 않습니다.

## dict로 최종 설정값 평탄화하기

여러 계층을 해석한 결과를 함수 밖으로 전달하거나 로그에 남길 때 `dict()`로 일반 사전을 만듭니다. 같은 키는 조회 시 선택되는 가장 앞쪽 값을 하나만 남깁니다.

```python
from collections import ChainMap


settings = ChainMap(
    {'theme': 'dark'},
    {'theme': 'light', 'page_size': 20},
)

print(dict(settings))
```

```text
{'theme': 'dark', 'page_size': 20}
```

`dict(settings)`는 복사 결과입니다. 이후에도 설정 계층을 유지하며 조회해야 한다면 `ChainMap` 자체를 전달합니다.

## 키 반복 순서와 조회 우선순위 구분하기

설정을 화면에 나열할 때 키 반복 순서가 조회 우선순위와 같다고 가정하지 않습니다. `ChainMap`은 뒤쪽 매핑부터 키를 훑어, 마지막 매핑부터 `update()`한 일반 사전과 같은 키 순서를 만듭니다.

```python
from collections import ChainMap


defaults = {'theme': 'light', 'page_size': 20}
user_settings = {'theme': 'dark', 'font_size': 16}
settings = ChainMap(user_settings, defaults)

print(list(settings))
```

```text
['theme', 'page_size', 'font_size']
```

값 조회는 여전히 앞쪽 `user_settings`가 우선입니다. 표시 순서가 중요하면 필요한 순서를 별도로 정해 출력합니다.

공식 Python [collections.ChainMap 문서](https://docs.python.org/3.12/library/collections.html#collections.ChainMap)의 조회 순서, `maps`, 참조 동작을 기준으로 Python 3.12 예제를 구성했습니다.
