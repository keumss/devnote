## SQLModel이 줄여 주는 모델 중복 이해하기

SQLModel은 SQLAlchemy의 테이블 매핑과 Pydantic의 데이터 모델 기능을 함께 사용할 수 있게 합니다. `table=True`인 클래스는 DB 테이블에 대응하고, `table=False`인 기본 SQLModel 클래스는 요청·응답처럼 DB에 직접 저장하지 않는 데이터 형식을 표현합니다.

하나의 클래스를 모든 계층에 그대로 노출하는 것이 목표는 아닙니다. 예를 들어 비밀 이름을 입력에서는 받되 공개 응답에서는 숨겨야 한다면 테이블 모델과 API 모델을 분리해야 합니다. 공통 필드만 상속해 중복을 줄이는 방식이 실전에서 안전합니다.

## 공통 필드와 테이블 모델 분리하기

기본 클래스에는 여러 모델이 공유할 필드를 두고, DB 전용 클래스에 기본 키와 `table=True`를 추가합니다. 생성 입력은 클라이언트가 ID를 임의로 지정하지 못하도록 별도 클래스로 둡니다.

```python
from sqlmodel import Field, SQLModel

class HeroBase(SQLModel):
    name: str = Field(index=True, min_length=1, max_length=50)
    age: int | None = Field(default=None, ge=0)

class Hero(HeroBase, table=True):
    id: int | None = Field(default=None, primary_key=True)
    secret_name: str

class HeroCreate(HeroBase):
    secret_name: str

payload = HeroCreate(name='Deadpond', secret_name='Dive Wilson')
print(payload.model_dump())
```

```text
{'name': 'Deadpond', 'age': None, 'secret_name': 'Dive Wilson'}
```

`id`는 DB가 생성하므로 생성 입력에 없고, 선택적인 `age`는 기본값 `None`으로 나타납니다. `min_length`, `max_length`, `ge`는 API 경계의 잘못된 입력을 일찍 발견하게 하지만 DB 제약도 별도로 설계해야 합니다.

## 엔진 연결 후 생성된 테이블 확인하기

학습 예제는 별도 서버 없이 실행할 수 있는 SQLite를 사용합니다. 모델을 모두 가져온 뒤 `SQLModel.metadata.create_all()`을 호출해야 메타데이터에 등록된 테이블이 생성됩니다.

```python
from sqlalchemy import inspect
from sqlmodel import SQLModel, create_engine

engine = create_engine('sqlite:///database.db', echo=False)
SQLModel.metadata.create_all(engine)

print(inspect(engine).get_table_names())
```

```text
['hero']
```

기본 테이블 이름은 클래스 이름을 소문자로 바꾼 `hero`입니다. `create_all()`은 없는 테이블을 만드는 학습·테스트 용도에는 편하지만 기존 운영 테이블의 컬럼을 안전하게 변경하지는 않습니다. 운영 스키마 변경에는 Alembic 리비전을 사용합니다.

## Session.add와 refresh로 생성 결과 확인하기

입력 모델을 테이블 객체로 바꾸고 세션에 추가합니다. `commit()`이 트랜잭션을 확정하고, `refresh()`는 DB가 만든 기본 키 같은 값을 객체에 다시 읽어 옵니다.

```python
from sqlmodel import Session

hero_data = HeroCreate(name='Deadpond', secret_name='Dive Wilson')
hero = Hero.model_validate(hero_data)

with Session(engine) as session:
    session.add(hero)
    session.commit()
    session.refresh(hero)

    print(hero.id, hero.name, hero.age)
```

새 데이터베이스에서 처음 실행했다면 다음처럼 자동 생성된 ID를 확인할 수 있습니다.

```text
1 Deadpond None
```

같은 파일 DB에서 예제를 반복하면 ID는 2 이상이 될 수 있습니다. `commit()` 전에 오류가 나면 세션 컨텍스트를 빠져나가기 전에 `rollback()`해야 같은 세션을 계속 사용할 수 있으며, 비밀 필드를 응답으로 직렬화하지 않도록 별도 응답 모델을 둡니다.
