title: "Part 2. 중급: 레이아웃과 재사용 UI 구성하기"
---

여러 화면이 같은 헤더·내비게이션·폼을 사용하면 HTML을 복사하는 대신 템플릿 상속, include, macro로 공통 구조를 분리합니다. 이 노트에서는 사진 블로그의 목록과 상세 화면을 확장하면서 템플릿별 책임과 FastAPI가 제공할 공통 context를 정리합니다.

## extends와 block으로 공통 레이아웃 만들기

여러 페이지의 문서 구조를 한곳에서 관리할 때 기본 레이아웃을 만들고 페이지 템플릿이 필요한 블록만 채우게 합니다. 템플릿 디렉터리를 역할별로 나누면 파일 이름만으로 용도를 찾기 쉽습니다.

```text
templates/
├── layouts/
│   └── base.html
├── pages/
│   ├── post-list.html
│   └── post-detail.html
├── partials/
│   ├── navigation.html
│   └── post-card.html
└── macros/
    └── forms.html
```

```html
<!-- templates/layouts/base.html -->
<!doctype html>
<html lang="ko">
  <head>
    <meta charset="utf-8">
    <title>{% block title %}사진 블로그{% endblock %}</title>
  </head>
  <body>
    <main>{% block content %}{% endblock %}</main>
  </body>
</html>
```

```html
<!-- templates/pages/post-list.html -->
{% extends 'layouts/base.html' %}

{% block title %}포스트 목록 | 사진 블로그{% endblock %}

{% block content %}
  <h1>포스트 목록</h1>
{% endblock %}
```

`extends`는 첫 번째 실행 태그로 두고, 자식 템플릿의 HTML은 부모가 선언한 `block` 안에 작성합니다. FastAPI는 `pages/post-list.html`만 렌더링하면 상속된 전체 문서를 응답합니다.

## super로 부모 블록의 기본 내용을 유지하기

부모 블록을 완전히 교체하지 않고 기본 내용을 앞뒤에 보탤 때 `super()`를 호출합니다. 공통 CSS를 유지하면서 페이지 전용 CSS를 추가하는 경우가 대표적입니다.

```html
<!-- layouts/base.html -->
{% block styles %}
  <link rel="stylesheet" href="{{ url_for('static', path='/styles.css') }}">
{% endblock %}
```

```html
<!-- pages/post-detail.html -->
{% block styles %}
  {{ super() }}
  <link rel="stylesheet" href="{{ url_for('static', path='/post-detail.css') }}">
{% endblock %}
```

`super()`가 없으면 자식의 `styles` 블록이 부모 블록을 모두 교체합니다. 전역 자산과 페이지 자산의 적용 순서를 템플릿에서 명시할 수 있습니다.

## include로 내비게이션과 포스트 카드 조립하기

현재 페이지의 context를 그대로 사용하는 작은 HTML 조각은 `include`로 분리합니다. 목록 안에서 카드 partial을 포함하면 카드 모양을 한 파일에서 바꿀 수 있습니다.

```html
<!-- templates/partials/navigation.html -->
<nav aria-label="주요 메뉴">
  <a href="{{ url_for('post_list') }}">포스트</a>
  <a href="{{ url_for('post_new_form') }}">새 포스트</a>
</nav>
```

```html
<!-- templates/partials/post-card.html -->
<article>
  <h2>
    <a href="{{ url_for('post_detail', post_id=post.id) }}">{{ post.title }}</a>
  </h2>
  <p>{{ post.content|truncate(80) }}</p>
</article>
```

```html
{% include 'partials/navigation.html' %}

{% for post in posts %}
  {% include 'partials/post-card.html' %}
{% else %}
  <p>아직 작성된 포스트가 없습니다.</p>
{% endfor %}
```

include는 기본적으로 현재 context의 `post`, `posts`, `url_for`에 접근합니다. 독립적인 조각이어야 한다면 `without context`를 붙이고 필요한 데이터를 다른 방식으로 명시합니다.

## macro로 폼 필드와 오류 표시 재사용하기

인수에 따라 조금씩 달라지는 HTML을 반복할 때 macro를 템플릿 함수처럼 정의합니다. 작성과 수정 화면이 같은 입력 필드와 오류 모양을 공유하기에 적합합니다.

```html
<!-- templates/macros/forms.html -->
{% macro text_field(name, label, value='', error='') %}
  <div class="field">
    <label for="{{ name }}">{{ label }}</label>
    <input id="{{ name }}" name="{{ name }}" value="{{ value }}">
    {% if error %}
      <p class="field-error">{{ error }}</p>
    {% endif %}
  </div>
{% endmacro %}
```

```html
{% from 'macros/forms.html' import text_field %}

<form method="post">
  {{ text_field('title', '제목', values.title, errors.title) }}
  {{ text_field('thumbnail_url', '썸네일 URL', values.thumbnail_url, errors.thumbnail_url) }}
  <button type="submit">저장</button>
</form>
```

macro가 반환한 HTML은 안전한 템플릿 결과로 취급되지만, 인수로 받은 `value`와 `error`는 계속 자동 이스케이프됩니다.

## 내장 필터로 목록과 텍스트 표시 다듬기

Python 데이터를 바꾸지 않고 화면의 길이·정렬·표시만 조절할 때 내장 필터를 조합합니다.

```html
<p>전체 {{ posts|length }}개</p>

{% for post in posts|sort(attribute='title') %}
  <h2>{{ post.title|trim }}</h2>
  <p>{{ post.content|striptags|truncate(100) }}</p>
{% endfor %}
```

`striptags`는 표시용 텍스트에서 태그를 제거하지만 신뢰할 수 없는 HTML을 안전하게 보관하는 도구는 아닙니다. 데이터베이스 조회 순서가 사용자에게 의미가 있다면 Python 쿼리에서 정렬하고, 필터는 화면 표현에만 사용합니다.

## env.filters로 날짜 표시 규칙 등록하기

여러 템플릿이 같은 Python 변환 규칙을 사용할 때 함수를 Jinja environment의 커스텀 필터로 등록합니다. environment 설정은 첫 템플릿을 렌더링하기 전에 완료합니다.

```python
from datetime import datetime

from fastapi.templating import Jinja2Templates


def korean_date(value: datetime) -> str:
    return value.strftime('%Y년 %m월 %d일')


templates = Jinja2Templates(directory='templates')
templates.env.filters['korean_date'] = korean_date
```

```html
<time datetime="{{ post.created_at.isoformat() }}">
  {{ post.created_at|korean_date }}
</time>
```

```text
2026년 07월 21일
```

필터는 전달받은 값을 문자열로 바꾸기만 하고 데이터베이스나 외부 API를 호출하지 않습니다. I/O는 경로 함수에서 끝내야 렌더링 시점의 지연과 실패를 예측할 수 있습니다.

## context_processors로 모든 화면에 공통 값 전달하기

앱 이름이나 현재 연도처럼 모든 화면에 필요한 값은 각 경로의 context에 반복하지 않고 context processor로 제공합니다. processor는 요청 하나를 받아 동기적으로 사전을 반환합니다.

```python
from datetime import datetime, timezone

from fastapi import Request
from fastapi.templating import Jinja2Templates


def common_context(request: Request) -> dict[str, object]:
    return {
        'site_name': '사진 블로그',
        'current_year': datetime.now(timezone.utc).year,
    }


templates = Jinja2Templates(
    directory='templates',
    context_processors=[common_context],
)
```

```html
<footer>© {{ current_year }} {{ site_name }}</footer>
```

context processor는 비동기 함수를 지원하지 않으므로 사용자·데이터베이스 정보는 의존성과 경로 함수에서 읽어 명시적인 context로 전달합니다.

공식 Jinja [Template Inheritance](https://jinja.palletsprojects.com/en/stable/templates/#template-inheritance), [Include](https://jinja.palletsprojects.com/en/stable/templates/#include), [Macros](https://jinja.palletsprojects.com/en/stable/templates/#macros), Starlette [Custom filters와 context processors](https://www.starlette.io/templates/)를 기준으로 구성했습니다.
