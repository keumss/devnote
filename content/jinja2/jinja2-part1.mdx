---
title: "Part 1. 기초: FastAPI에서 HTML 템플릿 렌더링하기"
---

Jinja2는 Python 데이터로 HTML을 만들어 서버에서 완성된 화면을 응답할 때 사용하는 템플릿 엔진입니다. FastAPI가 데이터 조회와 요청 처리를 맡고 Jinja2가 표시를 맡게 하면, 별도 프런트엔드 앱 없이도 블로그 목록·상세·관리 화면을 만들 수 있습니다. 이 노트에서는 작은 포스트 목록을 렌더링하며 템플릿의 기본 문법과 FastAPI 연결 방법을 익힙니다.

## Jinja2Templates와 TemplateResponse로 첫 HTML 응답 만들기

FastAPI에서 HTML 템플릿을 렌더링할 때 `jinja2`와 FastAPI의 표준 의존성을 설치합니다. Python 3.12 가상 환경을 만들고 템플릿과 정적 파일 디렉터리도 준비합니다.

```bash
python3.12 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install 'fastapi[standard]' jinja2
mkdir templates static
```

```text
.
├── main.py
├── static/
└── templates/
```

Jinja2는 HTML 문자열을 만들고, FastAPI가 제공하는 `Jinja2Templates`는 템플릿 디렉터리·요청·응답을 연결합니다. 브라우저에 HTML 화면을 보낼 때 애플리케이션 시작 시 `Jinja2Templates`를 한 번 만들고 각 경로에서 재사용합니다. `TemplateResponse`에는 현재 요청, 템플릿 파일 이름, 템플릿이 사용할 context를 전달합니다.

```python
# main.py
from fastapi import FastAPI, Request
from fastapi.responses import HTMLResponse
from fastapi.templating import Jinja2Templates


app = FastAPI()
templates = Jinja2Templates(directory='templates')


@app.get('/', response_class=HTMLResponse)
def home(request: Request):
    return templates.TemplateResponse(
        request=request,
        name='home.html',
        context={'page_title': '사진 블로그'},
    )
```

```html
<!-- templates/home.html -->
<!doctype html>
<html lang="ko">
  <head>
    <meta charset="utf-8">
    <title>{{ page_title }}</title>
  </head>
  <body>
    <h1>{{ page_title }}</h1>
  </body>
</html>
```

```bash
fastapi dev main.py
```

브라우저에서 `http://127.0.0.1:8000`을 열면 제목과 본문에 `사진 블로그`가 표시됩니다. `response_class=HTMLResponse`는 실제 응답을 대신 만들지 않고 API 문서에 HTML 응답임을 알려 줍니다.

## 변수 표현식으로 포스트 속성 출력하기

서버가 전달한 객체의 필드를 HTML에 넣을 때 `{{ ... }}` 변수 표현식을 사용합니다. 사전의 키와 객체의 속성 모두 점 표기법으로 읽을 수 있습니다.

```python
@app.get('/posts/1', response_class=HTMLResponse)
def post_detail(request: Request):
    post = {
        'title': '봄 산책',
        'author': {'name': '민수'},
        'content': '벚꽃이 피었습니다.',
    }
    return templates.TemplateResponse(
        request=request,
        name='post-detail.html',
        context={'post': post},
    )
```

```html
<!-- templates/post-detail.html -->
<article>
  <h1>{{ post.title }}</h1>
  <p>작성자: {{ post.author.name }}</p>
  <p>{{ post.content }}</p>
</article>
```

```text
봄 산책
작성자: 민수
벚꽃이 피었습니다.
```

템플릿은 전달받은 값을 표시하는 데 집중하고, 데이터베이스 조회나 권한 판단은 FastAPI 경로 함수에서 끝냅니다.

## if로 썸네일과 빈 상태 나누기

값의 존재 여부에 따라 다른 HTML을 보여 줄 때 `{% if %}` 블록을 사용합니다. 썸네일 URL이 없으면 깨진 이미지 대신 대체 문구를 표시합니다.

```html
{% if post.thumbnail_url %}
  <img src="{{ post.thumbnail_url }}" alt="{{ post.title }} 썸네일">
{% else %}
  <p>등록된 썸네일이 없습니다.</p>
{% endif %}
```

`None`, 빈 문자열, 빈 목록은 조건에서 거짓으로 평가됩니다. 데이터의 의미를 바꾸는 복잡한 분기는 Python에서 결정하고, 템플릿에는 `can_edit`처럼 화면이 바로 사용할 값을 넘깁니다.

## for와 else로 포스트 목록 렌더링하기

여러 포스트를 같은 카드 모양으로 출력할 때 `{% for %}`로 목록을 순회합니다. 반복할 항목이 없을 때는 같은 블록의 `{% else %}`로 빈 화면을 처리할 수 있습니다.

```python
@app.get('/posts', response_class=HTMLResponse)
def post_list(request: Request):
    posts = [
        {'id': 1, 'title': '봄 산책'},
        {'id': 2, 'title': '비 오는 날'},
    ]
    return templates.TemplateResponse(
        request=request,
        name='post-list.html',
        context={'posts': posts},
    )
```

```html
<!-- templates/post-list.html -->
<h1>포스트</h1>
<ul>
  {% for post in posts %}
    <li>{{ loop.index }}. {{ post.title }}</li>
  {% else %}
    <li>아직 작성된 포스트가 없습니다.</li>
  {% endfor %}
</ul>
```

```text
포스트
1. 봄 산책
2. 비 오는 날
```

`loop.index`는 1부터 시작하는 현재 순서입니다. 정렬과 페이지 나누기는 템플릿이 아니라 데이터 조회 단계에서 처리합니다.

## 필터로 표시 형식만 변환하기

문자열의 대소문자, 기본값, 목록 길이처럼 화면 표시만 바꿀 때 `|` 뒤에 필터를 연결합니다.

```html
<h1>{{ post.title|title }}</h1>
<p>{{ post.summary|default('요약이 없습니다.', true) }}</p>
<p>사진 {{ post.images|length }}장</p>
```

`default(..., true)`의 두 번째 인수는 변수가 정의되지 않은 경우뿐 아니라 빈 문자열 같은 거짓 값에도 기본 문구를 적용합니다. 날짜·통화처럼 프로젝트 규칙이 필요한 변환은 Part 2에서 커스텀 필터로 분리합니다.

## url_for로 경로 이름에서 링크 만들기

라우트 경로가 바뀌어도 템플릿 링크를 함께 유지하려면 문자열 URL 대신 `url_for()`를 사용합니다. FastAPI의 `Request`를 전달한 `TemplateResponse`에서는 Jinja2 context에 `url_for`가 자동으로 포함됩니다.

```python
@app.get('/posts/{post_id}', name='post_detail', response_class=HTMLResponse)
def post_detail(request: Request, post_id: int):
    post = {'id': post_id, 'title': '봄 산책'}
    return templates.TemplateResponse(
        request=request,
        name='post-detail.html',
        context={'post': post},
    )
```

```html
<a href="{{ url_for('post_detail', post_id=post.id) }}">
  {{ post.title }}
</a>
```

`post_detail`은 경로 함수의 `name`이고 `post_id`는 경로 매개변수 이름입니다. 템플릿 파일 이름과 라우트 이름은 서로 다른 역할을 합니다.

## StaticFiles로 CSS를 제공하기

HTML과 함께 CSS·이미지 같은 정적 파일을 제공할 때 `StaticFiles`를 별도 경로에 마운트합니다.

```python
from fastapi.staticfiles import StaticFiles


app.mount('/static', StaticFiles(directory='static'), name='static')
```

```css
/* static/styles.css */
body {
  max-width: 48rem;
  margin: 2rem auto;
  font-family: sans-serif;
}
```

```html
<link rel="stylesheet" href="{{ url_for('static', path='/styles.css') }}">
```

첫 번째 `/static`은 브라우저가 요청할 URL이고, `directory`는 서버가 파일을 찾을 디렉터리입니다. `name='static'`을 지정했으므로 템플릿에서도 `url_for('static', ...)`로 경로를 만들 수 있습니다.

## 자동 이스케이프로 사용자 입력을 안전하게 출력하기

사용자가 작성한 제목과 본문을 HTML에 넣을 때 태그가 코드로 실행되지 않도록 이스케이프해야 합니다. `Jinja2Templates(directory=...)`는 `.html`, `.htm`, `.xml` 템플릿의 자동 이스케이프를 기본으로 켭니다.

```python
post = {
    'title': '<script>alert("xss")</script>',
    'content': '사용자가 입력한 본문',
}
```

```html
<h1>{{ post.title }}</h1>
```

```text
<h1>&lt;script&gt;alert(&#34;xss&#34;)&lt;/script&gt;</h1>
```

브라우저에는 태그 문자가 제목 텍스트로 보이고 스크립트는 실행되지 않습니다. 사용자 입력에 `|safe`를 적용하면 이 보호를 제거하므로 사용하지 않습니다.
