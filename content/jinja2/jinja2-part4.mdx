## 화면용 데이터를 라우터에서 준비하기

템플릿에서 ORM 객체를 탐색하거나 DB를 조회하면 화면이 느려지고 테스트하기 어려워집니다. 라우터에서 화면에 필요한 값만 딕셔너리로 준비해 전달하면 템플릿은 표시 역할에만 집중합니다.

```python
from fastapi import Request

@app.get('/posts/{post_id}', name='render_post')
def render_post(post_id: int, request: Request):
    post = find_post_or_404(post_id)
    view_model = {
        'title': post.title,
        'author_name': post.author.name,
        'published_at': post.created_at.strftime('%Y-%m-%d'),
    }
    return templates.TemplateResponse(
        request=request,
        name='post_detail.html',
        context={'post': view_model},
    )
```

## 템플릿에서 URL과 정적 파일 만들기

라우트와 정적 파일의 주소는 배포 경로가 달라져도 동작하도록 헬퍼로 생성합니다. FastAPI에서 정적 폴더를 마운트한 뒤 `url_for()`로 CSS와 링크를 참조합니다.

```python
from fastapi.staticfiles import StaticFiles

app.mount('/static', StaticFiles(directory='app/static'), name='static')

@app.get('/', name='render_home')
def render_home(request: Request):
    return templates.TemplateResponse(request=request, name='index.html')
```

```html
<link rel="stylesheet" href="{{ url_for('static', path='style.css') }}">
<a href="{{ url_for('render_home') }}">홈</a>
```

## 응답으로 렌더링 결과 테스트하기

템플릿 화면은 상태 코드, 표시 텍스트, 주요 링크를 함께 확인합니다. 테스트 데이터는 별도로 준비해 실제 DB나 네트워크에 의존하지 않게 만듭니다.

```python
from fastapi.testclient import TestClient

client = TestClient(app)

def test_home_renders_empty_state() -> None:
    response = client.get('/')

    assert response.status_code == 200
    assert '현재 등록된 글이 없습니다.' in response.text
    assert 'href="/static/style.css"' in response.text
```

## 템플릿 오류를 운영에서 빠르게 찾기

배포 전에 실제 템플릿을 렌더링하는 테스트를 실행하고, 템플릿 파일 누락이나 변수 오타는 개발 환경에서 즉시 실패하게 설정합니다. 사용자에게는 상세 예외 대신 일반 오류 화면을 보여 주고, 원인은 서버 로그로 추적합니다.

```python
from jinja2 import Environment, FileSystemLoader, StrictUndefined

template_env = Environment(
    loader=FileSystemLoader('app/templates'),
    autoescape=True,
    undefined=StrictUndefined,
)

# 개발·테스트에서 누락된 변수를 즉시 발견합니다.
template_env.get_template('post_detail.html').render(post={'title': '예시'})
```
